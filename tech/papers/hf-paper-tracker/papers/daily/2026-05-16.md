### [T1] Long Context Pre-Training with Lighthouse Attention
- URL: https://arxiv.org/abs/2605.06554
- 重要度: 🔵Medium
- 要約:
  1. [課題] 極端な長文脈の事前学習は通常の自己注意の二乗コストがボトルネックになる。
  2. [手法] 学習中だけ階層的に系列を圧縮・復元する Lighthouse Attention を被せ、終盤でフルアテンションに戻す二段階学習を提案。
  3. [結果] 同条件のフルアテンション学習より総学習時間を短縮しつつ、復帰後の最終損失も改善した。
- 一言メモ: 長文の社内文書や決算資料を扱う基盤モデルの事前学習コストを抑える発想として使える。

### [T1] BEAM: Binary Expert Activation Masking for Dynamic Routing in MoE
- URL: https://arxiv.org/abs/2605.14438
- 重要度: ⚡High
- 要約:
  1. [課題] 固定 Top-K の MoE ルーティングは不要な expert 発火を招き、推論レイテンシを押し上げる。
  2. [手法] 学習可能な二値マスクで token ごとに expert sparsity を動的制御し、CUDA kernel と vLLM 統合まで含めて実装した。
  3. [結果] 元性能の98%以上を維持しつつ MoE FLOPs を最大85%削減し、デコード 2.5倍高速化・スループット 1.4倍を報告した。
- 一言メモ: Qwen3.5-397B-A17B級を PCIe 環境で回す前提にかなり近く、最優先で追う価値がある。

### [T1] Forcing-KV: Hybrid KV Cache Compression for Efficient Autoregressive Video Diffusion Models
- URL: https://arxiv.org/abs/2605.09681
- 重要度: 🔵Medium
- 要約:
  1. [課題] 自己回帰型生成では履歴フレームの KV cache が肥大化し、長系列で速度とメモリを圧迫する。
  2. [手法] attention head を静的役割と動的役割に分け、前者は構造的 pruning、後者は類似度ベース圧縮を行うハイブリッド KV 圧縮を導入した。
  3. [結果] 品質を維持しつつ cache メモリを30%削減し、条件によって 1.35倍から 2.82倍の高速化を達成した。
- 一言メモ: 動画向けだが、head ごとの役割分化に応じて KV を圧縮する考え方は長文推論の最適化に転用余地がある。

### [T2] WildTableBench: Benchmarking Multimodal Foundation Models on Table Understanding In the Wild
- URL: https://arxiv.org/abs/2605.01018
- 重要度: 🔵Medium
- 要約:
  1. [課題] 実世界の表画像はレイアウト崩れや複雑な数値推論を含むのに、既存評価は整形済み表やレンダリング表に偏っている。
  2. [手法] 多様な自然画像テーブル402枚と928問からなる in-the-wild 表理解ベンチマークを構築した。
  3. [結果] 21モデルを評価したところ 50%超は1モデルのみで、構造把握と数値推論の弱さが明確に残った。
- 一言メモ: 決算資料や IR スライド中の表を読む系タスクの現状把握に直結する。

### [T2] PreScam: A Benchmark for Predicting Scam Progression from Early Conversations

### [T3] WildClawBench: A Benchmark for Real-World, Long-Horizon Agent Evaluation
- URL: https://arxiv.org/abs/2605.10912
- 重要度: ⚡High
- 要約:
  1. [課題] 既存エージェント評価は短手数・模擬API・合成環境に寄り、実運用の CLI 長期タスクを十分に測れていない。
  2. [手法] 実ツールを使う native-runtime の 60 タスクを Docker 化し、ルール判定と状態監査と LLM/VLM 判定を組み合わせて評価した。

### [T3] STALE: Can LLM Agents Know When Their Memories Are No Longer Valid?
- URL: https://arxiv.org/abs/2605.06527
- 重要度: ⚡High
- 要約:
  1. [課題] 長期記憶エージェントは更新後の事実を保持しても、古い前提を無効化して振る舞いを変えるのが苦手である。
  2. [手法] 暗黙的な状態変化を含む 400 シナリオのベンチマークを作り、古い記憶の解消・誤前提拒否・更新後行動適応を測定した。
  3. [結果] 最良モデルでも総合 55.2% に留まり、構造化された状態統合を行う CUPMem が改善余地を示した。
- 一言メモ: 顧客属性やポートフォリオ条件が変化する金融対話では、古い前提の残留が事故につながるので重要。

- 要約:

### [T3] PREPING: Building Agent Memory without Tasks
- URL: https://arxiv.org/abs/2605.13880
- 重要度: 🔵Medium
- 要約:
  1. [課題] 新環境に入ったエージェントはタスク経験がない初期段階で memory cold-start に陥る。
  2. [手法] Proposer が合成練習タスクを生成し、Solver と Validator を通じて有用な軌跡だけを記憶に入れる pre-task memory 構築法を提案した。

- 一言メモ: 新しい社内ツールや MCP を追加した直後の立ち上がりを早める設計として相性が良い。

### [T3/T4] Self-Distilled Agentic Reinforcement Learning
- URL: https://arxiv.org/abs/2605.15155
- 重要度: ⚡High
- 要約:
  1. [課題] エージェント RL は軌跡報酬が粗く、通常の OPSD を多ターン agent に持ち込むと教師信号が不安定になりやすい。
  2. [手法] RL を主軸に据えつつ、教師の肯定的ギャップ token を強める gated self-distillation を補助損失として加える SDAR を提案した。
  3. [結果] ALFWorld・WebShop・Search-QA で GRPO を一貫して上回り、単純な GRPO+OPSD の不安定さも回避した。
- 一言メモ: 長手数の金融エージェントを強化するなら、RL 単独より token-level 蒸留併用が有力という示唆が強い。

### [T4] Achieving Gold-Medal-Level Olympiad Reasoning via Simple and Unified Scaling
- URL: https://arxiv.org/abs/2605.13301
- 重要度: ⚡High
- 要約:
  1. [課題] 難度の高い厳密推論では、長い推論軌跡を安定して学習・強化する統一レシピが不足している。

- 一言メモ: 構造化出力品質を上げたいなら、単なる instruction tuning ではなく curriculum 付き SFT と段階的 RL の設計が効きそうだ。

### [T4] Adaptive Teacher Exposure for Self-Distillation in LLM Reasoning
- URL: https://arxiv.org/abs/2605.11458
- 重要度: 🔵Medium
- 要約:
  1. [課題] 自己蒸留では教師が常に完全な参照推論を見ているため、学生能力との差が開きすぎて学習効率を下げる。
  2. [手法] 教師に見せる推論量を固定せず、学習状態に応じて露出率を調整する controller を導入した。
  3. [結果] Qwen3 1.7B/4B/8B で既存 self-distillation と RL ベースラインを安定して上回った。
- 一言メモ: 金融ドメイン SFT 後の reasoning 蒸留でも、教師情報を出しすぎない調整が過学習抑制に効きそうだ。
