### [T1] REAM: Merging Improves Pruning of Experts in LLMs
- URL: https://arxiv.org/abs/2604.04356
- 重要度: 🔵Medium
- 要約:
  1. [課題] 大規模 MoE の省メモリ化では expert 削除が性能劣化を招きやすく、特に生成性能を保った圧縮が難しい。
  2. [手法] expert を単純に落とす代わりに router 重みを使って類似 expert をグループ化し、重みを統合する REAM を提案した。
  3. [結果] 複数の MoE LLM で REAP などの既存 pruning 系をしばしば上回り、未圧縮モデルに近い性能も達成した。
- 一言メモ: PCIe 前提の MoE 運用では、expert 削減より expert 統合の方が品質を保ってメモリを削れる可能性がある。

### [T3] ClawsBench: Evaluating Capability and Safety of LLM Productivity Agents in Simulated Workspaces
- URL: https://arxiv.org/abs/2604.05172
- 重要度: ⚡High
- 要約:
  1. [課題] 実サービス上の agent 評価は危険で、既存ベンチも stateful な横断ワークフローや安全性を十分に再現できていない。
  2. [手法] Gmail・Slack・Calendar・Docs・Drive を模した高忠実度 mock 環境と 44 の multi-service task を持つ ClawsBench を構築した。
  3. [結果] 強い scaffold を与えても成功率は 39-64% に留まり、unsafe action rate も 7-33% と高く、8 種の危険行動パターンを整理した。
- 一言メモ: 金融エージェントでも本番接続前に、権限付き業務ツールを模した復元可能 sandbox 評価が必須だ。

### [T2] CUE-R: Beyond the Final Answer in Retrieval-Augmented Generation
- URL: https://arxiv.org/abs/2604.05467
- 重要度: 🔵Medium
- 要約:
  1. [課題] RAG 評価は最終回答中心で、各 retrieved evidence が実際にどれだけ推論へ寄与したかを測れていない。
  2. [手法] evidence を REMOVE・REPLACE・DUPLICATE で介入し、正答性・grounding・confidence error・trace divergence を測る CUE-R を提案した。
  3. [結果] multi-hop QA で重要 evidence の除去が大きな性能低下を生み、回答だけでは見えない証拠間相互作用を確認した。
- 一言メモ: 社内文書 RAG では「どの資料片が効いたか」を見る評価軸として、そのまま導入候補になる。

### [T3] Squeez: Task-Conditioned Tool-Output Pruning for Coding Agents
- URL: https://arxiv.org/abs/2604.04979
- 重要度: 🔵Medium
- 要約:
  1. [課題] coding agent は毎回長大な tool 出力をそのまま読み直し、文脈肥大とレイテンシ増大を招いている。
  2. [手法] 次の行動に必要な最小証拠ブロックだけを抽出する task-conditioned pruning 問題を定義し、LoRA 微調整モデルを学習した。
  3. [結果] 92% の入力 token を削減しつつ recall 0.86、F1 0.80 を達成し、大型 zero-shot や heuristic を上回った。
- 一言メモ: ツール出力の刈り込みは、長い企業文書や API 応答を扱う金融エージェントでも即効性が高い。

### [T4] DARE: Diffusion Large Language Models Alignment and Reinforcement Executor
- URL: https://arxiv.org/abs/2604.04215
- 重要度: 🔵Medium
- 要約:
  1. [課題] diffusion LLM の post-training 基盤はモデル族ごとに分断されており、SFT・PEFT・preference optimization・RL の比較再現が難しい。
  2. [手法] verl と OpenCompass を土台に、masked/block diffusion 系を横断して後学習と評価を共通実行できる DARE を整備した。
  3. [結果] LLaDA 系など複数ファミリで広いアルゴリズム対応と実用的な高速化を示し、dLLM 研究の共通基盤を提供した。
- 一言メモ: diffusion 系を触るなら、金融向け alignment 実験の再現基盤として価値がある。

### [T1] MegaTrain: Full Precision Training of 100B+ Parameter Large Language Models on a Single GPU
- URL: https://arxiv.org/abs/2604.05091
- 重要度: 🔵Medium
- 要約:
  1. [課題] 100B 超 LLM 学習は GPU メモリがボトルネックで、CPU offload 系も帯域制約で効率が出にくい。
  2. [手法] パラメータと optimizer state を host memory に置き、GPU を一時的計算装置として使う memory-centric 学習系と多 stream overlap を設計した。
  3. [結果] 単一 H200 で 120B 学習、14B では ZeRO-3 CPU offload 比 1.84 倍の throughput を達成した。
- 一言メモ: 直接の推論運用とは違うが、PCIe 帯域を前提にした重みストリーミング設計として示唆がある。

### [T3] How Well Do Agentic Skills Work in the Wild: Benchmarking LLM Skill Usage in Realistic Settings

- 重要度: 🔵Medium

- 一言メモ: 長い企業資料や時系列市場情報へ推論時に追随する仕組みとして面白いが、運用複雑性は高い。
