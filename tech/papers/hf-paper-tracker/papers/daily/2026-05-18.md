### [T1] Long Context Pre-Training with Lighthouse Attention
- URL: https://arxiv.org/abs/2605.06554
- 重要度: 🔵Medium
- 要約:
  1. [課題] 超長文コンテキストの事前学習は通常 attention の二乗コストが大きく、学習時間とメモリが重すぎる。
  2. [手法] 学習中だけ系列を階層圧縮してから終盤でフル attention に戻す Lighthouse Attention を被せ、勾配不要の対称圧縮で並列性を確保した。
  3. [結果] 同条件のフル attention 学習より総学習時間を短縮しつつ、最終 loss も改善した。
- 一言メモ: 長い社内文書や決算資料を扱う基盤モデルの継続事前学習コストを下げる候補として現実的。

### [T1] BEAM: Binary Expert Activation Masking for Dynamic Routing in MoE
- URL: https://arxiv.org/abs/2605.14438
- 重要度: ⚡High
- 要約:
  1. [課題] 固定 Top-K の MoE ルーティングは不要な expert 発火を生み、推論レイテンシと FLOPs を押し上げる。
  2. [手法] 学習可能な二値マスクで token ごとに expert 活性を動的制御し、CUDA kernel と vLLM 統合まで実装した。
  3. [結果] 元性能の 98% 以上を維持しつつ MoE 層 FLOPs を最大 85% 削減し、最大 2.5 倍の decoding 高速化を示した。
- 一言メモ: PCIe 接続前提の MoE 推論最適化に直結しており、現行インフラ制約へ最も近い改善候補。

### [T1] Forcing-KV: Hybrid KV Cache Compression for Efficient Autoregressive Video Diffusion Models
- URL: https://arxiv.org/abs/2605.09681
- 重要度: 🔵Medium
- 要約:
  1. [課題] 自己回帰生成では履歴の KV cache が肥大化し、速度とメモリを圧迫する。
  2. [手法] attention head を静的役割と動的役割に分け、片方は構造 pruning、もう片方は類似度ベース圧縮を行うハイブリッド KV 圧縮を導入した。
  3. [結果] 品質を保ちながら cache メモリを削減し、条件によって 1.35 倍から 2.82 倍の高速化を達成した。
- 一言メモ: 動画向けだが、head の役割別に KV を差別化圧縮する発想は長文 LLM 推論にも転用余地がある。

### [T2] WildTableBench: Benchmarking Multimodal Foundation Models on Table Understanding In the Wild
- URL: https://arxiv.org/abs/2605.01018
- 重要度: ⚡High
- 要約:
  1. [課題] 実世界の表画像は複雑なレイアウトと数値推論を含むが、既存評価は整形済み表やクリーン画像に偏っていた。
  2. [手法] 自然画像の表 402 枚と 928 問からなる in-the-wild 表理解ベンチマークを構築した。
  3. [結果] 21 モデル中 50% 超の精度は 1 モデルだけで、構造把握と数値推論の弱さが依然大きいと示した。
- 一言メモ: 決算資料や IR スライド内の表理解そのものなので、金融文書読解の評価設計に直結する。

### [T3] RouteProfile: Elucidating the Design Space of LLM Profiles for Routing
- URL: https://arxiv.org/abs/2605.00180
- 重要度: ⚡High
- 要約:
  1. [課題] LLM routing 研究は router 自体に注目しがちで、各モデル能力を表す profile 設計が十分整理されていなかった。

- 一言メモ: タスクごとに複数モデルやサブエージェントを振り分ける金融オーケストレーションで効く。

### [T3] STALE: Can LLM Agents Know When Their Memories Are No Longer Valid?
- URL: https://arxiv.org/abs/2605.06527
- 重要度: ⚡High
- 要約:
  1. [課題] 長期記憶エージェントは更新後の事実を保持しても、古い前提を無効化して行動を変えるのが苦手である。
  2. [手法] 暗黙的な状態変化を含む 400 シナリオのベンチマークを作り、古い記憶の解消・誤前提拒否・更新後行動適応を測定した。
  3. [結果] 最良モデルでも総合 55.2% に留まり、構造化状態統合の初期ベースラインが有効であることを示した。
- 一言メモ: 顧客属性や保有前提が変わる金融対話で、古い前提の残留を防ぐ設計に直結する。

### [T4] Adaptive Teacher Exposure for Self-Distillation in LLM Reasoning
- URL: https://arxiv.org/abs/2605.11458
- 重要度: 🔵Medium
- 要約:
  1. [課題] 自己蒸留では教師が完全な参照推論を見すぎるため、学生との能力差が開いて学習効率を落としやすい。
  2. [手法] 教師に見せる推論量を固定せず、学習状態に応じて露出率を調整する controller を導入した。

- 一言メモ: 金融ドメイン SFT 後の reasoning 蒸留でも、教師情報を出しすぎない設計が有効そうだ。

### [T4] Achieving Gold-Medal-Level Olympiad Reasoning via Simple and Unified Scaling
- URL: https://arxiv.org/abs/2605.13301
- 重要度: ⚡High
- 要約:
  1. [課題] 難度の高い厳密推論では、長い推論軌跡を安定して学習・強化する統一レシピが不足していた。
  2. [手法] reverse-perplexity curriculum による SFT、verifiable reward RL、proof-level RL、test-time scaling を一体化した単純な recipe を提示した。
  3. [結果] 30B-A3B backbone で 100K token 超の長軌跡を安定化し、数理・科学オリンピックで金メダル級性能を達成した。
- 一言メモ: 構造化出力品質を上げるにも、単なる instruction tuning でなく curriculum 付き SFT と段階的 RL が効く可能性が高い。

- URL: https://arxiv.org/abs/2605.15155
