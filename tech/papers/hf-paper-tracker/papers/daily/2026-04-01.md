### [T1] TAPS: Task Aware Proposal Distributions for Speculative Sampling
- URL: https://arxiv.org/abs/2603.27027
- 重要度: ⚡High
- 要約:
  1. [課題] Speculative decoding は下位 draft model の学習分布が下流タスクに合わないと受理長が伸びず、速度向上が頭打ちになる。
  2. [手法] 数学向け・対話向けなどタスク特化 drafter を用意し、推論時は confidence ベース routing と merged-tree verification で使い分ける。
  3. [結果] HASS/EAGLE-2 系で単一 drafter や checkpoint averaging を上回り、タスクごとの acceptance length を一貫して改善した。
- 一言メモ: PCIe 環境での推論高速化に直結し、金融QAと汎用対話で drafter を分ける設計の根拠になる。

### [T1] On Token's Dilemma: Dynamic MoE with Drift-Aware Token Assignment for Continual Learning of Large Vision Language Models
- URL: https://arxiv.org/abs/2603.27481
- 重要度: 🔵Medium
- 要約:
  1. [課題] 継続学習で expert を追加する MoE は expert 分離があっても routing drift により旧タスク忘却が起きる。
  2. [手法] LLaVA-DyMoE は token ごとの routing score 分布から曖昧 token を識別し、新 expert への流入を抑える drift-aware assignment と正則化を導入する。
  3. [結果] ベースライン比で平均最終精度を 7%以上改善し、forgetting を 12%削減した。
- 一言メモ: 金融 LoRA 継ぎ足し運用でも router 安定化が重要で、MoE 継続学習の実装ガードレールとして参考になる。

### [T1] Density-aware Soft Context Compression with Semi-Dynamic Compression Ratio
- URL: https://arxiv.org/abs/2603.25926
- 重要度: 🔵Medium
- 要約:
  1. [課題] 長文コンテキスト圧縮は一律 compression ratio だと情報密度の差を扱えず、要点の多い区間で性能が落ちる。
  2. [手法] 情報密度から離散的な圧縮率を選ぶ Discrete Ratio Selector を導入し、連続値制御ではなく量子化された比率で compressor を共同学習する。
  3. [結果] 静的圧縮のベースラインを上回り、性能と圧縮率の Pareto frontier を改善した。
- 一言メモ: 決算短信や社内メモの密度差が大きい RAG 前処理にそのまま応用しやすい。

### [T2] SEAR: Schema-Based Evaluation and Routing for LLM Gateways
- URL: https://arxiv.org/abs/2603.26728
- 重要度: ⚡High
- 要約:
  1. [課題] 本番 LLM gateway では品質評価と provider routing を同時に扱う必要があるが、細粒度で構造化された評価信号が不足している。
  2. [手法] 文脈・意図・応答特性・障害要因・コスト/遅延を約100列の relational schema に落とし込み、LLM 推論で SQL 可能な評価信号を生成する。
  3. [結果] 数千件の本番セッションで人手ラベルに対して高い信号精度を示し、品質を保ちながら大幅なコスト削減を実現した。
- 一言メモ: 最重要課題の構造化出力品質を監視しつつ、モデル切替を schema 駆動で自動化する設計に直結する。

### [T2] ChartNet: A Million-Scale, High-Quality Multimodal Dataset for Robust Chart Understanding
- URL: https://arxiv.org/abs/2603.27064
- 重要度: 🔵Medium
- 要約:
  1. [課題] チャート理解は図形・数表・自然言語の横断推論が必要だが、既存 VLM はこの統合が弱い。
  2. [手法] plotting code、画像、表、要約、QA を揃えた 150 万件規模の合成中心データセットを構築し、品質フィルタで整合性を担保した。
  3. [結果] ChartNet で fine-tuning したモデルは複数ベンチマークで一貫して性能向上した。
- 一言メモ: 財務チャートや可視化レポートの読解強化に近く、社内評価セット設計の雛形として有用。

### [T3] MuSEAgent: A Multimodal Reasoning Agent with Stateful Experiences
- URL: https://arxiv.org/abs/2603.27813
- 重要度: 🔵Medium
- 要約:
  1. [課題] 研究エージェントの trajectory-level retrieval は粗く、意思決定に効く局所経験をうまく再利用できない。

### [T3] Marco DeepResearch: Unlocking Efficient Deep Research Agents via Verification-Centric Design
- URL: https://arxiv.org/abs/2603.28376
- 重要度: ⚡High
- 要約:
  1. [課題] deep research agent は QA 合成、軌跡生成、推論時探索の各段階で検証不足が誤り伝播を引き起こす。
  2. [手法] QA データ合成、trajectory synthesis、test-time scaling の3層すべてに verification 機構を入れ、推論時は agent 自身を verifier として使う。
  3. [結果] BrowseComp 系で 8B クラスを大きく上回り、600 tool calls 制約下で一部 30B 級に匹敵または超過した。
- 一言メモ: 金融リサーチ agent の hallucination 抑制は verification-first に寄せるべきで、そのまま運用原則にできる。

### [T3] Trace2Skill: Distill Trajectory-Local Lessons into Transferable Agent Skills
- URL: https://arxiv.org/abs/2603.25158
- 重要度: ⚡High
- 要約:
  1. [課題] agent skill の手作業作成は拡張性が低く、自動生成は局所軌跡への過学習で壊れやすい。
  2. [手法] 複数サブエージェントで diverse な実行軌跡を並列分析し、局所 lesson を統合して conflict-free な skill directory に昇華する。
  3. [結果] spreadsheet・VisionQA・数理推論で強いベースラインを上回り、35B で進化させた skill が 122B にも転移した。
- 一言メモ: 社内タスクから skill を抽出して再利用する仕組みは、金融ドメイン運用のレバレッジが大きい。

### [T3] Learning to Commit: Generating Organic Pull Requests via Online Repository Memory
- URL: https://arxiv.org/abs/2603.26664
- 重要度: 🔵Medium
- 要約:
  1. [課題] coding agent は機能的に正しくても、過去の変更パターンを知らないため repository 固有の流儀に沿わない PR を出しがちである。
  2. [手法] 過去 commit を時系列で再演し、予測差分と正解 diff の差から style・内部 API・設計制約を online repository memory として蓄積する。
  3. [結果] 将来時点の実 PR 評価で、機能性だけでなく style consistency や internal API reuse を改善した。
- 一言メモ: エージェントに社内コードベース固有の変更様式を学ばせる実務設計として示唆が強い。
