### [T1] Federation of Experts: Communication Efficient Distributed Inference for Large Language Models
- URL: https://arxiv.org/abs/2605.06206
- 重要度: ⚡High
- 要約:
  1. [課題] 分散MoE推論では expert 間の all-to-all 通信がレイテンシとスループットの大きなボトルネックになる。
  2. [手法] KV head ごとに MoE cluster を分割する Federation of Experts を導入し、通信を単一GPU内またはノード内に閉じ込める。
  3. [結果] 同規模MoEと同等品質を保ちつつ、end-to-end latency 最大5.2x改善、TTFT 3.62x改善、TBT 1.95x改善を達成した。
- 一言メモ: PCIe 接続前提の Expert Parallelism に直結するので、現行ベースモデル運用の最重要候補。

### [T1] Orthrus: Memory-Efficient Parallel Token Generation via Dual-View Diffusion
- URL: https://arxiv.org/abs/2605.12825
- 重要度: 🔵Medium
- 要約:
  1. [課題] 自己回帰デコードは高品質だが逐次生成が遅く、拡散型並列生成は高速でも品質劣化や学習コストが重い。
  2. [手法] 凍結LLMに軽量モジュールを足し、同じKV cacheを共有する自己回帰 view と diffusion view を exact consensus で統合する。
  3. [結果] 推論品質を落とさず最大7.8x高速化し、追加メモリは O(1) に抑えた。
- 一言メモ: 金融用途の低レイテンシ化で、量子化MoEの上に重ねられる生成高速化レイヤとして面白い。

### [T3] FlowCompile: An Optimizing Compiler for Structured LLM Workflows
- URL: https://arxiv.org/abs/2605.13647
- 重要度: ⚡High
- 要約:
  1. [課題] マルチエージェント型の構造化LLMワークフローは、サブエージェント設定と推論予算の組合せが大きく最適化が難しい。
  2. [手法] ルーティングではなくコンパイラとして扱い、サブエージェントをプロファイルして compile-time に精度-遅延の Pareto 集合を生成する。
  3. [結果] ヒューリスティック構成や routing baseline を上回り、最大6.4xの高速化を示した。
- 一言メモ: 金融AIエージェントの固定グラフ型オーケストレーションを事前最適化する設計にそのまま接続できる。

### [T3] An Empirical Study of Automating Agent Evaluation
- URL: https://arxiv.org/abs/2605.11378
- 重要度: ⚡High
- 要約:
  1. [課題] エージェント評価はツール利用や中間推論を含むため、人手設計も自動化も失敗しやすい。
  2. [手法] 評価知識を skills・テンプレート・動的ドキュメント取得として埋め込む EvalAgent と、AgentEvalBench / Eval@1 を提案した。
  3. [結果] 単純な coding assistant の Eval@1 17.5% に対し、EvalAgent は 65% まで改善し、人間評価でも 79.5% 選好を得た。
- 一言メモ: 構造化出力品質やツール成功率を継続監視する評価パイプラインの作り方として実務価値が高い。

### [T3] EVA-Bench: A New End-to-end Framework for Evaluating Voice Agents
- URL: https://arxiv.org/abs/2605.13841
- 重要度: 🔵Medium
- 要約:
  1. [課題] 音声エージェント評価では、現実的な会話シミュレーションと音声固有の失敗計測を同時に扱うベンチマークが不足している。
  2. [手法] bot-to-bot 会話生成と再生成付きバリデーション、さらに EVA-A と EVA-X の複合指標を備えた end-to-end 評価基盤を構築した。
  3. [結果] 12システム評価で高い peak 性能と安定性能の乖離、アクセント・雑音への頑健性不足を定量化した。
- 一言メモ: 音声特化ではあるが、pass@1 と pass@k の分離評価は金融エージェントの信頼性評価にも流用しやすい。

### [T3] MemReread: Enhancing Agentic Long-Context Reasoning via Memory-Guided Rereading
- URL: https://arxiv.org/abs/2605.10268
- 重要度: 🔵Medium
- 要約:
  1. [課題] 長文コンテキスト処理でメモリ更新だけに依存すると、途中で捨てた証拠を後から回収しにくい。

- 一言メモ: 決算資料やXBRLの長文RAGで、再読ベースの証拠回収戦略として実装価値がある。

### [T3] SafeHarbor: Hierarchical Memory-Augmented Guardrail for LLM Agent Safety
- URL: https://arxiv.org/abs/2605.05704
- 重要度: ⚡High
- 要約:
  1. [課題] ツール利用エージェントの安全化は、厳しすぎると benign task まで拒否する over-refusal を招く。
  2. [手法] 文脈依存ルールを adversarial generation から抽出し、階層メモリに保持して動的注入する training-free guardrail を提案した。
  3. [結果] 有害要求への高拒否率を保ちながら benign utility も高く、GPT-4o 上で 63.6% utility と 93%以上の refusal rate を示した。
- 一言メモ: 金融領域のコンプライアンス制約を hard rule ではなく動的ルール注入で扱う方向に近い。

- 一言メモ: 社内文書や決算資料の multi-hop RAG を「コード実行可能な検索計画」に変える発想がかなり相性が良い。

### [T4] F-GRPO: Factorized Group-Relative Policy Optimization for Unified Candidate Generation and Ranking
- URL: https://arxiv.org/abs/2605.12995
- 重要度: ⚡High
- 要約:
  1. [課題] 検索候補生成とランキングを分離すると downstream utility とずれやすく、統合すると credit assignment が不安定になる。
  2. [手法] 共有LLMの上で生成フェーズとランキングフェーズを因子化し、それぞれに別の group-relative advantage を与えて end-to-end 最適化する。
  3. [結果] 推薦と multi-hop QA で GRPO や decoupled baseline を上回り、推論時の追加アーキテクチャ変更も不要だった。
- 一言メモ: RAG候補選定と再ランキングを一体化した学習は、構造化出力前段の evidence selection 改善に直結する。

### [T4] The Extrapolation Cliff in On-Policy Distillation of Near-Deterministic Structured Outputs
- URL: https://arxiv.org/abs/2605.08737
- 重要度: ⚡High
- 要約:
  1. [課題] 構造化出力タスクで on-policy distillation を強く回すと、ある閾値を超えた瞬間にフォーマット崩壊が起きうる。
  2. [手法] 教師確率・warm-start mass・clip 強度から clip-safe な臨界値 λ* を解析し、JSON などの structured output に拡張した。
  3. [結果] 事前登録した3テストで cliff の予測窓を再現し、閾値直下では小型学生が大幅に高性能化した。
- 一言メモ: 最重要改善ターゲットである構造化出力品質に直結しており、蒸留やRL後処理の安全域設計に効く。
