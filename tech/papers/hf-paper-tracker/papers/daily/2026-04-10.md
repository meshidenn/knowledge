### [T3] Combee: Scaling Prompt Learning for Self-Improving Language Model Agents
- URL: https://arxiv.org/abs/2604.04247
- 重要度: ⚡High
- 要約:
  1. [課題] 自己改善型エージェントのプロンプト学習は単一エージェント前提が多く、多数の実行トレースを並列活用すると品質が落ちやすい。
  2. [手法] 並列スキャン、augmented shuffle、動的バッチ制御を組み合わせて高並列でも崩れにくいプロンプト学習フレームワーク Combee を提案した。
  3. [結果] AppWorld などで従来法比最大17倍の高速化を達成しつつ、同等以上の精度と同コストを維持した。
- 一言メモ: マルチエージェント実行ログからプロンプト改善を回す運用に直結し、金融タスク用エージェントの継続改善基盤として試す価値が高い。

### [T3/T4] SEVerA: Verified Synthesis of Self-Evolving Agents
- URL: https://arxiv.org/abs/2603.25111
- 重要度: ⚡High
- 要約:
  1. [課題] 自己進化型エージェントは性能向上が見込める一方で、安全性や仕様準拠を保証できず本番適用が難しい。
  2. [手法] 生成モデル呼び出しごとに一階述語論理の契約を課す FGGM と、探索・検証・学習の3段階からなる SEVerA を導入した。
  3. [結果] プログラム検証やポリシー準拠ツール利用で制約違反ゼロを保ちながら、非制約型や既存法を上回る性能を示した。
- 一言メモ: 構造化出力やコンプライアンス制約を明示的に守らせたい金融エージェントにそのまま接続できる発想である。

### [T3/T4] RAGEN-2: Reasoning Collapse in Agentic RL
- URL: https://arxiv.org/abs/2604.06268
- 重要度: ⚡High
- 要約:
  1. [課題] エージェント RL ではエントロピーが安定していても入力非依存な思考テンプレートに崩壊する失敗が見逃される。
  2. [手法] 推論品質を入力内多様性と入力間識別性に分解し、相互情報量 proxy と報酬分散にもとづく SNR-Aware Filtering を提案した。
  3. [結果] 計画、Web ナビゲーション、コード実行などで相互情報量が性能と強く相関し、提案フィルタで推論品質と最終性能を改善した。
- 一言メモ: 金融エージェントの RL 調整で「もっともらしいが入力非依存」な失敗を監視する指標として極めて有用である。

### [T3] How Well Do Agentic Skills Work in the Wild: Benchmarking LLM Skill Usage in Realistic Settings
- URL: https://arxiv.org/abs/2604.04323
- 重要度: ⚡High
- 要約:
  1. [課題] スキル機構は理想化条件では効いても、実運用のように大量候補から検索・選択する設定では有効性が不明だった。
  2. [手法] 3.4万件の実スキル集合を使い、検索・選択・適合不足を含む現実的条件でスキル利用を系統評価した。
  3. [結果] 条件が現実的になるほど効果は急減した一方で、query-specific refinement により Terminal-Bench 2.0 で 57.7% から 65.5% へ改善した。
- 一言メモ: MCP や社内スキル群を増やすだけでは不十分で、検索後のスキル精製レイヤが必要だと示している。

### [T3] Squeez: Task-Conditioned Tool-Output Pruning for Coding Agents
- URL: https://arxiv.org/abs/2604.04979
- 重要度: ⚡High
- 要約:
  1. [課題] コーディングエージェントは長大なツール出力を毎回読み込み、コンテキスト浪費とレイテンシ増大を招いている。
  2. [手法] 次の一手に必要な最小証拠ブロックだけを残す task-conditioned tool-output pruning を定式化し、LoRA で小型モデルを学習した。
  3. [結果] 11,477例のベンチマークで 92% のトークン削減を達成しつつ、0.86 recall と 0.80 F1 を実現した。
- 一言メモ: 社内文書検索やツール連携の観測圧縮に転用すれば、長い証券資料やログを扱うエージェントの応答速度を改善できる。

### [T3] GBQA: A Game Benchmark for Evaluating LLMs as Quality Assurance Engineers
- URL: https://arxiv.org/abs/2604.02648
- 重要度: 🔵Medium
- 要約:
  1. [課題] LLM の自律的なバグ発見能力はコード生成より難しいのに、長期探索を伴う妥当な評価基盤が不足していた。
  2. [手法] 30本のゲームと124件の人手検証済みバグからなる GBQA と、ReAct ループとメモリを持つベースラインエージェントを整備した。
  3. [結果] 最良モデルでも検出率は 48.39% に留まり、自律 QA はなお未成熟であることを示した。
- 一言メモ: 金融エージェントの評価でも、正答率だけでなく長期探索下での失敗発見能力を別軸で測る必要がある。

### [T1] MARS: Enabling Autoregressive Models Multi-Token Generation
- URL: https://arxiv.org/abs/2604.07023
- 重要度: ⚡High
- 要約:
  1. [課題] 自己回帰 LLM は次トークンを1個ずつ生成するため、予測容易な区間でも推論スループットが頭打ちになる。
  2. [手法] 既存の instruction データによる継続学習だけで複数トークン同時生成を可能にする MARS と、block-level KV cache を導入した。
  3. [結果] ベース精度をほぼ維持したまま 1.5〜1.7倍のスループット向上と、Qwen2.5-7B で最大 1.71倍の実時間高速化を達成した。
- 一言メモ: PCIe 前提の推論環境ではトークン当たりの往復回数削減が効くので、低遅延化の観点で非常に相性がよい。

### [T1] REAM: Merging Improves Pruning of Experts in LLMs
- URL: https://arxiv.org/abs/2604.04356
- 重要度: ⚡High
- 要約:
  1. [課題] 大規模 MoE はメモリ負荷が高く、単純な expert pruning では性能低下が大きい。
  2. [手法] experts を削除する代わりに router 重みを参考にグルーピングして統合する REAM を提案した。
  3. [結果] 複数の MoE LLM と多様なベンチマークで既存 pruning 法をしばしば上回り、元モデルに近い性能を保った。
- 一言メモ: Qwen3.5-397B-A17B 級の圧縮では、量子化に加えて expert 統合を組み合わせる設計余地がある。

### [T1] Expert-Choice Routing Enables Adaptive Computation in Diffusion Language Models
- URL: https://arxiv.org/abs/2604.01622
- 重要度: 🔵Medium
- 要約:
  1. [課題] 拡散型言語モデルの MoE に自己回帰系由来の token-choice routing を使うと、負荷不均衡と計算配分の硬直性が起きる。
  2. [手法] expert-choice routing に置き換え、さらに denoising step ごとに expert capacity を変える適応計算を導入した。
  3. [結果] 高スループットと高速収束を実現し、既存の pretrained モデルも router の差し替えだけで精度改善と収束改善を得た。
- 一言メモ: 今後 dLLM 系を触るなら、router だけで計算配分を最適化できるという点は実装負荷に対して効果が大きい。

### [T2] DeonticBench: A Benchmark for Reasoning over Rules
- URL: https://arxiv.org/abs/2604.04443
- 重要度: ⚡High
- 要約:
  1. [課題] 税務や法規のような文脈依存ルール推論は高リスク領域なのに、長文かつ実務寄りの評価基盤が不足している。
  2. [手法] 税法、移民、住宅法、航空手荷物規則を対象に 6,232 問のベンチマークを構築し、自然言語推論と Prolog 変換の両方式を許容した。
  3. [結果] 最良モデルでも hard subset の成績は 44.4% や 46.6 macro-F1 に留まり、現在の RL でも安定解決できなかった。
- 一言メモ: 金融規制や社内規程の遵守判定に近い問題設定であり、将来の社内評価セット設計の雛形として重要である。

### [T2/T3] CUE-R: Beyond the Final Answer in Retrieval-Augmented Generation
- URL: https://arxiv.org/abs/2604.05467
- 重要度: ⚡High
- 要約:
  1. [課題] 従来の RAG 評価は最終回答中心で、各証拠文書が推論過程にどれだけ寄与したかを十分に測れていない。
  2. [手法] 証拠ごとに REMOVE、REPLACE、DUPLICATE の介入を加え、正答性、接地性、信頼度誤差、trace 変化を測る CUE-R を提案した。
  3. [結果] 証拠削除や置換は正答性と grounding を大きく損ない、multi-hop では複数証拠の相互作用が非加法的であることを示した。
- 一言メモ: 社内 RAG の評価を「答えが合うか」から「どの文書が本当に効いているか」へ進めるのに使える。

### [T4] Graph-Based Chain-of-Thought Pruning for Reducing Redundant Reflections in Reasoning LLMs
- URL: https://arxiv.org/abs/2604.05643
- 重要度: 🔵Medium
- 要約:
  1. [課題] CoT を RL で強化すると、広く浅い反省や重複再確認が増え、過剰思考で推論効率が悪化する。
  2. [手法] 線形 CoT を DAG 化し、枝単位と深さ単位の pruning を施したうえで SFT、DPO、GRPO を段階適用した。
  3. [結果] 平均推論トークンを 42% 削減しつつ、精度を維持または改善した。
- 一言メモ: 構造化出力の前に冗長な自己反省が膨らむ症状を抑える訓練レシピとして参考になる。

### [T4] DARE: Diffusion Large Language Models Alignment and Reinforcement Executor
- URL: https://arxiv.org/abs/2604.04215
- 重要度: 🔵Medium
- 要約:
  1. [課題] diffusion LLM の事後学習系はコードベースが断片化しており、SFT、PEFT、preference optimization、RL の比較と再現が難しい。
  2. [手法] verl と OpenCompass を土台に、masked と block diffusion の双方を統一的に扱う post-training 基盤 DARE を整備した。
  3. [結果] 複数モデル群に対して広いアルゴリズム対応と再現可能な評価系、実用的な高速化を提供した。
- 一言メモ: 当面は自己回帰系中心でも、次世代 dLLM を試す際の訓練基盤候補として押さえておく価値がある。

### [T4] Learning to Hint for Reinforcement Learning
- URL: https://arxiv.org/abs/2604.00698
- 重要度: 🔵Medium
- 要約:
  1. [課題] GRPO は全サンプルが同じ報酬になる advantage collapse を起こしやすく、難問から学習信号を得にくい。
  2. [手法] reasoner と同時に hinter policy を学習し、ヒント依存度を考慮した transfer-weighted reward で有効な補助情報を生成させた。
  3. [結果] 複数ベンチマークで通常の GRPO や既存のヒント法を一貫して上回った。
- 一言メモ: 金融ドメイン SFT 後の RL 段階で、難問を捨てずに学習可能な形へ変換する補助器として応用余地がある。
