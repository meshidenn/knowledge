### [T1] Stochastic KV Routing: Enabling Adaptive Depth-Wise Cache Sharing
- URL: https://arxiv.org/abs/2604.22782
- 重要度: ⚡High
- 要約:
  1. [課題] KV cache は推論コストとメモリ消費の大きな要因だが、時間方向の圧縮だけでは十分でなく層方向の冗長性も活かし切れていない。
  2. [手法] 学習時に各層が自層または前層の KV を確率的に参照する random cross-layer attention を導入し、深さ方向の cache sharing に適応させた。
  3. [結果] 事前学習・ファインチューニングの双方で多様なモデル族に cache 共有を成立させ、大きな性能劣化なしに KV メモリ削減の余地を示した。
- 一言メモ: PCIe 前提で大規模 MoE を回すなら、量子化だけでなく KV 共有も実運用のレイテンシ改善候補になる。

### [T1] How Much Is One Recurrence Worth? Iso-Depth Scaling Laws for Looped Language Models
- URL: https://arxiv.org/abs/2604.21106
- 重要度: 🔵Medium
- 要約:
  1. [課題] ループ型 LM の recurrence が、追加パラメータと比べてどれだけ実質的な容量向上を生むか定量化されていなかった。
  2. [手法] recurrence 数を変えた 116 本の事前学習 run から joint scaling law を当て、追加 recurrence の価値を表す exponent φ を推定した。
  3. [結果] φ=0.46 を得て、recurrence は一定の効果を持つが固有層の追加よりは弱く、hyperconnections で φ を 0.65 まで引き上げられると示した。
- 一言メモ: 効率化アーキテクチャを検討する際に、見かけの軽量化ではなく真の容量増分を見積もる基準として使える。

### [T2] Improving Robustness of Tabular Retrieval via Representational Stability
- URL: https://arxiv.org/abs/2604.24040
- 重要度: ⚡High
- 要約:
  1. [課題] 同じ表でも csv、html、markdown など serialization が違うだけで埋め込みと検索結果が大きくぶれ、table retrieval の安定性が低い。
  2. [手法] 各 serialization の embedding を noisy view とみなし、その centroid を正準表現として使うほか、frozen encoder 上に軽量 adapter を載せて centroid へ寄せた。
  3. [結果] MPNet、BGE-M3、ReasonIR、SPLADE などで centroid 表現が個別 serialization を総じて上回り、dense retriever の頑健性改善を確認した。
- 一言メモ: XBRL や決算表の取得品質は表現形式差分で簡単に崩れるため、金融 RAG にそのまま効く論点である。

### [T2] Contexts are Never Long Enough: Structured Reasoning for Scalable Question Answering over Long Document Sets
- URL: https://arxiv.org/abs/2604.22294
- 重要度: ⚡High
- 要約:
  1. [課題] 長大な文書集合 QA は chunking 後も evidence 集約がボトルネックで、context window 拡大だけでは限界がある。
  2. [手法] 重要情報を relational database に抽出し、provenance と rationale を使う reconciliation を挟んで SQL 上で推論する SLIDERS を提案した。
  3. [結果] 既存 3 benchmark で全 baseline を上回り、3.9M/36M token の新 benchmark でも次善法に対して大幅な改善を示した。
- 一言メモ: 社内文書・XBRL・決算資料を跨ぐ QA は、長文脈投入より構造化 state 化の方が筋が良い。

### [T2/T4] TexOCR: Advancing Document OCR Models for Compilable Page-to-LaTeX Reconstruction
- URL: https://arxiv.org/abs/2604.22880
- 重要度: ⚡High
- 要約:
  1. [課題] 既存 OCR は plain text や Markdown 止まりで、数式・参照・表構造を含む executable document としての忠実再構成が弱い。
  2. [手法] compilable LaTeX 再構成用の benchmark と学習 corpus を整備し、2B モデルを SFT と verifiable reward による RL で学習した。

- URL: https://arxiv.org/abs/2604.22565
- 重要度: 🔵Medium
- 要約:
  1. [課題] 長くノイジーな文脈では、LLM が決定的証拠を見落として推論を外しやすい。

- 一言メモ: 決算資料や規制文書の重要記述を前段で強調するだけでも、構造化出力の精度改善が狙える。

- URL: https://arxiv.org/abs/2604.21480
- 重要度: 🔵Medium
- 要約:

### [T3/T4] Rewarding the Scientific Process: Process-Level Reward Modeling for Agentic Data Analysis
- URL: https://arxiv.org/abs/2604.24198
- 重要度: ⚡High
- 要約:
  1. [課題] 汎用 PRM はデータ分析 agent の silent error や必要な試行錯誤を適切に評価できず、プロセス監督として弱い。
  2. [手法] 環境へ能動的に作用して中間状態を検査できる DataPRM と、correctable error を区別する ternary reward を導入した。

- 一言メモ: 表計算や財務分析を行う agent の学習では、最終正答だけでなく途中の分析手順に報酬を与えるべきだと示している。

### [T3] Memanto: Typed Semantic Memory with Information-Theoretic Retrieval for Long-Horizon Agents
- URL: https://arxiv.org/abs/2604.22085
- 重要度: ⚡High
- 要約:
  1. [課題] 長期エージェントの memory は graph 構築や多段 retrieval の運用コストが高く、低遅延で回しにくい。
  2. [手法] 13 種の typed semantic memory、conflict resolution、temporal versioning を、no-indexing の情報理論検索で支える memory layer を提案した。

- 一言メモ: 顧客履歴や案件進行を扱う金融エージェントでは、複雑な KG より低遅延 typed memory の方が先に効く可能性が高い。

### [T4] Why Fine-Tuning Encourages Hallucinations and How to Fix It
- URL: https://arxiv.org/abs/2604.15574
- 重要度: ⚡High
- 要約:
  1. [課題] SFT で新事実を学ばせると、事前学習済み知識への干渉が起きて hallucination がむしろ増えることがある。
  2. [手法] continual learning の観点から、出力分布の drift を抑える self-distillation 型 SFT と、不要時の parameter freezing を検討した。
  3. [結果] hallucination の主因が overlapping semantic representation 間の interference だと示しつつ、既存知識の破壊を抑えながら factual learning を改善した。
- 一言メモ: 金融専門用語を追加学習する際、精度向上と既存常識維持を両立する設計指針として重要である。

### [T4] For-Value: Efficient Forward-Only Data Valuation for finetuning LLMs and VLMs
- URL: https://arxiv.org/abs/2508.10180
- 重要度: ⚡High
- 要約:
  1. [課題] データ価値推定は gradient 計算に依存しがちで、B 級モデルのデータ選別には重すぎる。
  2. [手法] 最終 hidden representation と prediction error の整合から価値を閉形式で求める、single forward pass の data valuation を提案した。
  3. [結果] influential data と mislabeled data の検出で gradient 系 baseline と同等以上を保ちながら、大幅な効率改善を示した。
- 一言メモ: SFT データ品質が課題の現状では、金融コーパスの選別とラベル監査を前処理で軽く回せる点が大きい。
