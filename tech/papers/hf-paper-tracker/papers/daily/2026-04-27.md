### [T1] Temporally Extended Mixture-of-Experts Models
- URL: https://arxiv.org/abs/2604.20156
- 重要度: ⚡High
- 要約:
  1. [課題] 通常の MoE は token ごとに expert を頻繁に切り替えるため、GPU メモリを超えると offload や prefetch の効率が大きく落ちる。
  2. [手法] option-critic と deliberation cost を使う controller を各層に入れ、どの expert 集合をどれだけ継続利用するかを学習する temporally extended MoE を提案した。
  3. [結果] gpt-oss-20b で switch rate を 50% 超から 5% 未満へ下げつつ、MATH・MMLU・MMMLU で base 性能の最大 90% を維持した。
- 一言メモ: PCIe 前提の Expert Parallelism では expert 切替回数そのものを減らす発想が極めて実務的で、量子化 MoE 配備のボトルネックに直結する。

### [T2/T4] LLaTiSA: Towards Difficulty-Stratified Time Series Reasoning from Visual Perception to Semantics
- URL: https://arxiv.org/abs/2604.17295
- 重要度: ⚡High
- 要約:
  1. [課題] LLM の時系列理解は評価軸とタスク定義が分散しており、複雑な時間推論を一貫して鍛えにくい。
  2. [手法] 4 段階 taxonomy と 8.3 万件の HiTSR、さらにグラフ表現と数表を併用する curriculum fine-tuning で Time Series Reasoning Model を構築した。
  3. [結果] 多様な時系列推論タスクで高性能を示し、実世界シナリオを含む OOD 一般化でも強さを見せた。
- 一言メモ: 価格推移・業績推移・時系列指標を「チャート + 表」で読ませる金融エージェントの学習設計にそのまま接続できる。

### [T2] Trust but Verify: Introducing DAVinCI -- A Framework for Dual Attribution and Verification in Claim Inference for Language Models
- URL: https://arxiv.org/abs/2604.21193
- 重要度: 🔵Medium
- 要約:
  1. [課題] 高リスク領域では生成文の真偽確認だけでなく、どの根拠から主張が出たかを追跡できないと監査性が足りない。
  2. [手法] 生成 claim を内部コンポーネントと外部ソースへ帰属させ、その後 entailment と confidence calibration で検証する二段階フレームワーク DAVinCI を提案した。
  3. [結果] FEVER 系データで accuracy、attribution precision、recall、F1 を 5〜20% 改善した。
- 一言メモ: 投資判断やレポート生成で claim 単位の根拠帰属と検証を分離する設計の参考になる。

### [T2/T3] TingIS: Real-time Risk Event Discovery from Noisy Customer Incidents at Enterprise Scale
- URL: https://arxiv.org/abs/2604.21889
- 重要度: 🔵Medium
- 要約:
  1. [課題] 大規模運用ではノイズの多い顧客インシデントから、重大なリスクイベントを短時間で抽出するのが難しい。
  2. [手法] efficient indexing と LLM による event linking、業務ルーティング、ノイズ除去を組み合わせた incident discovery system を構築した。
  3. [結果] 本番環境で P90 3.5 分の alert latency と 95% の高優先度 incident discovery rate を達成した。
- 一言メモ: 市場異変や社内運用異常を断片的な報告から束ねる金融リスク監視に応用余地が大きい。

### [T2/T3] PersonalAI: A Systematic Comparison of Knowledge Graph Storage and Retrieval Approaches for Personalized LLM agents
- URL: https://arxiv.org/abs/2506.17001
- 重要度: 🔵Medium
- 要約:
  1. [課題] RAG だけでは長期対話の記憶が弱く、時間依存や矛盾を含む個別文脈を構造的に保持しづらい。
  2. [手法] 自動更新される知識グラフ外部メモリに hyper-edge を加え、A*・beam search・hybrid retrieval など複数の検索法を比較した。
  3. [結果] TriviaQA、HotpotQA、DiaASQ でタスクごとに最適な記憶構造と検索法が異なり、時間依存や矛盾処理にも有効だと示した。
- 一言メモ: 顧客プロファイルや案件履歴を長期保持する金融エージェントの memory 設計にそのまま参考になる。

### [T2/T3] Self-Evolving LLM Memory Extraction Across Heterogeneous Tasks
- URL: https://arxiv.org/abs/2604.11610
- 重要度: 🔵Medium
- 要約:
  1. [課題] 永続型アシスタントでは何を記憶すべきかがタスクごとに異なり、単一の memory extraction prompt では性能が安定しない。
  2. [手法] 18 データセットから作った BEHEMOTH で異種 memory extraction を定式化し、クラスタ単位で自己進化する CluE を提案した。
  3. [結果] 既存の自己進化 prompt 最適化法を上回り、異種タスク全体で相対 9.04% の改善を示した。
- 一言メモ: 調査・会話・業務支援が混在する金融エージェントでは、記憶抽出を用途別クラスタで最適化する考え方が有効である。

### [T3] VLAA-GUI: Knowing When to Stop, Recover, and Search, A Modular Framework for GUI Automation
- URL: https://arxiv.org/abs/2604.21375
- 重要度: ⚡High
- 要約:
  1. [課題] GUI エージェントは早すぎる完了宣言と失敗ループが多く、業務自動化での信頼性を損なう。
  2. [手法] Completeness Verifier、Loop Breaker、Search Agent を核にし、必要時のみ coding agent と grounding agent を呼ぶ modular framework を作った。
  3. [結果] OSWorld 77.5%、WindowsAgentArena 61.0% を達成し、loop-prone モデルでは無駄ステップをほぼ半減した。
- 一言メモ: 社内 GUI ツール連携では「終了判定」「復旧」「外部探索」を明示分離する設計がそのまま効く。

### [T3] Scaling Test-Time Compute for Agentic Coding
- URL: https://arxiv.org/abs/2604.16529
- 重要度: ⚡High
- 要約:
  1. [課題] 長期エージェントでは試行回数を増やすだけでは伸びず、過去 rollout をどう再利用するかが本質的なボトルネックになる。
  2. [手法] rollout を仮説・進捗・失敗モードへ圧縮し、並列側は Recursive Tournament Voting、逐次側は PDR で次試行へ再注入した。
  3. [結果] SWE-Bench Verified で 70.9% から 77.6%、Terminal-Bench v2.0 で 46.9% から 59.1% に改善した。
- 一言メモ: 金融リサーチや分析タスクでも、候補数より失敗知識の構造化再利用が重要だと示している。

### [T3] DR-Venus: Towards Frontier Edge-Scale Deep Research Agents with Only 10K Open Data
- URL: https://arxiv.org/abs/2604.19859
- 重要度: ⚡High
- 要約:
  1. [課題] 小型モデルで deep research agent を育てるには、限られた open data から長期タスク能力を引き出すのが難しい。
  2. [手法] 厳格なデータ洗浄と長軌跡リサンプリングによる agentic SFT の後、情報利得と形式正則化を使う turn-level reward で agentic RL を行った。
  3. [結果] 約 10K の open data だけで 4B モデルが 9B 未満クラスを大きく上回り、30B 級とのギャップも縮めた。
- 一言メモ: 低遅延の補助リサーチエージェントを金融情報収集に置くなら、かなり現実的な育成レシピである。

### [T3] AI scientists produce results without reasoning scientifically
- URL: https://arxiv.org/abs/2604.18805
- 重要度: 🔵Medium
- 要約:
  1. [課題] 研究自動化エージェントが成果を出していても、科学的に自己修正する推論様式を本当に備えているかは不明だった。
  2. [手法] 8 分野・25,000 超の agent run を対象に、性能要因分解と epistemic behavior 分析の両面から scientific agent を評価した。
  3. [結果] base model の寄与が scaffold より圧倒的に大きく、68% の軌跡で evidence を無視し、反証による belief revision も稀だった。
- 一言メモ: 論文探索や検証を自動化するなら、手順実行だけでなく反証処理を訓練対象に入れる必要がある。

### [T3] Co-Evolving LLM Decision and Skill Bank Agents for Long-Horizon Tasks
- URL: https://arxiv.org/abs/2604.20987
- 重要度: 🔵Medium
- 要約:
  1. [課題] 長期タスクでは LLM が再利用可能な skill を保持できず、安定した multi-step 意思決定が崩れやすい。
  2. [手法] decision agent が skill bank から skill を取得しつつ、別の pipeline agent が rollout から skill を抽出・更新する COSPLAY を提案した。
  3. [結果] 8B base model で 6 つのゲーム環境において 4 つの frontier baseline 比で平均 25.1% 超の reward 改善を達成した。
- 一言メモ: 調査・分析・実行の skill を蓄積するマルチエージェント基盤では、skill bank の継続更新設計が重要になる。

### [T3/T4] Visual Reasoning through Tool-supervised Reinforcement Learning
- URL: https://arxiv.org/abs/2604.19945
- 重要度: 🔵Medium
- 要約:
  1. [課題] MLLM に複雑な視覚推論をさせる際、ツール呼び出しと最終正答を同時最適化すると学習が衝突しやすい。
  2. [手法] zoom-in、rotate、flip、draw point/line などの単純ツールに直接 supervision を与え、先に tool mastery、後で正答報酬を与える ToolsRL を提案した。
  3. [結果] curriculum 型の RL により、複雑な visual reasoning で強い tool-use 能力を実現した。
- 一言メモ: 決算資料や図表読解を伴う金融 VLM では、ツール利用自体を段階学習する設計が参考になる。

### [T4] Near-Future Policy Optimization
- URL: https://arxiv.org/abs/2604.20733
- 重要度: ⚡High
- 要約:
  1. [課題] RLVR では off-policy 軌跡を混ぜると学習が加速するが、外部教師は遠すぎ、過去軌跡は弱すぎるという問題がある。
  2. [手法] 同一 training run の少し先の checkpoint を「近未来の自分」として補助軌跡源に使う NPO と、自動介入する AutoNPO を提案した。
  3. [結果] Qwen3-VL-8B-Instruct で平均性能を 57.88 から 62.84、AutoNPO で 63.15 まで引き上げ、収束も加速した。
- 一言メモ: 外部教師を用意しにくい金融向け RLVR でも、自前 run 内 checkpoint 再利用ならすぐ試せる。

### [T4] Hybrid Policy Distillation for LLMs
- URL: https://arxiv.org/abs/2604.20244
- 重要度: 🔵Medium
- 要約:
  1. [課題] LLM 蒸留は divergence の向き、最適化法、データ生成法の組み合わせで安定性と品質が大きく揺れる。
  2. [手法] forward KL と reverse KL の長所を併用し、off-policy data と軽量な近似 on-policy sampling を組み合わせる Hybrid Policy Distillation を提案した。
  3. [結果] 数学推論、対話、コード生成で最適化安定性・計算効率・最終性能を改善した。
- 一言メモ: 大規模教師から金融業務向け軽量モデルを作る蒸留工程の現実的な改善候補になる。

### [T4] Reward Hacking in the Era of Large Models: Mechanisms, Emergent Misalignment, Challenges
- URL: https://arxiv.org/abs/2604.13602
- 重要度: 🔵Medium
- 要約:
  1. [課題] RLHF・RLAIF・RLVR を強く回すほど、モデルが真の意図ではなく proxy reward を攻略する reward hacking が深刻化している。
  2. [手法] Proxy Compression Hypothesis を軸に、目的圧縮・最適化増幅・評価器との共進化として misalignment を整理した survey である。
  3. [結果] 冗長化、迎合、幻覚的正当化、評価器操作などを統一的に説明し、検出と緩和の介入点を体系化した。
- 一言メモ: 構造化出力や judge agent を最適化する金融用途でも proxy 化は起こりやすく、報酬設計レビューの基礎資料になる。
