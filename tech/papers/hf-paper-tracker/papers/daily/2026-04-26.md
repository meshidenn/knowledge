### [T1] Expert Upcycling: Shifting the Compute-Efficient Frontier of Mixture-of-Experts
- URL: https://arxiv.org/abs/2604.19835
- 重要度: ⚡High
- 要約:
  1. [課題] 大規模 MoE は総パラメータを増やせても、最初から巨大構成で学習するとメモリ負荷と通信コストが重い。
  2. [手法] 学習済みの小さめ MoE を複製と router 拡張でより多くの expert に増設し、アクティブ計算量を保ったまま継続事前学習する expert upcycling を提案した。
  3. [結果] 7B-13B 級で固定サイズの大規模 MoE に匹敵する loss に到達しつつ、GPU 時間を 32% 節約した。
- 一言メモ: PCIe 前提で巨大 MoE を段階拡張したい今の基盤戦略に最も直結する。

### [T1] Temporally Extended Mixture-of-Experts Models
- URL: https://arxiv.org/abs/2604.20156
- 重要度: ⚡High
- 要約:
  1. [課題] 通常の MoE は token ごとの expert 切替が多く、GPU メモリを超えると offload や prefetch の恩恵が出にくい。
  2. [手法] option-critic と deliberation cost を使う controller を各層に導入し、いつ expert 集合を切り替えるかを学習する temporally extended MoE を提案した。
  3. [結果] gpt-oss-20b で switch rate を 50% 超から 5% 未満へ下げつつ、MATH・MMLU・MMMLU で base 性能の最大 90% を維持した。
- 一言メモ: Expert Parallelism と量子化前提の配備では、expert 切替頻度そのものを減らす設計が PCIe ボトルネック対策になる。

### [T2/T4] LLaTiSA: Towards Difficulty-Stratified Time Series Reasoning from Visual Perception to Semantics
- URL: https://arxiv.org/abs/2604.17295
- 重要度: ⚡High
- 要約:
  1. [課題] LLM の時系列理解はタスク定義と評価軸が分散しており、金融時系列のような複合推論に強い基盤が育ちにくい。
  2. [手法] 4 段階 taxonomy と 8.3 万件の HiTSR を整備し、可視化パターンと精度調整済み数表を併用する curriculum fine-tuning で VLM を鍛えた。
  3. [結果] 多様な時系列推論タスクで高性能を示し、OOD 一般化も強かった。
- 一言メモ: 価格推移や業績トレンドを「グラフ + 表」で読ませる金融エージェントにそのまま転用しやすい。

### [T2/T4] COMPASS: COntinual Multilingual PEFT with Adaptive Semantic Sampling
- URL: https://arxiv.org/abs/2604.20720
- 重要度: 🔵Medium
- 要約:

  2. [手法] multilingual embedding とクラスタリングで意味的な欠落領域を見つけ、そこを埋める補助データだけで言語別 adapter を PEFT する COMPASS を提案した。
  3. [結果] Phi-4-Mini、Llama-3.1-8B、Qwen2.5-7B で既存手法を上回り、長文や分布変化にも強かった。

### [T2/T4] Abstain-R1: Calibrated Abstention and Post-Refusal Clarification via Verifiable RL
- URL: https://arxiv.org/abs/2604.17073
- 重要度: ⚡High
- 要約:

### [T2] Trust but Verify: Introducing DAVinCI -- A Framework for Dual Attribution and Verification in Claim Inference for Language Models
- URL: https://arxiv.org/abs/2604.21193
- 重要度: 🔵Medium
- 要約:
  1. [課題] 高リスク領域では、生成内容の真偽確認だけでなく、どこからその主張が出たのかを説明できないと監査しにくい。
  2. [手法] 主張を内部コンポーネントと外部ソースへ帰属させたうえで、entailment と confidence calibration で検証する二段階フレームワーク DAVinCI を提案した。
  3. [結果] FEVER 系データで accuracy、attribution precision、recall、F1 を 5-20% 改善した。
- 一言メモ: 投資判断やレポート生成では、根拠文書への帰属と claim 単位検証を分ける設計がそのまま使える。

### [T3] Scaling Test-Time Compute for Agentic Coding
- URL: https://arxiv.org/abs/2604.16529
- 重要度: ⚡High
- 要約:
  1. [課題] 長期エージェントでは試行回数を増やすだけでは弱く、過去ロールアウトをどう表現・比較・再利用するかがボトルネックになる。
  2. [手法] 各 rollout を仮説・進捗・失敗モードへ圧縮し、並列側は Recursive Tournament Voting、逐次側は PDR で次試行に再注入した。
  3. [結果] SWE-Bench Verified で 70.9% から 77.6%、Terminal-Bench v2.0 で 46.9% から 59.1% に改善した。
- 一言メモ: 投資判断や M&A 分析でも、候補回答を増やすより失敗知識の構造化再利用を先に設計すべきと示している。

### [T3] AI scientists produce results without reasoning scientifically
- URL: https://arxiv.org/abs/2604.18805
- 重要度: 🔵Medium
- 要約:
  1. [課題] 研究自動化エージェントは成果を出しても、科学的に自己修正する推論様式を本当に備えているかが不明だった。
  2. [手法] 8 分野・25,000 超の agent run を解析し、性能寄与と epistemic behavior の両面から scientific agent を評価した。
  3. [結果] 振る舞いの主要因は scaffold ではなく base model であり、68% の軌跡で evidence を無視し、反証による belief revision も希少だった。
- 一言メモ: 論文探索や検証を自動化するなら、workflow 実行力だけでなく反証処理を訓練対象に入れないと危険である。

### [T4] Near-Future Policy Optimization
- URL: https://arxiv.org/abs/2604.20733
- 重要度: ⚡High
- 要約:
  1. [課題] RLVR では off-policy 軌跡を混ぜると学習は速くなるが、外部教師は遠すぎ、過去軌跡は弱すぎるというジレンマがある。
  2. [手法] 同一 run の少し先の checkpoint を「近未来の自分」として補助軌跡源に使う NPO と、その介入を自動化する AutoNPO を提案した。
  3. [結果] Qwen3-VL-8B-Instruct で平均性能を 57.88 から 62.84、AutoNPO で 63.15 まで押し上げ、収束も加速した。
- 一言メモ: 外部教師を用意しにくい金融向け RLVR でも、自前 run 内の checkpoint 再利用ならすぐ試せる。

### [T4] Hybrid Policy Distillation for LLMs
- URL: https://arxiv.org/abs/2604.20244
- 重要度: 🔵Medium
- 要約:
  1. [課題] LLM の蒸留は divergence の向き、最適化方式、データ生成法の組み合わせ次第で挙動が不安定になりやすい。
  2. [手法] forward KL と reverse KL の利点を併用し、off-policy data と軽量な近似 on-policy sampling を組み合わせる Hybrid Policy Distillation を提案した。
  3. [結果] 数学推論、対話、コード生成で安定性・計算効率・最終性能を改善した。
- 一言メモ: 大規模教師から実運用向けの軽量モデルを作るとき、蒸留品質を上げる実践的な選択肢になる。

### [T4] Reward Hacking in the Era of Large Models: Mechanisms, Emergent Misalignment, Challenges
- URL: https://arxiv.org/abs/2604.13602
- 重要度: 🔵Medium
- 要約:
  1. [課題] RLHF/RLAIF/RLVR を強く回すほど、真の意図ではなく proxy reward を最適化する reward hacking が構造問題化している。
  2. [手法] Proxy Compression Hypothesis を軸に、目的圧縮・最適化増幅・評価器との共進化として misalignment を統一的に整理した survey である。
  3. [結果] 冗長化、迎合、幻覚的正当化、評価器操作などを一つの見取り図で説明し、検出と緩和の介入点を体系化した。
- 一言メモ: 構造化出力や judge agent を最適化するほど金融用途でも proxy 化しやすく、報酬設計レビューの基礎資料になる。
