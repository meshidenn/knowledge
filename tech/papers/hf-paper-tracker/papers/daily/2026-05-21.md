### [T1] OSCAR: Offline Spectral Covariance-Aware Rotation for 2-bit KV Cache Quantization
- URL: https://arxiv.org/abs/2605.17757
- 重要度: ⚡High
- 要約:
  1. [課題] 長文推論で有効なINT2 KVキャッシュ量子化は、既存の単純回転法だと注意構造と噛み合わず精度崩壊しやすい。
  2. [手法] 注意先の共分散をオフライン推定して固定回転とクリッピング閾値を設計し、paged KV-cache互換のINT2 attention kernelまで含めたOSCARを提案。
  3. [結果] Qwen3系や358B級モデルでBF16に近い精度を保ちつつ、KVメモリ約8倍削減、同一メモリ予算で最大7倍のスループット向上を示した。
- 一言メモ: PCIe接続・量子化必須の自分たちの推論条件にそのまま刺さる、今日のT1本命。

### [T1] Draft Less, Retrieve More: Hybrid Tree Construction for Speculative Decoding
- URL: https://arxiv.org/abs/2605.20104
- 重要度: 🔵Medium
- 要約:
  1. [課題] speculative decodingの大きなdraft treeは受理率を上げても、VRAM帯域と計算量が重く実効速度が伸びにくい。
  2. [手法] 枝刈りで空いた計算予算に予測トークン取得を差し込む`prune-then-graft`方式で、受理長の低下をほぼゼロコストで補償するGraftを提案。
  3. [結果] 学習不要のまま短文・長文・大規模モデルで新しい速度精度フロンティアを示し、Qwen3-235BではEAGLE-3比で平均最大21.8%高速化した。
- 一言メモ: 大型MoEのレイテンシ改善策として、量子化と並ぶ実装候補になりうる。

### [T2] Active Learners as Efficient PRP Rerankers
- URL: https://arxiv.org/abs/2605.14236
- 重要度: ⚡High
- 要約:
  1. [課題] LLMのpairwise ranking promptingは比較ノイズや順序依存が強く、通常のソートではcall budget制約下で安定したtop-Kを作りにくい。
  2. [手法] PRP rerankingをノイズ付きペア比較のactive learningとして再定式化し、1回呼び出しで位置バイアスを平均化するrandomized-direction oracleを導入。
  3. [結果] call制約下でNDCG@10あたりの効率を改善し、双方向比較コストなしでより信頼できる上位文書選別を実現した。
- 一言メモ: 社内文書RAGや決算資料検索の再ランキング改善に直結し、少ない推論回数で検索品質を上げやすい。

### [T2] SAGA: A Sequence-Adaptive Generative Architecture for Multi-Horizon Probabilistic Forecasting with Adaptive Temporal Conformal Prediction
- URL: https://arxiv.org/abs/2605.19014
- 重要度: 🔵Medium
- 要約:
  1. [課題] 財務・政策系の長期所得予測では、従来のパラメトリックモデルが長期非線形構造や不確実性を十分に捉えにくい。
  2. [手法] 不規則な表形式パネル系列向けdecoder-only Transformerと、有限標本被覆保証を持つtemporal conformal calibrationを組み合わせたSAGAを提案。
  3. [結果] スウェーデンの大規模縦断データで10年先CRPSを31.9%、20年先MAEを37.7%改善し、予測区間も名目被覆率に近く保った。
- 一言メモ: LLM本体ではないが、金融エージェントの予測モジュールや不確実性提示の設計として参考になる。

### [T2][T4] DocAtlas: Multilingual Document Understanding Across 80+ Languages
- URL: https://arxiv.org/abs/2605.12623
- 重要度: 🔵Medium
- 要約:
  1. [課題] 低資源言語を含む文書理解は学習データ不足と既存モデル由来アノテーションの偏りで性能が頭打ちになりやすい。
  2. [手法] 82言語・9タスクのOCR/文書理解データセットをレンダリング起点で高精度生成し、正例信号として描画由来ground truthを使うDPO適応を行った。
  3. [結果] SFTが外部分布性能を落とす条件でも、DPOは多言語適応を安定化させ、in-domainとout-of-domainの両方を改善した。
- 一言メモ: 多言語決算資料やXBRL周辺文書の前処理品質を上げるうえで、文書理解と選好最適化をつなぐ視点が有用。

### [T3] Harnessing LLM Agents with Skill Programs
- URL: https://arxiv.org/abs/2605.17734
- 重要度: ⚡High
- 要約:
  1. [課題] 既存のagent skillは自然言語メモに留まりがちで、いつ・どう介入するかが曖昧なため再利用性が低かった。
  2. [手法] skillを実行可能なProgram Functionsへ昇格し、失敗しやすい状態で次行動修正や補助文脈注入を行うHASPを提案した。
  3. [結果] web search、数学、codingでtraining-free系とtraining-based系の両方を上回り、web-search reasoningではReAct比25%改善した。
- 一言メモ: 社内分析手順やMCP利用規約を「助言」ではなく実行可能skillとして資産化する設計に近い。

### [T3] EnvFactory: Scaling Tool-Use Agents via Executable Environments Synthesis and Robust RL
- URL: https://arxiv.org/abs/2605.18703
- 重要度: ⚡High
- 要約:
  1. [課題] tool-use agent向けRLは、現実的で頑健な実行環境不足と、人間らしい暗黙意図を持つ訓練データ不足が律速になっていた。
  2. [手法] 実在リソースから状態付き実行環境を自動探索・検証し、topology-aware samplingで自然なmulti-turn軌跡を合成するEnvFactoryを構築。
  3. [結果] 85環境・2,575軌跡という比較的少量データでも、Qwen3系でBFCLv3やMCP-Atlasなどを最大15%改善した。
- 一言メモ: 金融向けMCPツール群でも、先に検証可能な環境生成を整える方がRLデータを増やすより効く可能性が高い。

### [T3] OpenComputer: Verifiable Software Worlds for Computer-Use Agents
- URL: https://arxiv.org/abs/2605.19769
- 重要度: ⚡High
- 要約:
  1. [課題] computer-use agent評価は最終状態判定が曖昧で、細かなアプリ状態や部分達成を監査しにくかった。
  2. [手法] 実アプリ向けstate verifier、自己進化する検証層、機械判定可能なタスク生成、軌跡記録付き評価ハーネスを統合したOpenComputerを提案。
  3. [結果] 33アプリ・1,000タスクを整備し、LLM-as-judgeより人手判定に近い検証を実現しつつ、既存frontier agentsの頑健性不足を露呈させた。
- 一言メモ: 金融業務自動化でも「正答率」より検証可能な部分報酬と状態検査APIが先に必要だと分かる。

### [T3] PEEK: Context Map as an Orientation Cache for Long-Context LLM Agents
- URL: https://arxiv.org/abs/2605.19932
- 重要度: 🔵Medium
- 要約:
  1. [課題] 長文エージェントは同じ文書群やコードベースを何度も読むのに、従来法は文脈そのものへの再利用可能な方位情報を保持できていなかった。
  2. [手法] 文脈の中身・構造・有用エンティティを小さなcontext mapとして保持し、Distiller・Cartographer・Evictorで固定長更新するPEEKを提案。
  3. [結果] 長文推論と情報集約で強力な基線を6.3〜34.0%上回り、反復回数とコストも大きく削減した。
- 一言メモ: 社内文書RAGや継続的な企業調査では、毎回全文再読するより「方位キャッシュ」を持つ方が実運用に近い。

### [T4] Process Rewards with Learned Reliability
- URL: https://arxiv.org/abs/2605.15529
- 重要度: ⚡High
- 要約:
  1. [課題] 既存のprocess reward modelは各ステップに単一スコアしか出さず、その報酬をどこまで信用すべきか分からない。
  2. [手法] 成功確率と信頼度を同時に学習するBetaPRMを導入し、その信頼度に応じて追加計算を振り分けるAdaptive Computation Allocationを組み合わせた。
  3. [結果] 複数の推論ベンチでBest-of-Nの精度とトークン効率を改善し、固定Best-of-16比で最大33.57%のトークン削減を達成した。
- 一言メモ: 構造化出力スキーマの自己検証でも、「正しそうか」だけでなく「どれだけ確からしいか」を使う設計が重要になる。

### [T4] Not Every Rubric Teaches Equally: Policy-Aware Rubric Rewards for RLVR
- URL: https://arxiv.org/abs/2605.20164
- 重要度: ⚡High
- 要約:
  1. [課題] rubric-based rewardは人間が重視する基準と、現時点で学習信号として効く基準を同一視してしまい、RLの効率を落としやすい。
  2. [手法] 最終目的のrubricは保ちつつ、rollout間の識別力に応じて基準ごとの報酬重みを動的調整するPOW3Rを提案。
  3. [結果] 3つの基礎方策・2データセットで30比較中24勝し、strict completionも向上させながら学習収束を2.5〜4倍高速化した。
- 一言メモ: 構造化出力の各項目を同じ強さで教えるのではなく、未達成項目へ重みを寄せる実務的ヒントになる。

### [T4] CEPO: RLVR Self-Distillation using Contrastive Evidence Policy Optimization
- URL: https://arxiv.org/abs/2605.19436
- 重要度: 🔵Medium
- 要約:
  1. [課題] RLVRでは正解軌跡中の決定的な推論トークンと単なる言い回しが同じ報酬を受け、信用割当が粗い。
  2. [手法] 正解教師だけでなくバッチ内の誤答軌跡も対照として使い、「正答が支持し誤答が支持しない」トークンに焦点を当てるCEPOを提案。
  3. [結果] 2B・4B規模のマルチモーダル数理推論でGRPOを安定して上回り、既存の自己蒸留法が起こす情報漏洩問題も回避した。
- 一言メモ: 正しいJSONでも本当に効いた推論ステップはどこかを切り分けたい自分たちのアライメント設計に近い。
