### [T2/T4] Programming with Data: Test-Driven Data Engineering for Self-Improving LLMs from Raw Corpora
- URL: https://arxiv.org/abs/2604.24819
- 重要度: ⚡High
- 要約:
  1. [課題] ドメイン知識を LLM に移す fine-tuning は失敗原因を訓練データ側へ遡って診断しにくく、足りないデータを闇雲に足す運用になりやすい。
  2. [手法] コーパスから抽出した構造化知識表現を学習データと評価の共通基盤にし、training=compilation、benchmark=unit test、data repair=debugging とみなす Programming with Data を定式化した。
  3. [結果] 16 分野で concept-level gap や reasoning-chain break を特定してデータ修復し、モデル規模やアーキテクチャを跨いで一貫した改善を示した。
- 一言メモ: 金融 SFT データ品質が最重要課題の現状では、専門知識注入を「データを増やす作業」から「データをデバッグする作業」へ変える視点が極めて実用的である。

### [T4] Why Fine-Tuning Encourages Hallucinations and How to Fix It
- URL: https://arxiv.org/abs/2604.15574
- 重要度: ⚡High
- 要約:
  1. [課題] SFT で新しい事実を教えると、事前学習済み知識への干渉で hallucination が増えることがある。
  2. [手法] continual learning の観点から、出力分布 drift を抑える self-distillation 型 SFT と、知識追加が不要な場面での parameter freezing を検討した。
  3. [結果] hallucination の主因が重なり合う意味表現間の interference だと示しつつ、既存知識を壊しにくい学習法で factual learning を改善した。
- 一言メモ: 金融用語を追加学習しても既存の一般常識や推論能力を壊さない SFT 設計の基礎としてそのまま効く。

### [T4] For-Value: Efficient Forward-Only Data Valuation for finetuning LLMs and VLMs
- URL: https://arxiv.org/abs/2508.10180
- 重要度: ⚡High
- 要約:
  1. [課題] 学習データの価値推定は gradient ベースだと重く、B 級 LLM/VLM のデータ選別や誤ラベル検出を高頻度で回しにくい。
  2. [手法] 最終 hidden representation と prediction error の整合から、single forward pass だけで data value を閉形式推定する For-Value を提案した。
  3. [結果] influential data と mislabeled data の検出で gradient 系 baseline と同等以上を維持しつつ、大幅な効率改善を達成した。
- 一言メモ: 金融 SFT コーパスの選別と監査を軽く回せるので、LoRA/QLoRA 前のデータ品質管理の自動化候補になる。

### [T2] Improving Robustness of Tabular Retrieval via Representational Stability
- URL: https://arxiv.org/abs/2604.24040
- 重要度: ⚡High
- 要約:
  1. [課題] 同じ表でも csv、html、markdown など serialization が違うだけで embedding と retrieval 結果が大きくぶれる。
  2. [手法] 各 serialization を noisy view とみなし、embedding centroid を正準表現として使うほか、frozen encoder 上に軽量 adapter を載せて centroid へ寄せた。
  3. [結果] MPNet、BGE-M3、ReasonIR、SPLADE などで centroid 表現が個別形式を総じて上回り、dense retriever の頑健性を改善した。
- 一言メモ: XBRL や決算表の取得品質は表現形式差分で簡単に崩れるため、金融 RAG の table retrieval 改善に直結する。

### [T2] RaV-IDP: A Reconstruction-as-Validation Framework for Faithful Intelligent Document Processing
- URL: https://arxiv.org/abs/2604.23644
- 重要度: 🔵Medium
- 要約:
  1. [課題] IDP パイプラインは抽出結果が原文に忠実かを内在的に検証できず、誤抽出がそのまま KB や RAG に流れ込みやすい。
  2. [手法] 抽出後に専用 reconstructor で対象領域を再構成し、原画像 crop と比較して fidelity を測る reconstruction-as-validation を導入した。
  3. [結果] ラベル不要の grounded quality signal に基づき、閾値以下なら structured vision fallback を発火する自己検証ループを実装した。
- 一言メモ: 決算資料や開示文書の ingestion でも、抽出精度そのものより「壊れた抽出を downstream に流さない」設計が重要である。

### [T3] Recursive Multi-Agent Systems
- URL: https://arxiv.org/abs/2604.25917
- 重要度: ⚡High
- 要約:
  1. [課題] 既存マルチエージェントは text ベースのやり取りが重く、協調を深めるほど token コストと遅延が増えやすい。
  2. [手法] 複数 agent 全体を latent-space 上の再帰計算として扱い、RecursiveLink と inner-outer loop learning で system 全体を共同最適化する RecursiveMAS を提案した。
  3. [結果] 9 benchmark 平均で精度を 8.3% 改善しつつ、1.2-2.4 倍の推論高速化と 34.6-75.6% の token 削減を示した。
- 一言メモ: 金融エージェント群でも agent 間通信をそのまま増やすのでなく、協調そのものを低トークン化する設計が次のボトルネックになる。

### [T3] From Skills to Talent: Organising Heterogeneous Agents as a Real-World Company
- URL: https://arxiv.org/abs/2604.22446
- 重要度: ⚡High
- 要約:
  1. [課題] マルチエージェントは固定チーム編成と密結合 orchestration に縛られ、実行中の能力補充や継続改善が弱い。
  2. [手法] skill・tool・runtime を束ねた Talent、typed organizational interface、Talent Market、E^2R tree search を備える OneManCompany を提案した。
  3. [結果] PRDBench で 84.67% の success rate を達成し、既存 SOTA を 15.48 ポイント上回った。
- 一言メモ: 金融業務の agent 群は単発フローより「組織レイヤ」を持つ方が、MCP や外部ツール追加に強い構成になる。

### [T3] ClawMark: A Living-World Benchmark for Multi-Turn, Multi-Day, Multimodal Coworker Agents
- URL: https://arxiv.org/abs/2604.23781
- 重要度: ⚡High
- 要約:
  1. [課題] 実務エージェントは日を跨いで状態変化へ追従する必要があるが、既存 benchmark は単一の静的 episode に偏っている。
  2. [手法] filesystem、email、calendar、knowledge base、spreadsheet を含む stateful sandbox 上で multi-turn・multi-day タスクを評価する benchmark を構築した。
  3. [結果] 最強モデルでも strict Task Success は 20.0% に留まり、外生的な環境更新後に性能が大きく落ちることを示した。
- 一言メモ: 市況変化や新規資料流入が前提の金融ワークフローでは、この種の living-world 評価が実運用性能に近い。

### [T3] AutoResearchBench: Benchmarking AI Agents on Complex Scientific Literature Discovery
- URL: https://arxiv.org/abs/2604.25256
- 重要度: ⚡High
- 要約:
  1. [課題] 研究エージェントの文献探索能力は重要だが、深い概念理解と網羅探索を同時に測る benchmark が不足していた。
  2. [手法] 特定論文を追い当てる Deep Research と、条件に合う論文集合を集める Wide Research からなる literature discovery benchmark を作成した。
  3. [結果] 最強クラスの LLM でも Deep Research 9.39%、Wide Research 9.31% IoU と極めて低く、研究探索が未解決課題であると示した。
- 一言メモ: 自動リサーチ agent を社内知識探索や市場調査へ転用する前に、探索そのものの難しさを正面から測る必要がある。

### [T3/T4] TCOD: Exploring Temporal Curriculum in On-Policy Distillation for Multi-turn Autonomous Agents
- URL: https://arxiv.org/abs/2604.24005
- 重要度: 🔵Medium
- 要約:
  1. [課題] on-policy distillation は multi-turn agent では誤りの累積で trajectory-level KL が暴れ、学習が不安定になりやすい。
  2. [手法] 学生に見せる trajectory depth を短いものから長いものへ段階的に広げる temporal curriculum を導入した TCOD を提案した。
  3. [結果] 3 つの multi-turn benchmark で vanilla OPD を最大 18 ポイント上回り、KL の安定化と teacher 超えの一般化も確認した。
- 一言メモ: 金融向けの小型実行モデルへ teacher の手順を蒸留するなら、長い agent trajectory を一気に教えない方が安全である。

### [T3] Toward Scalable Terminal Task Synthesis via Skill Graphs
- URL: https://arxiv.org/abs/2604.25727
- 重要度: 🔵Medium
- 要約:

  2. [手法] scenario-mediated skill graph から workflow path をサンプルし、multi-agent harness で executable task に具体化する SkillSynth を提案した。

### [T3] Efficient Agent Evaluation via Diversity-Guided User Simulation
- URL: https://arxiv.org/abs/2604.21480
- 重要度: 🔵Medium
- 要約:
  1. [課題] 顧客対応 agent の評価を線形 rollout に頼ると、同じ prefix を何度も生成して計算が無駄になり、稀な失敗も拾いにくい。
  2. [手法] 重要分岐点で会話状態を snapshot し、多様性を誘導した user 応答で枝分かれ探索する DIVERT を提案した。
  3. [結果] 標準的な Monte Carlo rollout より token あたり多くの failure を発見し、より広い interaction path を効率良くカバーした。
- 一言メモ: 金融アシスタントの検証では、成功率の平均より深い failure mode をどれだけ早く掘れるかが価値になる。

### [T3/T4] BARRED: Synthetic Training of Custom Policy Guardrails via Asymmetric Debate
- URL: https://arxiv.org/abs/2604.25203
- 重要度: 🔵Medium
- 要約:
  1. [課題] カスタムポリシー用 guardrail は汎用安全モデルでは足りず、専用分類器を作るにもラベル付きデータが高コストで集めにくい。
  2. [手法] タスク記述と少量の未ラベル例だけから、次元分解と multi-agent debate で高忠実な合成学習データを作る BARRED を提案した。
  3. [結果] この合成データで学習した小型モデルが、専用 guardrail や proprietary reasoning model を一貫して上回った。
- 一言メモ: コンプライアンスや開示規制に合わせた金融専用ガードレールを、人手アノテーションを抑えて作る道筋として有望である。

### [T3] The Last Harness You'll Ever Build
- URL: https://arxiv.org/abs/2604.21003
- 重要度: 🔵Medium
- 要約:
  1. [課題] 新しい業務ドメインごとに prompt、tool、orchestration、evaluation を人手で harness 設計するコストが高すぎる。
  2. [手法] 単一タスク上で harness 自体を改善する Harness Evolution Loop と、その進化プロトコル自体を横断最適化する Meta-Evolution Loop の二層構造を提案した。
  3. [結果] 新規ドメイン適応を人手の harness engineering なしで素早く収束させる枠組みとして、meta-learning 的な対応関係を定式化した。
- 一言メモ: 金融ユースケースごとの agent 調整を毎回手作業で作るのでなく、評価付き self-improving harness に寄せる発想が重要になる。

### [T3/T4] Taming Actor-Observer Asymmetry in Agents via Dialectical Alignment
- URL: https://arxiv.org/abs/2604.19548
- 重要度: 🔵Medium
- 要約:
  1. [課題] self-reflection と相互監査を持つ multi-agent は、actor と observer で失敗原因の帰属がぶれる認知バイアスを抱えうる。
  2. [手法] Ambiguous Failure Benchmark でそのぶれを測定し、thesis-antithesis-synthesis に基づく dialectical alignment と GRPO で ReTAS を学習した。
  3. [結果] 多くのモデルで 20% 超の perspective-induced inconsistency を確認し、ReTAS により attribution inconsistency と fault resolution を改善した。
- 一言メモ: reviewer agent や compliance checker を組み込む金融オーケストレーションでは、役割差で判定基準がずれる問題を無視できない。
