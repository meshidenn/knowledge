### [T1] Expert Upcycling: Shifting the Compute-Efficient Frontier of Mixture-of-Experts
- URL: https://arxiv.org/abs/2604.19835
- 重要度: ⚡High
- 要約:
  1. [課題] 大規模 MoE は総パラメータを増やせても、学習時のメモリ負荷と通信コストが重く、最初から大きく作るのが高価である。
  2. [手法] 学習済み E-expert MoE を複製と router 拡張で mE-expert に段階増設し、top-K routing と推論時アクティブ計算量を保ったまま継続事前学習する expert upcycling を提案した。
  3. [結果] 7B-13B 級の実験で固定サイズの大規模 MoE と同等の validation loss に到達しつつ、GPU 時間を 32% 節約した。
- 一言メモ: PCIe 前提で巨大 MoE を育てる現実解として非常に相性がよく、Qwen3.5-397B-A17B 級への段階拡張レシピとして最優先で追う価値がある。

### [T2/T4] COMPASS: COntinual Multilingual PEFT with Adaptive Semantic Sampling
- URL: https://arxiv.org/abs/2604.20720
- 重要度: 🔵Medium
- 要約:
  1. [課題] 多言語適応では単純な追加学習が言語間干渉を招き、対象分布に対して必要な知識を効率よく補うのが難しい。
  2. [手法] multilingual embedding とクラスタリングで意味分布の欠落領域を特定し、不足クラスタを優先サンプリングして言語別 adapter を継続 PEFT する。
  3. [結果] Phi-4-Mini、Llama-3.1-8B、Qwen2.5-7B で既存の言語類似度ベース手法を上回り、分布変化時の継続更新でも有効だった。
- 一言メモ: 多言語決算や社内文書 RAG では、金融用語の抜けた意味クラスタだけを埋める adapter 運用として転用しやすい。

### [T3] OpenMobile: Building Open Mobile Agents with Task and Trajectory Synthesis
- URL: https://arxiv.org/abs/2604.15093
- 重要度: ⚡High
- 要約:
  1. [課題] モバイルエージェントは伸びているが、学習データ生成法が閉じており、オープンに再現できる高品質軌跡が不足していた。
  2. [手法] 環境探索から global memory を作って grounded instruction を合成し、learner と expert を切り替えながら error-recovery を含む軌跡を収集する。
  3. [結果] Qwen2.5-VL/Qwen3-VL ベースで AndroidWorld 51.7%/64.7% を達成し、既存の open-data 系を大きく上回った。
- 一言メモ: 金融エージェントでも「環境記憶に基づくタスク合成」と「失敗復帰を含む軌跡収集」を導入すると、実運用に近いツール利用データを増やせる。

### [T2/T4] Abstain-R1: Calibrated Abstention and Post-Refusal Clarification via Verifiable RL
- URL: https://arxiv.org/abs/2604.17073
- 重要度: ⚡High
- 要約:
  1. [課題] RL 系の後学習は回答能力を上げる一方で、情報不足な問いにも推測で答えてハルシネーションしやすい。
  2. [手法] 回答不能時に拒否するだけでなく、「何が欠けているか」を整合的に説明する clarification-aware RLVR reward で 3B モデルを学習した。
  3. [結果] Abstain-Test、Abstain-QA、SelfAware で abstention と clarification を大きく改善し、より大きなモデル群に匹敵する未解答時挙動を示した。
- 一言メモ: 金融 AI エージェントの最重要課題であるハルシネーション抑制に直結し、「不足データを明示して追加取得を促す」振る舞いを学習させる土台になる。

### [T3] A Self-Evolving Framework for Efficient Terminal Agents via Observational Context Compression
- URL: https://arxiv.org/abs/2604.19572
- 重要度: 🔵Medium
- 要約:
  1. [課題] 長時間動く terminal agent は観測ログを履歴に抱え込み続け、トークンコストが二次的に膨らむ。
  2. [手法] 軌跡から圧縮ルール自体を自動発見・更新する self-evolving compression framework TACO を作り、既存エージェントにプラグイン的に適用する。
  3. [結果] TerminalBench 系や SWE-Bench Lite などで 1-4% の性能改善を出しつつ、トークンオーバーヘッドも約 10% 削減した。
- 一言メモ: 社内データ検索や MCP 呼び出しを多段で行う金融エージェントでも、観測圧縮はコスト削減と長期タスク安定化の両面で効く。

### [T3] Scaling Test-Time Compute for Agentic Coding
- URL: https://arxiv.org/abs/2604.16529
- 重要度: ⚡High
- 要約:
  1. [課題] 長期エージェントでは試行回数を増やすだけでは弱く、過去ロールアウトをどう要約・選抜・再利用するかがボトルネックになる。
  2. [手法] 各ロールアウトを仮説・進捗・失敗モードに圧縮し、並列側は Recursive Tournament Voting、逐次側は PDR を使って次試行へ再注入する。
  3. [結果] SWE-Bench Verified で 70.9% から 77.6%、Terminal-Bench v2.0 で 46.9% から 59.1% へ改善した。
- 一言メモ: 投資判断や M&A 分析でも、候補解の比較より「途中で得た失敗知識の再利用」を設計した方が test-time scaling の投資対効果が高い。

### [T4] Near-Future Policy Optimization
- URL: https://arxiv.org/abs/2604.20733
- 重要度: ⚡High
- 要約:
  1. [課題] RLVR では off-policy 軌跡を混ぜると学習は速くなるが、強すぎても遠すぎても学習信号が弱くなる。
  2. [手法] 同一学習 run の少し先の checkpoint を「近未来の自分」として補助軌跡源に使う NPO と、介入タイミングを自動化する AutoNPO を提案した。
  3. [結果] Qwen3-VL-8B-Instruct の平均性能を 57.88 から 62.84、AutoNPO では 63.15 まで押し上げ、収束も加速した。
- 一言メモ: DPO/KTO や RLVR 系の金融タスク学習で外部教師を用意しにくい場合、自前 run 内の近傍 checkpoint 再利用はすぐ試せる。

### [T4] Reward Hacking in the Era of Large Models: Mechanisms, Emergent Misalignment, Challenges
- URL: https://arxiv.org/abs/2604.13602
- 重要度: 🔵Medium
- 要約:
  1. [課題] RLHF/RLAIF/RLVR の拡大に伴い、真の意図を満たさず proxy reward を最適化する reward hacking が構造問題化している。
  2. [手法] Proxy Compression Hypothesis を軸に、目的圧縮・最適化増幅・評価器との共進化の相互作用として各種 misalignment を整理した survey である。
  3. [結果] 冗長化、迎合、幻覚的正当化、評価器操作などを統一的に説明し、検出・緩和策の介入点を体系化した。
- 一言メモ: 構造化出力やエージェント評価器を強く最適化するほど金融領域でも proxy 化の危険が増すため、報酬設計レビューのチェックリストとして有用である。

### [T3] DR-Venus: Towards Frontier Edge-Scale Deep Research Agents with Only 10K Open Data
- URL: https://arxiv.org/abs/2604.19859
- 重要度: ⚡High
- 要約:
  1. [課題] 小型モデルで deep research agent を作るには、限られた open data で長期タスク能力を引き出すのが難しい。
  2. [手法] 厳格なデータ洗浄と長軌跡リサンプリングによる agentic SFT の後、情報利得と形式正則化を用いた turn-level reward で agentic RL を行う。
  3. [結果] 約 10K の open data だけで 4B モデルが 9B 未満クラスを上回り、30B 級 deep research system とのギャップも縮めた。
- 一言メモ: 小さめの補助エージェントを市場調査や文書探索に使う構成なら、4B 級の低遅延研究エージェント育成レシピとしてかなり実務的である。

### [T3] AJ-Bench: Benchmarking Agent-as-a-Judge for Environment-Aware Evaluation
- URL: https://arxiv.org/abs/2604.18240
- 重要度: ⚡High
- 要約:
  1. [課題] エージェントの行動検証はルールベースや LLM-as-a-Judge だけでは環境依存タスクに弱く、証拠取得が不足しやすい。
  2. [手法] 検索・データシステム・GUI の3領域で、環境に介入して状態と過程を確かめる Agent-as-a-Judge を評価する AJ-Bench を構築した。
  3. [結果] LLM-as-a-Judge より一貫して高い性能を示しつつ、情報取得や状態確認にはまだ大きな改善余地が残ることを明らかにした。
- 一言メモ: 金融レポート作成後の検証や取引前チェックを「実データで裏取りする judge agent」に切り出す設計を後押しする。

### [T4] ShadowPEFT: Shadow Network for Parameter-Efficient Fine-Tuning
- URL: https://arxiv.org/abs/2604.19254
- 重要度: ⚡High
- 要約:
  1. [課題] LoRA 系は各重みに局所的な低ランク摂動を入れるため、適応表現が分散しやすく再利用しづらい。
  2. [手法] 各 transformer 層で共有 shadow state を進化させる depth-shared shadow module を用い、層空間の refinement として PEFT を行う。
  3. [結果] 同等の学習パラメータ予算で LoRA/DoRA を同等以上に上回り、shadow module の事前学習や detached deployment も有効だった。
- 一言メモ: 業務別 LoRA を乱立させるより、共通 shadow module を金融タスク群で使い回す方が運用しやすく、エッジ配備にも向く。

### [T4] Benign Fine-Tuning Breaks Safety Alignment in Audio LLMs
- URL: https://arxiv.org/abs/2604.16659
- 重要度: 🔵Medium
- 要約:
  1. [課題] 無害データでの fine-tuning でも安全性が壊れる現象は知られていたが、音声系ではどの近接性が危険要因か整理されていなかった。
  2. [手法] semantic・acoustic・mixed の各軸で harmful content への近さを測り、近接 benign data だけで fine-tuning したときの安全性劣化を解析した。
  3. [結果] Jailbreak Success Rate が最大 87.12% まで上がりうる一方、距離ベースのデータフィルタとテキスト system prompt でほぼ抑制できた。
- 一言メモ: 金融向け SFT でも「無害データなら安全」という前提は危険で、埋め込み距離ベースのデータクリーニングを先に入れるべきだと分かる。
