  2. [手法] 並列 trajectory を探索環境として扱い、候補解や中間情報を軽量 tool で必要時に inspect/search する AggAgent を提案した。
  3. [結果] 6 benchmark・3 model family で既存集約法を平均最大 5.3%、deep research task では最大 10.3% 上回った。
- 一言メモ: 複数 analyst agent による企業調査や投資仮説検証の最終統合 agent として、trajectory 検索型集約が有望。

### [T4] How Alignment Routes: Localizing, Scaling, and Controlling Policy Circuits in Language Models
- URL: https://arxiv.org/abs/2604.04385
- 重要度: 🔵Medium
- 要約:
  1. [課題] alignment 済み LLM が refusal や安全方針をどの内部回路で実行しているか、規模横断での因果構造は十分に分かっていなかった。
  2. [手法] intermediate-layer attention gate と deeper amplifier heads を interchange testing や knockout cascade で局在化し、複数モデルに共通する routing motif を調べた。
  3. [結果] gate は出力寄与が小さくても因果的に必須で、符号化や cipher により detection-layer pattern matching が破られると policy bypass が起きると示した。
- 一言メモ: 金融特化 FT 後の安全性監査では、表面 benchmark だけでなく policy routing が保持されているかを見る必要がある。

### [T4] SCOPE: Signal-Calibrated On-Policy Distillation Enhancement with Dual-Path Adaptive Weighting
- URL: https://arxiv.org/abs/2604.10688
- 重要度: 🔵Medium
- 要約:
  1. [課題] On-policy RL は outcome reward が疎で、OPD の token-level KL を全 rollout に一様適用すると信号品質の差を無視してしまう。
  2. [手法] 誤答 trajectory には teacher-perplexity weighted KL、正答 trajectory には student-perplexity weighted MLE を使う dual-path adaptive weighting を提案した。
  3. [結果] 6 reasoning benchmark で Avg@32 を平均 11.42%、Pass@32 を 7.30% 改善した。
- 一言メモ: 構造化出力の正答・誤答 trajectory を分け、境界事例だけ強める金融 schema alignment に相性がよい。

### [T4] The Past Is Not Past: Memory-Enhanced Dynamic Reward Shaping
- URL: https://arxiv.org/abs/2604.11297
- 重要度: ⚡High
- 要約:
  1. [課題] RL 後の LLM は同じ誤り pattern を繰り返しやすく、通常の entropy regularization だけでは recurrent failure を抑えにくい。
  2. [手法] 過去 rollout の中間表現を保存し、density-based clustering で頻出 error cluster を検出して動的に reward penalty を与える MEDS を提案した。
  3. [結果] 5 dataset・3 base model で既存法を一貫して上回り、pass@1 を最大 4.13pt、pass@128 を最大 4.37pt 改善した。
- 一言メモ: 金融 agent の schema 違反、根拠不足、同じ計算ミスを失敗 cluster として報酬に反映する設計に直結する。
