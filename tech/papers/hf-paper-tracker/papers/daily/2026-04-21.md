### [T3] The Amazing Agent Race: Strong Tool Users, Weak Navigators
- URL: https://arxiv.org/abs/2604.10261
- 重要度: ⚡High
- 要約:
  1. [課題] 既存のツール利用ベンチは線形タスクに偏り、実運用で重要な分岐・合流を伴うエージェント性能を測れていない。
  2. [手法] Wikipedia 上の DAG 型ツールチェーンを辿る 1,400 問のベンチマーク AAR を構築し、ナビゲーション・ツール利用・算術を分離評価した。
  3. [結果] 最高性能でも正答率 37.2% に留まり、主要な失敗要因はツール呼び出し自体ではなく「正しい場所へ辿る」ナビゲーションだった。
- 一言メモ: 金融AIエージェントでも、ツール精度より先に「どの資料・どの API に進むか」の探索設計を評価軸に入れるべきと示している。

### [T4] Where does output diversity collapse in post-training?
- URL: https://arxiv.org/abs/2604.16027
- 重要度: ⚡High
- 要約:
  1. [課題] ポストトレーニング後の LLM は出力多様性が低下し、探索的デコードや多様な候補生成の効果を削いでしまう。
  2. [手法] Olmo 3 の Think・Instruct・RL-Zero 系列を比較し、SFT・DPO・推論形式のどこで多様性崩壊が起きるかを分解した。
  3. [結果] 崩壊の主因は推論時フォーマットではなく学習データ構成で、特に SFT と DPO の設計差が多様性損失を左右することを示した。
- 一言メモ: 構造化出力の安定化を狙う SFT/DPO でも、候補多様性を潰しすぎると自己検証や rerank が弱くなるためデータ設計の見直し材料になる。

### [T4] DiPO: Disentangled Perplexity Policy Optimization for Fine-grained Exploration-Exploitation Trade-Off
- URL: https://arxiv.org/abs/2604.13902
- 重要度: 🔵Medium
- 要約:
  1. [課題] RLVR では難問への探索と易問での安定学習の両立が難しく、報酬最適化が不安定になりやすい。
  2. [手法] perplexity に基づいてサンプルを探索領域と活用領域に分け、双方向の報酬配分で細粒度に学習制御する DiPO を提案した。
  3. [結果] 数学推論と function calling で既存法より安定かつ高性能に動作し、探索・活用バランス改善の有効性を示した。
- 一言メモ: function calling を含むので、金融タスク向け DPO/KTO 前段の RL 調整でツール成功率を上げる設計に転用しやすい。

- URL: https://arxiv.org/abs/2604.15151
- 重要度: ⚡High

- 一言メモ: 金融AIエージェントのコード生成評価を一般ベンチで代用せず、実行可能性と意味整合まで含めて測る必要がある。

### [T2] Web Retrieval-Aware Chunking (W-RAC) for Efficient and Cost-Effective Retrieval-Augmented Generation Systems
- URL: https://arxiv.org/abs/2604.04936
- 重要度: ⚡High
- 要約:
  1. [課題] Web 文書の chunking は固定長だと検索品質が落ち、agentic chunking だとコストとデバッグ性が悪化する。
  2. [手法] HTML 由来の構造単位を ID 付きで保持し、LLM にはテキスト生成ではなく「検索向きのまとまり決定」だけをさせる W-RAC を提案した。
  3. [結果] 従来法と同等以上の検索性能を保ちつつ、chunking 関連の LLM コストを一桁削減した。
- 一言メモ: 社内文書・XBRL・IR 資料の取り込みで、chunk 生成を安く透明化できる実装パターンとして即試せる。

### [T3] GTA-2: Benchmarking General Tool Agents from Atomic Tool-Use to Open-Ended Workflows
- URL: https://arxiv.org/abs/2604.15715
- 重要度: ⚡High
- 要約:
  1. [課題] 短いツール呼び出し精度だけでは、実務ワークフローを最後まで完了できるエージェント性能を測れない。
  2. [手法] 実ユーザークエリと実運用ツールを使い、atomic task と open-ended workflow を階層的に評価する GTA-2 を構築した。
  3. [結果] 原子的タスクでも 50% 未満、ワークフロー完遂率は上位モデルでも 14.39% と大きな性能崖が確認された。
- 一言メモ: 金融エージェントも単発 tool-call ベンチでは足りず、調査からレポート作成までの checkpoint ベース評価が必要になる。

### [T3] Cut Your Losses! Learning to Prune Paths Early for Efficient Parallel Reasoning
- URL: https://arxiv.org/abs/2604.16029
- 重要度: 🔵Medium
- 要約:
  1. [課題] 並列推論は精度向上に効く一方、初期ミスを含む無駄な経路に計算資源を浪費しやすい。
  2. [手法] 経路刈り込み手法を体系化し、内部信号から prefix 段階で打ち切る学習型手法 STOP を提案した。
  3. [結果] 1.5B〜20B の複数モデルで効率と精度を両立し、固定計算予算下で AIME25 精度を大きく改善した。
- 一言メモ: マルチエージェントの探索木でも、早期失敗判定を入れれば PCIe 環境の推論コストを抑えやすい。

### [T2/T3] Don't Retrieve, Navigate: Distilling Enterprise Knowledge into Navigable Agent Skills for QA and RAG
- URL: https://arxiv.org/abs/2604.14572
- 重要度: ⚡High
- 要約:
  1. [課題] 通常の RAG は取得済み断片の受動消費に留まり、コーパス全体の構造を見渡しながら探索・後戻りできない。
  2. [手法] 文書群を階層的スキルディレクトリへオフライン蒸留し、サーブ時はエージェントが要約ツリーを辿って必要文書へ進む Corpus2Skill を提案した。

- 一言メモ: 社内文書 RAG を「検索」から「ナビゲーション可能な知識空間」へ変える設計で、複雑な金融調査に相性が良い。

### [T4] How to Fine-Tune a Reasoning Model? A Teacher-Student Cooperation Framework to Synthesize Student-Consistent SFT Data
- URL: https://arxiv.org/abs/2604.14164
- 重要度: ⚡High
- 要約:
  1. [課題] 強い教師モデルの合成データで reasoning model を SFT すると、教師と生徒の文体差で性能がむしろ落ちることがある。
  2. [手法] style token は生徒、非 style token は教師が交互生成する Teacher-Student Cooperation により、生徒整合な合成 SFT データを作る TESSY を提案した。
  3. [結果] GPT-OSS-120B 教師での code generation では、教師データ直学習の性能低下を反転し、LiveCodeBench-Pro と OJBench を大幅改善した。
- 一言メモ: 金融用 SFT データでも、専門性は強教師から借りつつ出力様式は自前モデルに合わせる設計が有効そうだ。

### [T4] C2: Scalable Rubric-Augmented Reward Modeling from Binary Preferences
- URL: https://arxiv.org/abs/2604.13618
- 重要度: 🔵Medium
- 要約:
  1. [課題] rubric 付き reward modeling は有効だが、人手 rubric は高コストで、低品質 rubric は逆に判断を壊す。
  2. [手法] 二値選好だけから有用 rubric と有害 rubric を対照生成し、協調的 rubric 生成器と批判的検証器を同時学習する C2 を提案した。
  3. [結果] 外部 rubric 注釈なしで RM-Bench や AlpacaEval 2.0 を改善し、小型 reward model でも大きな rubric 供給モデル並みの性能に近づけた。
- 一言メモ: 構造化出力や compliance 判定の報酬設計で、評価観点そのものを学習させる発想として有用。

### [T4] LongAct: Harnessing Intrinsic Activation Patterns for Long-Context Reinforcement Learning
- URL: https://arxiv.org/abs/2604.14922
- 重要度: 🔵Medium
- 要約:
  1. [課題] 長文脈 RL は更新対象が広すぎて非効率で、文脈が伸びるほど学習が不安定になりやすい。

- 一言メモ: 決算資料や規制文書の長文脈 RL 調整では、全重み更新より要所だけ触る方が現実的かもしれない。

### [T4] ASGuard: Activation-Scaling Guard to Mitigate Targeted Jailbreaking Attack
- URL: https://arxiv.org/abs/2509.25843
- 重要度: 🔵Medium
- 要約:
  1. [課題] 安全調整済み LLM でも時制変更のような軽微な言い換えで拒否行動が崩れる脆弱性がある。
  2. [手法] 脆弱性に関与する attention head を機械論的に特定し、channel-wise scaling を使った予防的 fine-tuning を行う ASGuard を提案した。
  3. [結果] 4 モデルで targeted jailbreaking 成功率を下げつつ、過剰拒否や一般性能低下を抑えた。
- 一言メモ: 金融領域の compliance ガードでも、表層ルール追加より脆弱回路への targeted 調整の方が実務的な可能性がある。

### [T2] UniDoc-RL: Coarse-to-Fine Visual RAG with Hierarchical Actions and Dense Rewards
- URL: https://arxiv.org/abs/2604.14967
- 重要度: 🔵Medium
- 要約:
  1. [課題] Visual RAG は粗い retrieval 信号に依存しがちで、図表や文書画像の重要領域を取り逃しやすい。
  2. [手法] 文書検索から画像選択、領域 crop までを階層アクションとして扱い、dense reward で学習する UniDoc-RL を提案した。

- 一言メモ: PDF 決算書や図表入り IR 資料を扱う金融 RAG で、ページ全体 retrieval から領域選択へ進む設計の参考になる。
