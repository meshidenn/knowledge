### [T4] Reinforcement Learning via Value Gradient Flow
- URL: https://arxiv.org/abs/2604.14265
- 重要度: 🔵Medium
- 要約:
  1. [課題] LLM後学習やオフラインRLでは、行動分布の外側で価値を過大最適化しやすく、既存の policy gradient や reject sampling は大規模化で扱いづらい。
  2. [手法] 参照分布から価値誘導分布への写像を最適輸送として捉え、粒子を価値勾配で動かす discrete gradient flow により暗黙的な正則化を行う VGF を提案した。
  3. [結果] D4RL、OGBench、LLM RL タスクで先行法を上回り、輸送予算の調整で test-time scaling も可能にした。
- 一言メモ: 報酬最適化の暴走を抑えつつ探索幅を制御する考え方で、金融特化後学習の安定化に応用余地がある。

### [T2][T3] Don't Retrieve, Navigate: Distilling Enterprise Knowledge into Navigable Agent Skills for QA and RAG
- URL: https://arxiv.org/abs/2604.14572
- 重要度: ⚡High
- 要約:
  1. [課題] 通常のRAGは検索結果を受け身で消費するだけなので、コーパス全体の構造を見渡せず、戻り探索や分散証拠の統合が弱い。
  2. [手法] 文書群を階層的スキルディレクトリへオフライン蒸留し、推論時はエージェントが要約ツリーを段階的に辿って本文へ到達する Corpus2Skill を提案した。
  3. [結果] 企業向けQAベンチマーク WixQA で dense retrieval、RAPTOR、agentic RAG を全評価指標で上回った。
- 一言メモ: 社内文書・XBRL・決算資料を「検索」ではなく「ナビゲーション可能な知識資産」に変える設計で、金融RAGの精度と監査性に直結する。

### [T4] Switch-KD: Visual-Switch Knowledge Distillation for Vision-Language Models
- URL: https://arxiv.org/abs/2604.14629
- 重要度: 🔵Medium
- 要約:
  1. [課題] VLM蒸留では視覚とテキストを別々に教師化しがちで、マルチモーダル整合が崩れて知識移転が不安定になる。
  2. [手法] 学生の視覚出力を教師の言語経路に通して共通のテキスト確率空間で蒸留する Visual-Switch Distillation と、双方向で差分を揃える DBiLD 損失を導入した。
  3. [結果] 0.5B TinyLLaVA が 3B 教師から知識を受け継ぎ、10ベンチ平均で 3.6 ポイント改善した。
- 一言メモ: 財務資料や図表VLMを小型運用する際、モダリティ整合を崩さず能力圧縮する実務的レシピとして参考になる。

### [T4] How to Fine-Tune a Reasoning Model? A Teacher-Student Cooperation Framework to Synthesize Student-Consistent SFT Data
- URL: https://arxiv.org/abs/2604.14164
- 重要度: ⚡High
- 要約:
  1. [課題] 強い教師の合成データで reasoning model をSFTすると、教師と生徒の文体分布のズレで性能が落ちることがある。
  2. [手法] 教師と生徒が交互にトークン生成し、教師の推論力と生徒の文体整合を両立する合成データを作る TESSY を提案した。
  3. [結果] 通常の教師生成データでは Qwen3-8B が LiveCodeBench-Pro で 3.25% 低下した一方、TESSY は 11.25% 改善し、OJBench でも 6.68% 改善した。
- 一言メモ: 金融SFTでも「正答」だけでなく「自モデルに馴染む書き方」で学習させることが重要だと示している。

### [T4] C2: Scalable Rubric-Augmented Reward Modeling from Binary Preferences
- URL: https://arxiv.org/abs/2604.13618
- 重要度: 🔵Medium
- 要約:
  1. [課題] rubric付き reward model は信頼性が高いが、良質な rubric 注釈の作成コストが重く、低品質 rubric は逆効果になりやすい。
  2. [手法] 二値選好だけから有益 rubric と誤誘導 rubric の対を合成し、協調的 rubric generator と批判的 verifier を同時学習する C2 を提案した。
  3. [結果] 外部 rubric 注釈なしで RM-Bench で最大 6.5 ポイント、AlpacaEval 2.0 長さ制御 win rate で 6.0 ポイント改善した。
- 一言メモ: 構造化出力の採点基準を rubric 化し、少量の選好から評価器を育てる方向にそのまま使える。

### [T4] LongAct: Harnessing Intrinsic Activation Patterns for Long-Context Reinforcement Learning
- URL: https://arxiv.org/abs/2604.14922
- 重要度: 🔵Medium
- 要約:
  1. [課題] 長文脈RLは重要な特徴に学習を集中できず、長距離依存を活かした最適化が非効率になりやすい。
  2. [手法] 長文脈処理時の高振幅 Q/K 活性に着目し、その活性に結び付く重みだけを疎に更新する saliency-guided sparse update を導入した。
  3. [結果] LongBench v2 で約 8% 改善し、RULER でも汎化を伸ばし、GRPO や DAPO など複数のRL法で一貫して有効だった。
- 一言メモ: 長い決算資料や社内文書を扱う金融エージェントの長文脈後学習を、低コストで効率化するヒントになる。

### [T4] ASGuard: Activation-Scaling Guard to Mitigate Targeted Jailbreaking Attack
- URL: https://arxiv.org/abs/2509.25843
- 重要度: 🔵Medium
- 要約:
  1. [課題] 安全化済みLLMでも、時制変更のような単純な言い換えで拒否挙動が崩れる targeted jailbreak が起きる。
  2. [手法] 回路解析で脆弱な attention head を特定し、channel-wise scaling vector で活性を補正して preventative fine-tuning へ組み込む ASGuard を提案した。
  3. [結果] 4つのLLMで攻撃成功率を下げつつ、通常能力と過剰拒否のバランスを保った。
- 一言メモ: 金融コンプライアンス領域では拒否・警告方針の堅牢性が重要で、機構的に狙い撃ちで補強する発想は価値が高い。

### [T2][T3] UniDoc-RL: Coarse-to-Fine Visual RAG with Hierarchical Actions and Dense Rewards

  2. [手法] 文書検索、再ランキング、画像選択、領域クロップ、推論を階層アクションとして統合し、各段階に dense reward を与える RL フレームワーク UniDoc-RL を提案した。

- 一言メモ: 決算PDFや図表付き資料をページ→図表→領域と掘り下げる金融ドキュメント読解エージェントにかなり近い。

### [T1] KV Packet: Recomputation-Free Context-Independent KV Caching for LLMs
- URL: https://arxiv.org/abs/2604.13226
- 重要度: ⚡High
- 要約:
  1. [課題] 標準的なKV cache は文脈依存なので、別コンテキストへ差し込む際に再計算が必要となり TTFT が増える。
  2. [手法] キャッシュ済み文書を不変パケットとして扱い、軽量な soft-token adapter を自己教師あり蒸留で学習して文脈の継ぎ目だけを補正する KV Packet を提案した。
  3. [結果] Llama-3.1 と Qwen2.5 で near-zero FLOPs と低TTFTを実現しつつ、完全再計算に近い F1 を維持した。
- 一言メモ: 頻出の社内資料や定番プロンプト断片を再利用しやすく、PCIe 環境での待ち時間削減に直接効く。

### [T2] ROSE: An Intent-Centered Evaluation Metric for NL2SQL
- URL: https://arxiv.org/abs/2604.12988
- 重要度: ⚡High
- 要約:
  1. [課題] NL2SQL の標準指標 Execution Accuracy は構文差や誤った正解SQLに弱く、ユーザー意図との整合を十分に測れていない。
  2. [手法] 予測SQLが質問意図に沿うかを独立評価する Prover と、正解SQLを証拠に反証する Refuter を組み合わせた intent-centered metric の ROSE を提案した。
  3. [結果] 専門家評価との一致で次点手法を Cohen's Kappa で約 24% 上回り、19手法の再評価から既存ランキングの歪みも示した。
- 一言メモ: 構造化出力スキーマ品質が最重要課題の現状では、SQLに限らず「意図中心評価」へ寄せる設計がそのまま効く。

### [T4] Self-Distillation Zero: Self-Revision Turns Binary Rewards into Dense Supervision
- URL: https://arxiv.org/abs/2604.12002
- 重要度: ⚡High
- 要約:
  1. [課題] RLVR は二値報酬しか得られず学習信号が疎で、外部教師や高品質デモに依存した蒸留はコストが高い。
  2. [手法] 同一モデルに Generator と Reviser の二役を担わせ、回答と二値報酬から改善版を生成させて、そのトークン分布を元モデルへ自己蒸留する SD-Zero を提案した。
  3. [結果] 数理・コード推論でベースモデル比 10%以上改善し、RFT、GRPO、SDFT を同条件で上回った。
- 一言メモ: 人手ラベルを増やしにくい金融タスクでも、検証可能な正誤信号から密な教師信号を自作できるのが強い。

### [T4] Target Policy Optimization
- URL: https://arxiv.org/abs/2604.06159
- 重要度: ⚡High
- 要約:
  1. [課題] PPO や GRPO は「どの出力へ確率質量を移すか」と「どう更新するか」を同時に解くため、疎報酬で不安定になりやすい。
  2. [手法] スコア済み候補から目標分布 q を先に構成し、その分布へ cross-entropy で追従する二段階最適化 TPO を提案した。
  3. [結果] bandit、Transformer 系列タスク、billion-scale LLM RLVR で easy task は既存法並み、疎報酬では PG、PPO、GRPO、DG を大きく上回った。
- 一言メモ: スキーマ遵守や拒否判断のような sparse reward 設定で、後学習の安定化候補としてかなり実務的。

### [T3] Memory Transfer Learning: How Memories are Transferred Across Domains in Coding Agents
- URL: https://arxiv.org/abs/2604.14004
- 重要度: ⚡High
- 要約:
  1. [課題] エージェントの記憶活用は同一ドメイン内に閉じがちで、異種タスク間の共通知見を十分に再利用できていない。
  2. [手法] 異なるベンチマークを横断する統一メモリプールを構築し、具体的トレースから抽象インサイトまで4種の記憶表現で転移性を比較した。
  3. [結果] 6つのコーディングベンチで平均 3.7% 改善し、低レベル痕跡より高レベルな検証ルーチンやメタ知識が転移を主導することを示した。
- 一言メモ: 金融エージェントでも案件固有回答より、検証手順や失敗パターンを記憶化する方が再利用価値が高い。

### [T3] Exploration and Exploitation Errors Are Measurable for Language Model Agents
- URL: https://arxiv.org/abs/2604.13151
- 重要度: 🔵Medium
- 要約:
  1. [課題] LM agent の失敗を探索不足と活用不足に分解して測る指標がなく、改善ポイントが見えにくい。
  2. [手法] 部分観測グリッドと未知DAGからなる可制御環境を作り、行動列から exploration / exploitation error を policy-agnostic に算出する評価法を設計した。
  3. [結果] 最先端エージェントでも両面の失敗が顕著で、reasoning model と最小限の harness engineering が改善に効くことを示した。
- 一言メモ: 金融ワークフローでも、失敗を「調査不足」と「判断活用不足」に分けて計測できると改善が進めやすい。

### [T4] From P(y|x) to P(y): Investigating Reinforcement Learning in Pre-train Space
- URL: https://arxiv.org/abs/2604.14142
- 重要度: 🔵Medium
- 要約:
  1. [課題] RLVR は条件付き分布 P(y|x) の最適化に留まり、ベースモデルが持つ出力空間そのものの制約を越えにくい。
  2. [手法] 事前学習空間の周辺分布 P(y) を報酬駆動で更新する PreRL と、負例強化で誤答空間を削ってから通常RLへ移る DSRL を提案した。
  3. [結果] NSR-PreRL により transition thought を 14.89 倍、reflection thought を 6.54 倍増やし、強いベースラインを上回った。
- 一言メモ: 金融推論の土台能力そのものを押し上げたいなら、SFT後の微修正だけでなく pre-train space 側の介入も視野に入る。
