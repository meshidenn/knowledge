### [T4] Self-Distilled RLVR
- URL: https://arxiv.org/abs/2604.03128
- 重要度: ⚡High
- 要約:
  1. [課題] 自己蒸留だけに頼る on-policy 学習は、特権情報の漏洩で更新が不安定になりやすい。
  2. [手法] 環境からの検証可能報酬で更新方向を決めつつ、自己蒸留で token-level の更新強度だけを与える RLSD を提案する。
  3. [結果] RLVR と OPSD の長所を両立し、より高い収束上限と安定した長期学習を実現した。
- 一言メモ: DPO/KTO 以外の後学習でも、正解情報の漏洩を避けながら細粒度な学習信号を足す設計として重要。

### [T3] CORAL: Towards Autonomous Multi-Agent Evolution for Open-Ended Discovery
- URL: https://arxiv.org/abs/2604.01658
- 重要度: ⚡High
- 要約:
  1. [課題] 既存の研究エージェントは固定ルール依存が強く、長期の知識蓄積を伴う open-ended discovery に弱い。
  2. [手法] 永続メモリ、非同期マルチエージェント実行、heartbeat 介入、分離 evaluator を備えた自律進化基盤を構築する。
  3. [結果] 10 タスクで SOTA を更新し、固定探索より 3〜10 倍高い改善率を少ない評価回数で達成した。
- 一言メモ: 金融リサーチ agent の「探索役・評価役・記憶役」分離と長期実験ループ設計に直結する。

### [T3] Therefore I am. I Think
- URL: https://arxiv.org/abs/2604.01202
- 重要度: ⚡High
- 要約:
  1. [課題] 推論モデルが本当に熟考してから行動を決めているのか、それとも先に決めて後から合理化しているのかは不明だった。
  2. [手法] 生成前活性からツール呼び出し判断を線形プローブで復号し、activation steering で決定方向を因果操作する。
  3. [結果] 思考トークン出力前から意思決定が高精度に検出され、介入で行動反転や熟考量の変化も起こせた。
- 一言メモ: エージェントの CoT を監査根拠として過信せず、事前決定バイアス前提の監視設計が必要だと分かる。

### [T3] Agentic-MME: What Agentic Capability Really Brings to Multimodal Intelligence?
- URL: https://arxiv.org/abs/2604.03016
- 重要度: 🔵Medium
- 要約:
  1. [課題] 既存の multimodal agent 評価は最終回答偏重で、ツールを正しく使ったかや無駄な overthinking を測れない。
  2. [手法] visual tool と web search を統合し、2,000 超の stepwise checkpoint で過程検証する Agentic-MME を作る。
  3. [結果] 最良モデルでも総合 56.3%、高難度では 23.0% に落ち、実世界の multimodal agent はまだ不十分だと示した。
- 一言メモ: 最終精度だけでなく、途中のツール選択と手順効率まで評価する設計は金融 agent 評価にもそのまま必要。

### [T3] Signals: Trajectory Sampling and Triage for Agentic Interactions
- URL: https://arxiv.org/abs/2604.00356
- 重要度: ⚡High
- 要約:
  1. [課題] 本番エージェントの軌跡は膨大かつ非決定的で、改善に効く事例だけを見つけるコストが高い。
  2. [手法] misalignment・loop・exhaustion などの軽量シグナルを live interaction に付与し、情報価値の高い軌跡を triage する。
  3. [結果] τ-bench で informative trajectory 比率 82% を達成し、heuristic 74% と random 54% を上回った。
- 一言メモ: 運用ログから SFT や preference data の候補だけを抜く後処理基盤として実装価値が高い。

### [T3] Ask or Assume? Uncertainty-Aware Clarification-Seeking in Coding Agents
- URL: https://arxiv.org/abs/2603.26233
- 重要度: ⚡High
- 要約:
  1. [課題] 現実のタスクはしばしば仕様不足だが、エージェントは確認せず自律実行に寄りすぎる。
  2. [手法] 曖昧性検知とコード実行を分離した uncertainty-aware マルチエージェント構成を導入する。

- 一言メモ: 金融業務の曖昧な依頼では、まず確認質問へ切り替える制御を入れるべきだと裏づける。

### [T3] AgentHazard: A Benchmark for Evaluating Harmful Behavior in Computer-Use Agents
- URL: https://arxiv.org/abs/2604.02947
- 重要度: ⚡High
- 要約:
  1. [課題] computer-use agent は各 step が局所的に妥当に見えても、連鎖すると危険行動に到達しうる。
  2. [手法] 有害目標と一見正当な操作列を組にした 2,653 件の AgentHazard benchmark で累積的危険行動を評価する。
  3. [結果] 現行システムは高い攻撃成功率を示し、Qwen3-Coder を用いた Claude Code では 73.63% に達した。
- 一言メモ: 金融オペレーション自動化では、単発ツール権限制御だけでなく長手順全体の危険判定が必要になる。

### [T3] AgentSocialBench: Evaluating Privacy Risks in Human-Centered Agentic Social Networks
- URL: https://arxiv.org/abs/2604.01487
- 重要度: 🔵Medium
- 要約:
  1. [課題] 複数ユーザー・複数ドメインをまたぐ agent 協調では、単一 agent より複雑な privacy leakage が起きうる。
  2. [手法] 階層的 sensitivity label と social graph を持つ 7 類型のシナリオで privacy risk を測る AgentSocialBench を提案する。
  3. [結果] cross-domain / cross-user 協調で leakage 圧力が強く、抽象化指示が逆に機微情報の議論を増やす abstraction paradox も確認した。
- 一言メモ: 顧客情報や社内機密を跨ぐ金融マルチエージェントでは、プロンプト指示だけで守る設計は危険だと分かる。

### [T3] Investigating Autonomous Agent Contributions in the Wild: Activity Patterns and Code Change over Time
- URL: https://arxiv.org/abs/2604.00917
- 重要度: 🔵Medium
- 要約:
  1. [課題] OSS における自律コーディングエージェントの実利用パターンと長期保守性は十分に把握されていない。
  2. [手法] 約 11 万件の OSS PR を用いて、複数エージェントの活動様式と生成コードの survival / churn を比較した。

- 一言メモ: 開発 agent は解決率だけでなく、後続改修コストまで含めて評価指標を置くべきだと分かる。

### [T1/T3] Executing as You Generate: Hiding Execution Latency in LLM Code Generation
- URL: https://arxiv.org/abs/2604.00491
- 重要度: 🔵Medium
- 要約:
  1. [課題] コード生成後に実行する直列パイプラインでは、生成器と実行器の待ち時間が重なって総遅延が大きい。
  2. [手法] AST ベース chunking と gated execution を用い、生成中コードを先行実行する三段パイプライン Eager を提案する。
  3. [結果] 非重複実行遅延を最大 99.9%、総遅延を最大 55% 削減した。
- 一言メモ: agent の tool execution を生成と重ねる設計は、低遅延が重要な実運用系でそのまま効く。

### [T2] MDPBench: A Benchmark for Multilingual Document Parsing in Real-World Scenarios
- URL: https://arxiv.org/abs/2603.28130
- 重要度: 🔵Medium
- 要約:
  1. [課題] 既存の文書解析評価は英語中心かつ整形済み文書寄りで、実運用の多言語・撮影文書を十分に測れていない。
  2. [手法] 17 言語・3,400 枚の digital / photographed document を含む多言語文書解析 benchmark を構築した。

- 一言メモ: 多言語 IR 資料や PDF ingest の前処理品質を測る実務ベンチとしてそのまま使える。

### [T1] Swift-SVD: Theoretical Optimality Meets Practical Efficiency in Low-Rank LLM Compression
- URL: https://arxiv.org/abs/2604.01609
- 重要度: ⚡High
- 要約:
  1. [課題] LLM 配備では重みと KV cache のメモリ帯域負荷が大きく、既存 SVD 圧縮は最適性か実用速度のどちらかを失いやすい。
  2. [手法] 活性共分散を逐次集約して一度だけ固有値分解する、training-free かつ activation-aware な closed-form 圧縮法を提案する。
  3. [結果] 6 モデル・8 データセットで SOTA を上回り、圧縮時間を 3〜70 倍高速化した。
- 一言メモ: PCIe 前提の大規模推論で、AWQ/INT4 に加えて low-rank 圧縮を重ねる現実的な選択肢になる。

### [T1/T4] Brainstacks: Cross-Domain Cognitive Capabilities via Frozen MoE-LoRA Stacks for Continual LLM Learning
- URL: https://arxiv.org/abs/2604.01152
- 重要度: 🔵Medium
- 要約:
  1. [課題] 継続学習で複数ドメインの専門性を積み増すと、忘却や adapter 干渉で品質が崩れやすい。
  2. [手法] QLoRA 4bit 上で MoE-LoRA stack を順次追加し、直交制約と outcome-based meta-router で必要な stack だけを合成する。

- 一言メモ: 金融・会計・対話などの LoRA を後付け合成したい現状の方針と相性が良い。

### [T4] Apriel-Reasoner: RL Post-Training for General-Purpose and Efficient Reasoning
- URL: https://arxiv.org/abs/2604.02007
- 重要度: 🔵Medium
- 要約:
  1. [課題] 多領域 RLVR は rollout 長や難易度差が大きく、精度と推論コストの両立が難しい。
  2. [手法] 適応的ドメイン sampling と difficulty-aware length penalty を導入し、難問だけ長く考えさせる RL 後学習を行う。
  3. [結果] 16K 出力予算でベースモデルを広範に改善しつつ、30〜50% 短い reasoning trace で強い open model 群に匹敵した。
- 一言メモ: 推論コスト制約が厳しい金融用途では、長考を一律許さず難易度連動で制御する設計が有効。
