### [T3] Signals: Trajectory Sampling and Triage for Agentic Interactions
- URL: https://arxiv.org/abs/2604.00356
- 重要度: 🔵Medium
- 要約:
  1. [課題] 本番運用中のエージェント軌跡は膨大かつ非決定的で、改善に使うべき失敗事例の選別コストが高い。
  2. [手法] モデル呼び出し不要の軽量シグナルで misalignment・loop・exhaustion などを付与し、情報価値の高い軌跡をトリアージする。
  3. [結果] τ-bench で情報価値の高い軌跡比率 82% を達成し、ヒューリスティック 74%・ランダム 54% を上回り、効率も 1.52 倍だった。
- 一言メモ: マルチエージェント運用ログから DPO/KTO 用の選別データを作る基盤としてそのまま転用しやすい。

### [T2] Forecasting Supply Chain Disruptions with Foresight Learning
- URL: https://arxiv.org/abs/2604.01298
- 重要度: ⚡High
- 要約:
  1. [課題] ノイズの多い非構造入力から、稀だが高インパクトな供給網障害を校正付きで予測するのは汎用モデルでは難しい。
  2. [手法] 実際の障害発生結果を教師信号にして、LLM に確率予測を出させる end-to-end の foresight learning を学習する。
  3. [結果] GPT-5 を含む強力なベースラインより精度・校正・適合率で上回り、明示的プロンプトなしでもより構造的な確率推論を獲得した。
- 一言メモ: 金融イベント予測やアラート生成で、説明文ではなく「校正済み確率」を返すドメイン特化モデル設計の参考になる。

### [T3] CORAL: Towards Autonomous Multi-Agent Evolution for Open-Ended Discovery
- URL: https://arxiv.org/abs/2604.01658
- 重要度: ⚡High
- 要約:
  1. [課題] 既存の研究エージェントは探索規則や制御ロジックが固定的で、長期の知識蓄積を伴う open-ended discovery に弱い。
  2. [手法] 共有永続メモリ、非同期マルチエージェント実行、heartbeat 介入、分離 evaluator を備えた自律進化フレームワークを導入する。
  3. [結果] 数学・アルゴリズム・システム最適化の 10 タスクで SOTA を更新し、固定探索ベースラインより 3〜10 倍高い改善率を少ない評価回数で達成した。
- 一言メモ: 金融リサーチ agent 群の「探索役・評価役・記憶役」分離設計に直結する。

### [T2] Working Notes on Late Interaction Dynamics: Analyzing Targeted Behaviors of Late Interaction Models
- URL: https://arxiv.org/abs/2603.26259
- 重要度: 🔵Medium
- 要約:
  1. [課題] Late Interaction 検索は高性能だが、長文バイアスや MaxSim の実効挙動が十分に理解されていない。
  2. [手法] NanoBEIR 上で、長さバイアスと top-1 以降の類似度分布を state-of-the-art な Late Interaction モデルで解析する。
  3. [結果] 因果型だけでなく双方向型でも極端条件では長さバイアスが出ることを示し、MaxSim は実質 top-1 文書トークンを主に利用していると確認した。
- 一言メモ: 社内文書・決算資料 RAG で multi-vector retriever を使うなら、長文優遇の補正設計が必要になる。

### [T3] Ask or Assume? Uncertainty-Aware Clarification-Seeking in Coding Agents
- URL: https://arxiv.org/abs/2603.26233
- 重要度: 🔵Medium
- 要約:
  1. [課題] 現実のタスクはしばしば指示が不足しているが、エージェントは確認せず勝手に進めて失敗しやすい。
  2. [手法] 曖昧性検出と実行を分離した uncertainty-aware のマルチエージェント構成で、必要時のみ clarification を発火させる。
  3. [結果] underspecified SWE-bench Verified で 69.40% の解決率となり、単一エージェントの 61.20% を上回った。
- 一言メモ: 金融業務で曖昧な依頼を受けた際に、勝手な仮定より確認質問を優先させる安全策として有効。

### [T2] MDPBench: A Benchmark for Multilingual Document Parsing in Real-World Scenarios
- URL: https://arxiv.org/abs/2603.28130
- 重要度: 🔵Medium
- 要約:
  1. [課題] 文書解析ベンチマークは英語中心かつ整形済み文書偏重で、実運用の多言語・撮影文書を十分に評価できていない。
  2. [手法] 17 言語・デジタル文書・撮影文書を含む 3,400 画像の多言語 document parsing benchmark を構築する。
  3. [結果] オープンモデルは撮影文書で平均 17.8%、非ラテン文字で 14.0% の大幅劣化を示し、Gemini3-Pro などの閉源系が相対的に頑健だった。
- 一言メモ: 多言語決算資料や PDF ingest の弱点が見えやすく、金融 RAG の前処理品質評価に使える。

### [T1/T4] Brainstacks: Cross-Domain Cognitive Capabilities via Frozen MoE-LoRA Stacks for Continual LLM Learning
- URL: https://arxiv.org/abs/2604.01152
- 重要度: 🔵Medium
- 要約:
  1. [課題] 継続学習で複数ドメインの専門性を積み増すと、忘却や adapter 干渉で生成品質が崩れやすい。
  2. [手法] 4bit QLoRA 上での MoE-LoRA、凍結スタックの残差ブースティング、直交制約、メタルータを組み合わせた積層型 continual FT を提案する。
  3. [結果] 単一 LoRA より 2.5 倍速く収束し、stack 蓄積で壊れた品質をルーティングで回復しつつ、認知スキルの横断再利用も観測した。
- 一言メモ: PCIe 前提の量子化 MoE 基盤で、金融・会計・対話などの LoRA を後付け合成したい場合の有力な雛形になる。

### [T3] Omni-SimpleMem: Autoresearch-Guided Discovery of Lifelong Multimodal Agent Memory
- URL: https://arxiv.org/abs/2604.01007
- 重要度: ⚡High
- 要約:
  1. [課題] 長期運用エージェントでは、マルチモーダル経験の保持・整理・想起の設計空間が広すぎて人手探索が追いつかない。
  2. [手法] 自律研究パイプラインが約 50 回の実験を回し、バグ修正・アーキテクチャ変更・プロンプト改善を通じて lifelong memory を自動発見する。
  3. [結果] LoCoMo で F1 を 0.117 から 0.598、Mem-Gallery で 0.254 から 0.797 へ伸ばし、最大寄与はハイパーパラメータではなくバグ修正や設計変更だった。
- 一言メモ: エージェント記憶は「検索器追加」より、失敗分析と自動実験ループ込みで改善する方が効くことを示している。

### [T4] DataFlex: A Unified Framework for Data-Centric Dynamic Training of Large Language Models
- URL: https://arxiv.org/abs/2603.26164
- 重要度: ⚡High
- 要約:
  1. [課題] データ選別・混合比最適化・再重み付けは別々の実装に分断され、再現性と比較可能性が低い。
  2. [手法] LLaMA-Factory 上に sample selection・domain mixture adjustment・sample reweighting を統合した動的データ学習基盤を実装する。
  3. [結果] 動的選別は static full-data 学習を一貫して上回り、DoReMi/ODM も MMLU と perplexity を改善し、元実装よりランタイム効率も良かった。
- 一言メモ: 最大課題の一つである SFT データ品質改善に対して、学習法より先にデータ制御面を整備すべきという示唆が強い。

### [T3/T4] SKILL0: In-Context Agentic Reinforcement Learning for Skill Internalization
- URL: https://arxiv.org/abs/2604.02268
- 重要度: ⚡High
- 要約:
  1. [課題] skill retrieval はノイズ、長いコンテキスト、知識の非内在化という根本制約を持つ。
  2. [手法] 学習時は完全 skill 文脈から始め、役立つ skill を段階的に引き剥がしながらゼロショット自律実行へ移行する in-context RL を行う。
  3. [結果] ALFWorld で +9.7%、Search-QA で +6.6% 改善しつつ、各 step の文脈長を 0.5k token 未満に抑えた。
- 一言メモ: MCP/ツール手順を毎回注入する代わりに、重要スキルをモデル内在化してレイテンシと文脈コストを削る方向性として重要。

### [T4] S0 Tuning: Zero-Overhead Adaptation of Hybrid Recurrent-Attention Models
- URL: https://arxiv.org/abs/2604.01168
- 重要度: 🔵Medium
- 要約:
  1. [課題] 少量の検証済みデータでモデル適応したいが、LoRA でもマージや推論コストの負担が残る。
  2. [手法] ハイブリッド recurrent-attention モデルの各層で初期 state 行列だけを最適化する zero-overhead PEFT を提案する。
  3. [結果] Qwen3.5-4B 系で LoRA を HumanEval で大きく上回り、MATH-500 や GSM8K への転移も示しつつ、推論オーバーヘッドは増えなかった。
- 一言メモ: 将来 recurrent-hybrid 系を採るなら、金融タスク別の軽量切替 PEFT として LoRA 代替候補になる。
