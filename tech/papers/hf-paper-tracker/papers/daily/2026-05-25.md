### [T4] LoREnc: Low-Rank Encryption for Securing Foundation Models and LoRA Adapters
- URL: https://arxiv.org/abs/2605.13163
- 重要度: 🔵Medium
- 要約:
  1. [課題] オンデバイス向け基盤モデルとLoRAアダプタは知財漏洩やモデル復元攻撃に弱い。
  2. [手法] 学習不要で基盤重みの支配的な低ランク成分を抑制し、認可済みアダプタ側で欠落情報を補償するLoREncを提案した。
  3. [結果] 未認可利用では出力が崩壊し、認可利用では性能を維持したまま計算オーバーヘッドを1%未満に抑えた。
- 一言メモ: 社内配布用LoRAや顧客別アダプタを守る実装テンプレとして有用で、金融向け微調整資産の保護に直結する。

### [T3] Rule2DRC: Benchmarking LLM Agents for DRC Script Synthesis with Execution-Guided Test Generation
- URL: https://arxiv.org/abs/2605.15669
- 重要度: 🔵Medium
- 要約:
  1. [課題] 自然言語ルールから正しい実行可能スクリプトを作るタスクでは、コード類似度評価では実用性能を測れない。
  2. [手法] 1,000件のrule-to-scriptタスクと実行ベース評価を備えたRule2DRCを構築し、実行結果で候補を切り分けるtester agentも導入した。
  3. [結果] 実行誘導のテスト生成により、Best-of-N選択の精度を大きく改善した。
- 一言メモ: 金融エージェントでも「SQL・Excel式・バリュエーション手順」を実行結果で検証する設計が重要だと示す。

### [T3] Efficient Agentic Reasoning Through Self-Regulated Simulative Planning
- URL: https://arxiv.org/abs/2605.22138
- 重要度: ⚡High
- 要約:
  1. [課題] 既存のagentic reasoningは必要以上に長い思考列を生成し、トークン効率と精度の両立が難しい。
  2. [手法] 未来状態を模擬する計画器と、いつどこまで計画するかを決める自己制御器を分離したSR^2AMを提案した。
  3. [結果] 30B級で大規模モデルに競合するPass@1を達成しつつ、推論トークンを25.8〜95.3%削減した。
- 一言メモ: レイテンシ制約の厳しい金融マルチエージェントで「毎回深く考えない」制御を学習させる方向としてかなり実践的。

### [T4] Unsupervised Process Reward Models
- URL: https://arxiv.org/abs/2605.10158
- 重要度: ⚡High
- 要約:
  1. [課題] Process Reward Modelは有効だが、各推論ステップの専門家アノテーションが高コストでスケールしない。
  2. [手法] 次トークン確率から誤りステップ位置を推定する教師なしPRMを学習し、最終解の正解ラベルにも依存しない形にした。
  3. [結果] 誤り検出でLLM-as-a-Judgeを上回り、test-time scalingやRL報酬としても教師ありPRM級の性能を示した。
- 一言メモ: 構造化出力や金融推論の中間ステップ監督を安価に増やせる可能性があり、DPO/KTO前段の品質管理に効く。

### [T4] DelTA: Discriminative Token Credit Assignment for Reinforcement Learning from Verifiable Rewards
- URL: https://arxiv.org/abs/2605.21467
- 重要度: ⚡High
- 要約:
  1. [課題] RLVRでは応答レベル報酬がフォーマット等の高頻度トークンに埋もれ、真に効くトークンへの信用割当が弱い。
  2. [手法] 正負応答を分ける識別的方向を強調するトークン係数を導入し、RLVR更新を再重み付けするDelTAを提案した。
  3. [結果] 数学7ベンチで同スケール強ベースラインを上回り、コード生成や他バックボーンにも一般化した。
- 一言メモ: JSON schema準拠や根拠付き出力のような疎な成功条件を強化したい金融エージェントに相性がよい。

### [T1] Full Attention Strikes Back: Transferring Full Attention into Sparse within Hundred Training Steps
- URL: https://arxiv.org/abs/2605.16928
- 重要度: ⚡High
- 要約:
  1. [課題] 長文推論では完全注意の二乗コストが重く、疎化は精度・学習コスト・実装負荷のトレードオフが大きい。
  2. [手法] 少数のretrieval headだけフルKVを保持し、16次元インデクサと動的top-p選択で疎注意へ短時間転写するRTPurboを提案した。
  3. [結果] 数百ステップの適応でほぼ無損失のまま、1M文脈で最大9.36倍prefill高速化、約2倍decode高速化を達成した。
- 一言メモ: 長文RAGとツール軌跡を扱うMoE基盤の推論速度改善候補として今日いちばん直接的。

### [T1] KVServe: Service-Aware KV Cache Compression for Communication-Efficient Disaggregated LLM Serving
- URL: https://arxiv.org/abs/2605.13734
- 重要度: ⚡High
- 要約:
  1. [課題] 分離型サービングではKVキャッシュ転送がネットワークと保存の支配的ボトルネックになる。
  2. [手法] 圧縮手法群を戦略空間として統合し、Bayesian profilingとオンラインbandit制御でサービス条件ごとに最適圧縮を選ぶKVServeを提案した。
  3. [結果] vLLM統合実験でPD分離時に最大9.13倍JCT改善、KV分離時に最大32.8倍TTFT短縮を示した。
- 一言メモ: PCIe前提の複数GPU構成では通信最適化の価値が高く、EP運用や長文RAGの実サービス設計に直結する。

### [T1] Gated DeltaNet-2: Decoupling Erase and Write in Linear Attention
- URL: https://arxiv.org/abs/2605.22791
- 重要度: 🔵Medium
- 要約:
  1. [課題] 線形注意は定数メモリで有利だが、圧縮メモリの消去と書き込みを同一ゲートで制御すると情報破壊が起きやすい。
  2. [手法] erase gateとwrite gateを分離し、channel-wise decay込みで高速並列学習できるGated DeltaNet-2を提案した。
  3. [結果] 1.3B規模でMamba系や既存delta-rule系を上回り、長文検索系ベンチで特に強い性能を示した。
- 一言メモ: 直接MoE置換ではないが、長文金融文書の低メモリ推論アーキテクチャ候補として監視価値がある。

### [T2/T4] Training Large Language Models to Predict Clinical Events
- URL: https://arxiv.org/abs/2605.12817
- 重要度: 🔵Medium
