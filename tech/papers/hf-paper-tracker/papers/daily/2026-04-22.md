### [T1] River-LLM: Large Language Model Seamless Exit Based on KV Share
- URL: https://arxiv.org/abs/2604.18396
- 重要度: ⚡High
- 要約:
  1. [課題] decoder-only LLM の early exit は、途中層を飛ばすと後続トークンに必要な KV cache が欠け、理論上の層削減が実測速度に繋がりにくい。
  2. [手法] 学習不要の KV-Shared Exit River を導入し、欠落 KV を自然に補いながら token-level early exit を行う River-LLM を提案した。
  3. [結果] 数学推論とコード生成で品質を保ちながら 1.71〜2.16 倍の実効高速化を達成した。
- 一言メモ: PCIe 前提の多GPU推論で重要なレイテンシ削減に直結し、MoE 本体をいじらず速度改善を狙える点が実務向き。

### [T3] Symbolic Guardrails for Domain-Specific Agents: Stronger Safety and Security Guarantees Without Sacrificing Utility
- URL: https://arxiv.org/abs/2604.15579
- 重要度: ⚡High
- 要約:
  1. [課題] 高リスク業務のツール利用エージェントでは、学習ベースや neural guardrail だけでは安全制約を保証できない。
  2. [手法] 80 本の安全・セキュリティ系ベンチを精査し、どのポリシー要件が symbolic guardrail で強制可能かを整理したうえで各種ベンチで実証した。
  3. [結果] 明示された要件の 74% は比較的単純な symbolic guardrail で保証可能で、τ^2-Bench などで utility を落とさず安全性を改善した。
- 一言メモ: 金融AIエージェントの権限制御や compliance 制約は、曖昧なプロンプト規律より symbolic rule で先に固める方が監査しやすい。

### [T3] On the Reliability of Computer Use Agents
- URL: https://arxiv.org/abs/2604.17849
- 重要度: ⚡High
- 要約:
  1. [課題] computer-use agent は同じタスクでも再実行ごとに成否がぶれ、単発成功率だけでは実運用の信頼性を評価できない。
  2. [手法] OSWorld 上で同一タスクの反復実行を行い、実行時の確率性、タスク仕様の曖昧さ、行動変動の3因子に分けて分析した。
  3. [結果] 信頼性はモデル能力だけでなく仕様の書き方と挙動の安定性に強く左右され、反復評価と曖昧さ解消の対話設計が必要だと示した。
- 一言メモ: 金融ワークフローでも「一度できた」では不十分で、同一条件の再現性を KGI に入れないと運用事故が残る。

### [T4] The Illusion of Certainty: Decoupling Capability and Calibration in On-Policy Distillation
- URL: https://arxiv.org/abs/2604.16830
- 重要度: ⚡High
- 要約:
  1. [課題] on-policy distillation は精度を上げても過信を強めやすく、実運用時の信頼度が能力と切り離されて悪化する。
  2. [手法] 教師側の特権情報と配備時情報のずれが誤較正を生むと定式化し、モデル自身の rollout から empirical confidence を学ぶ CaOPD を提案した。
  3. [結果] 複数モデル・複数領域で能力を保ちながら calibration を改善し、OOD や continual learning 条件でも頑健性を示した。
- 一言メモ: 金融用途では「自信満々の誤答」が最悪なので、構造化出力の正確さだけでなく confidence calibration を別目標で最適化すべきと分かる。

### [T2] On the Robustness of LLM-Based Dense Retrievers: A Systematic Analysis of Generalizability and Stability
- URL: https://arxiv.org/abs/2604.16576
- 重要度: ⚡High
- 要約:
  1. [課題] decoder-only LLM を使う dense retriever は性能が伸びている一方、データセット跨ぎの汎化性とノイズ耐性の検証が不足していた。
  2. [手法] 30 データセット横断で一般化性能を混合効果モデルで評価し、typo・paraphrase・corpus poisoning への安定性も系統比較した。
  3. [結果] instruction-tuned 系は強いが reasoning 特化モデルには specialization tax があり、語彙ゆらぎには強くても semantic perturbation には脆いことを示した。
- 一言メモ: 社内文書・決算資料 RAG では、単純な平均スコアより query 言い換えや表記揺れ耐性を先に見ないと本番品質を誤る。

### [T3] GenericAgent: A Token-Efficient Self-Evolving LLM Agent via Contextual Information Density Maximization (V1.0)
- URL: https://arxiv.org/abs/2604.17091
- 重要度: ⚡High
- 要約:
  1. [課題] 長期タスクのエージェントは、ツール記述・記憶・環境ログが文脈を圧迫し、重要情報が埋もれて性能が落ちる。
  2. [手法] 最小ツール集合、階層的オンデマンド記憶、検証済み軌跡から SOP とコードを作る自己進化、文脈圧縮層を統合した GA を提案した。
  3. [結果] 複数の agent system を上回りつつ、使用トークン数とインタラクション数を大きく削減し、継続的な性能向上も示した。
- 一言メモ: PCIe 環境では test-time compute が高価なので、「長く考える」より「情報密度を上げる」設計の方が効率改善に繋がる。

### [T3] Agents Explore but Agents Ignore: LLMs Lack Environmental Curiosity
- URL: https://arxiv.org/abs/2604.17609
- 重要度: ⚡High
- 要約:
  1. [課題] 既存 agent は環境内で有用情報を発見しても、その意味を汲んで方針を変える environmental curiosity が弱い。
  2. [手法] Terminal-Bench、SWE-Bench、AppWorld の環境中に完全な解法を意図的に埋め込み、発見率と活用率のギャップを測定した。
  3. [結果] 解法を見つけても活用できないケースが多く、道具立て・test-time compute・訓練分布が curiosity を左右することを示した。
- 一言メモ: 金融エージェントでも、証拠を見つけた後に戦略更新できるかを別途評価しないと「見たのに使えない」失敗が残る。

### [T2/T3] Latent Preference Modeling for Cross-Session Personalized Tool Calling
- URL: https://arxiv.org/abs/2604.17886
- 重要度: ⚡High
- 要約:
  1. [課題] ユーザーはツール呼び出しに必要な詳細を毎回明示しないため、複数セッションにまたがる嗜好記憶がないと under-specified input が増える。
  2. [手法] Preference Recall・Induction・Transfer を含む MPT ベンチを作り、嗜好を仮説として保持して generate-verify-refine で更新する PRefine を提案した。
  3. [結果] full-history prompting の 1.24% のトークン量で tool-calling 精度を改善し、単なる事実記憶より「選好理由」の保持が重要と示した。
- 一言メモ: 投資家やアナリストごとの出力粒度・優先指標・禁止操作を跨セッションで保持する設計にそのまま応用できる。

### [T4] GFT: From Imitation to Reward Fine-Tuning with Unbiased Group Advantages and Dynamic Coefficient Rectification
- URL: https://arxiv.org/abs/2604.14258
- 重要度: ⚡High
- 要約:
  1. [課題] SFT は知識注入に強い一方で単一路線依存やエントロピー崩壊を招きやすく、その後の RL との接続も滑らかでない。
  2. [手法] 多様な応答群から正規化 advantage を作る Group Advantage Learning と、重みの暴走を抑える Dynamic Coefficient Rectification を組み込んだ GFT を提案した。
  3. [結果] SFT 系手法を一貫して上回り、後段の RL への接続も安定化した。
- 一言メモ: 金融用 SFT で専門用語を入れつつ、その後の DPO/KTO に繋ぎたい現状にかなり近い後学習レシピ。

### [T3] SkillFlow:Benchmarking Lifelong Skill Discovery and Evolution for Autonomous Agents
- URL: https://arxiv.org/abs/2604.17308
- 重要度: ⚡High
- 要約:
  1. [課題] 既存評価は「与えられた skill を使えるか」が中心で、skill を発見・修復・再利用する lifelong learning 能力を測れていない。
  2. [手法] 20 ファミリ 166 タスクからなる SkillFlow を構築し、skill なし初期状態から逐次タスクを解きながら skill library を更新する設定で評価した。
  3. [結果] lifelong skill evolution は有効だが、skill usage の多さは utility を保証せず、発見・パッチ・転移それぞれに大きな能力差が残った。
- 一言メモ: 金融エージェントでも skill を増やすだけでは足りず、いつ呼ぶか・いつ修正するかまで含めて評価する必要がある。

### [T4] When Can LLMs Learn to Reason with Weak Supervision?
- URL: https://arxiv.org/abs/2604.18574
- 重要度: ⚡High
- 要約:
  1. [課題] RLVR は有効だが、高品質 reward を常に用意できないため、少量データ・ノイズ報酬・自己教師報酬下で何が効くかが重要になる。
  2. [手法] 弱教師条件を横断して学習ダイナミクスを分析し、事前特性として reasoning faithfulness が一般化成否を予測するかを検証した。
  3. [結果] 一般化するモデルは reward saturation 前の伸長期を持ち、明示的 reasoning trace による SFT と domain continual pretraining の併用が有効だった。
- 一言メモ: 金融タスクでは完全な報酬設計が難しいので、まず faithful な中間推論を作る SFT を厚くしてから弱報酬 RL に入るのが現実的。

### [T3] ClawEnvKit: Automatic Environment Generation for Claw-Like Agents
- URL: https://arxiv.org/abs/2604.18543
- 重要度: ⚡High
- 要約:
  1. [課題] claw-like agent の訓練・評価環境は人手作成が前提で、継続的な能力差分の計測や targeted training がスケールしない。
  2. [手法] 自然言語記述から task specification・tool interface・scoring を自動生成し、妥当性や多様性まで検証する ClawEnvKit を提案した。

### [T4] Crowded in B-Space: Calibrating Shared Directions for LoRA Merging
- URL: https://arxiv.org/abs/2604.16826
- 重要度: ⚡High
- 要約:
  1. [課題] 複数 LoRA を後から merge すると干渉で性能が落ちやすく、特にどこで情報衝突が起きるかが不明瞭だった。
  2. [手法] LoRA の出力側行列 B に共有方向が集中する点を特定し、過剰共有方向だけを事前補正する data-free 手法 Pico を提案した。
  3. [結果] 数学・コード・金融・医療を含む 8 ベンチで既存 merge 手法を平均 3.4〜8.3 ポイント改善し、全データ joint LoRA を超えるケースも示した。
- 一言メモ: 金融サブドメイン別 LoRA を後段で統合したい場合、単純 merge ではなく B-space 校正を挟む価値が高い。
