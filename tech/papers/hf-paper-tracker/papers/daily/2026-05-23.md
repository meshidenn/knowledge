### [T1] Full Attention Strikes Back: Transferring Full Attention into Sparse within Hundred Training Steps
- URL: https://arxiv.org/abs/2605.16928
- 重要度: ⚡High
- 要約:
  1. [課題] 長文コンテキスト推論ではフルアテンションの二乗コストが大きく、効率化手法は精度か追加学習コストを犠牲にしがちだった。
  2. [手法] フルアテンション LLM の内在的疎性を利用し、少数の retrieval head のみフル KV を保持しつつ軽量 indexer と動的 top-p 選択で疎化する RTPurbo を提案した。
  3. [結果] 数百 step の適応だけで 1M 文脈の prefill 最大 9.36 倍、decode 約 2.01 倍を達成しつつ精度劣化をほぼ抑えた。
- 一言メモ: 長文 RAG とマルチエージェントのコンテキスト肥大に直撃するので、PCIe 環境での MoE 推論最適化候補として最優先。

### [T1] KVServe: Service-Aware KV Cache Compression for Communication-Efficient Disaggregated LLM Serving
- URL: https://arxiv.org/abs/2605.13734
- 重要度: ⚡High
- 要約:
  1. [課題] 分離型 LLM Serving では KV キャッシュがネットワーク越しの主要ボトルネックになり、固定的な圧縮設定では workload や SLO 変動に追従できない。
  2. [手法] 圧縮方式を再構成可能な戦略空間として統合し、Bayesian profiling とオンライン bandit controller で状況依存に最適プロファイルを選ぶ KVServe を提案した。
  3. [結果] vLLM 統合評価で PD-separated serving の JCT を最大 9.13 倍、KV-disaggregated の TTFT を最大 32.8 倍改善した。
- 一言メモ: PCIe 前提の複数 GPU 配備では通信最適化が重要なので、EP 構成時の KV 転送制御にそのまま応用発想がある。

### [T1] Gated DeltaNet-2: Decoupling Erase and Write in Linear Attention
- URL: https://arxiv.org/abs/2605.22791
- 重要度: 🔵Medium
- 要約:
  1. [課題] 線形注意は定数メモリで有利だが、圧縮メモリ更新時に erase と write を同一ゲートで制御すると長文依存の保持が崩れやすい。
  2. [手法] channel-wise erase gate と write gate を分離し、decay を保ったまま高速並列学習できる Gated Delta Rule-2 を導入した。
  3. [結果] 1.3B 規模で KDA や Mamba 系を上回り、特に長文 retrieval 系ベンチで優位性を示した。
- 一言メモ: ソフトマックス注意以外の長文基盤を探る材料として有用だが、現行 MoE 本線に直結するかは追加検証が必要。

### [T3] Efficient Agentic Reasoning Through Self-Regulated Simulative Planning
- URL: https://arxiv.org/abs/2605.22138
- 重要度: ⚡High
- 要約:
  1. [課題] Agentic LLM は必要以上に長い推論を行いがちで、token 消費の増加に対して精度向上が安定しない。
  2. [手法] world model に基づく simulative reasoning、計画深度を決める self-regulation、反応的 execution の 3 系統に分けた SR^2AM を提案した。
  3. [結果] 30B モデルで 120B〜1T 級相当の Pass@1 に迫りつつ、既存 agentic LLM 比で reasoning token を 25.8〜95.3% 削減した。
- 一言メモ: 金融エージェントの計画深度をケース別に制御できれば、精度を落とさずレイテンシと API コストを下げやすい。

### [T3] Rule2DRC: Benchmarking LLM Agents for DRC Script Synthesis with Execution-Guided Test Generation
- URL: https://arxiv.org/abs/2605.15669
- 重要度: 🔵Medium
- 要約:
  1. [課題] コード生成エージェントの評価は類似度依存になりやすく、実行正しさや候補選別の実運用性を捉えにくかった。
  2. [手法] 1,000 タスク・13,921 レイアウトの実行ベース評価基盤 Rule2DRC と、実行結果から識別的テストを作る SplitTester を導入した。
  3. [結果] 評価時に正解テストを agent に渡さずに機能正しさを測定でき、Best-of-N 選別性能も大きく改善した。
- 一言メモ: 金融ツール実行エージェントでも、コード類似ではなく実行結果で候補を絞る評価設計が必要だと再確認できる。

### [T2/T3] ClinSeekAgent: Automating Multimodal Evidence Seeking for Agentic Clinical Reasoning
- URL: https://arxiv.org/abs/2605.20176
- 重要度: ⚡High
- 要約:
  1. [課題] 既存の臨床エージェント研究は curated evidence 前提が多く、実運用で必要な生データからの能動的証拠収集を扱えていなかった。
  2. [手法] KB 検索、EHR 走査、画像ツール呼び出しを通じて仮説更新しながら証拠収集する inference-time agent と、その軌跡蒸留パイプラインを提案した。
  3. [結果] テキスト EHR・マルチモーダル双方で大幅改善し、蒸留した 35B-A3B モデルもベースラインを大きく上回った。
- 一言メモ: 社内文書・XBRL・決算資料をまたぐ金融 RAG でも、受動的検索より evidence-seeking 軌跡の学習が効く可能性が高い。

### [T2/T3] Spreadsheet-RL: Advancing Large Language Model Agents on Realistic Spreadsheet Tasks via Reinforcement Learning
- URL: https://arxiv.org/abs/2605.22642
- 重要度: ⚡High
- 要約:
  1. [課題] 汎用 LLM をプロンプトだけで spreadsheet 操作させる方式は、実務的な多段ワークフローで失敗しやすい。
  2. [手法] Excel 環境上の Spreadsheet Gym、start-goal シート自動収集、finance/supply-chain を含む Domain-Spreadsheet を使う RL fine-tuning 枠組みを構築した。
  3. [結果] SpreadsheetBench と domain-specific ベンチの両方で Pass@1 を大きく改善し、実運用寄りタスクでの一般化を示した。
- 一言メモ: 財務モデリングやポートフォリオ分析の表計算自動化に直結し、金融エージェントの具体的な作業面で最も試しやすい。

### [T3] Maestro: Reinforcement Learning to Orchestrate Hierarchical Model-Skill Ensembles
- URL: https://arxiv.org/abs/2605.22177
- 重要度: 🔵Medium
- 要約:
  1. [課題] 単一 LLM と固定ルールに依存するエージェントは、モデルやスキルの補完関係を活かし切れない。
  2. [手法] モデル選択・スキル選択・終了判断を逐次意思決定として扱い、4B の軽量 orchestrator を outcome-based RL で学習する Maestro を提案した。
  3. [結果] 10 のマルチモーダルベンチ平均で GPT-5 と Gemini-2.5-Pro を上回り、未学習のモデルやスキル追加にも一般化した。
- 一言メモ: 金融タスク別に「汎用 reasoning モデル」「表計算スキル」「RAG 検索器」を切り替える設計の有力な参照例。

### [T2] TransitLM: A Large-Scale Dataset and Benchmark for Map-Free Transit Route Generation
- URL: https://arxiv.org/abs/2605.22355
- 重要度: 🔵Medium
- 要約:
  1. [課題] 経路計画は従来ルールベース基盤への依存が強く、非構造化入力から end-to-end に学習する大規模データ基盤が不足していた。
  2. [手法] 4 都市・1,300 万件超の route planning 記録を継続事前学習コーパス兼ベンチとして整備した。
  3. [結果] 学習した LLM は明示的地図なしでも妥当なルートを生成し、GPS から適切な駅への暗黙 grounding を示した。
- 一言メモ: 金融でもルールエンジン直書きではなく、大量の業務軌跡から構造化出力を学ぶ方針の後押しになる。

### [T4] DelTA: Discriminative Token Credit Assignment for Reinforcement Learning from Verifiable Rewards
- URL: https://arxiv.org/abs/2605.21467
- 重要度: ⚡High
- 要約:
  1. [課題] RLVR では応答単位の報酬がどの token にどう配賦されるかが曖昧で、書式のような高頻度 token に更新が引っ張られやすい。
  2. [手法] token-gradient の正負側 centroid をより識別的にする係数を推定し、共有的な更新方向を抑える DelTA を提案した。
  3. [結果] 数学 7 ベンチで同規模強ベースラインを上回り、コード生成や異なる backbone にも一般化した。
- 一言メモ: 構造化出力品質を上げたいなら、正答だけでなく schema 準拠に効く token へ報酬を寄せる発想が重要。

### [T4] Unsupervised Process Reward Models
- URL: https://arxiv.org/abs/2605.10158
- 重要度: ⚡High
- 要約:
  1. [課題] PRM は強力だが、ステップ単位アノテーションの収集コストが高く、推論監督の大規模化を阻む。
  2. [手法] 次 token 確率から最初の誤り位置を評価する教師なしスコア関数で uPRM を学習し、最終解答ラベルも不要にした。
  3. [結果] 誤り位置同定で LLM-as-a-Judge を最大 15pt 上回り、test-time scaling と RL reward の両方でも supervised PRM 級の効果を示した。
- 一言メモ: 金融 reasoning データの逐次ラベル付けは高価なので、教師なし PRM はアライメントの現実解になりうる。

### [T3/T4] ACC: Compiling Agent Trajectories for Long-Context Training
- URL: https://arxiv.org/abs/2605.21850
- 重要度: ⚡High
- 要約:
  1. [課題] 長文 reasoning 学習用データの作成は高コストで、通常の agent SFT は tool response を mask してしまい文脈統合能力の監督が不足する。
  2. [手法] 検索・ソフトウェア開発・DB クエリの agent 軌跡を、質問と散在する観測証拠を束ねた long-context QA にコンパイルする ACC を提案した。
  3. [結果] Qwen3-30B-A3B を 235B 級に近い MRCR/GraphWalks 水準まで押し上げつつ、一般能力を維持した。
- 一言メモ: 自社エージェントの実行ログをそのまま長文 SFT データへ変換できるので、RAG とツール利用を同時に鍛えやすい。

### [T4] From Reasoning Chains to Verifiable Subproblems: Curriculum Reinforcement Learning Enables Credit Assignment for LLM Reasoning
- URL: https://arxiv.org/abs/2605.22074
- 重要度: 🔵Medium
- 要約:
  1. [課題] hard reasoning では最終正解ロールアウトが希少で、通常の outcome-based RLVR では partial progress を学習信号に変えにくい。
  2. [手法] 参照 reasoning chain から検証可能な subproblem を作り、各 subproblem 位置ごとに報酬正規化する SCRL を提案した。
  3. [結果] 7 数学ベンチで GRPO を安定して上回り、AIME/IMO 系でも pass@1 と pass@64 を改善した。
- 一言メモ: 複雑な金融分析を検証可能な中間課題へ分割すれば、DPO/KTO 以外の RL 強化にも道筋ができる。

### [T2/T4] Training Large Language Models to Predict Clinical Events
- URL: https://arxiv.org/abs/2605.12817
- 重要度: 🔵Medium
- 要約:
  1. [課題] 時系列ノートから将来イベント予測の監督信号を作るのは難しく、手設計特徴や専用分類器に依存しやすい。
  2. [手法] 過去文脈・将来イベント質問・後続記録で解決されるラベルという形式に変換し、LoRA adapter で予測タスクを学習した。
  3. [結果] 小規模 adapter でも base model より calibration と Brier score を改善し、一部で GPT-5 の point estimate を上回った。
- 一言メモ: 金融でも時系列メモやレポートから「将来イベント質問」を生成する形で、少量 LoRA による予測特化が試せる。
