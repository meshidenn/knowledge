### [T2+T3] Can LLM Agents Be CFOs? A Benchmark for Resource Allocation in Dynamic Enterprise Environments
- URL: https://arxiv.org/abs/2603.23638
- 重要度: ⚡High
- 要約:
  1. [課題] LLMエージェントが不確実性下での長期リソース配分（CFO的意思決定）を行えるか未検証

- 一言メモ: 投資判断・ポートフォリオ管理エージェントが直面する長期意思決定課題の定量的評価として自社ベンチマーク設計の参考に

### [T4] Why Does Self-Distillation (Sometimes) Degrade the Reasoning Capability of LLMs?
- URL: https://arxiv.org/abs/2603.24472
- 重要度: ⚡High
- 要約:
  1. [課題] 自己蒸留（self-distillation）が数学的推論で応答長を短縮しながら性能を最大40%低下させる現象が未解明
  2. [手法] コンディショニング情報量とタスクカバレッジを制御した実験で、教師の不確実性表現（epistemic verbalization）の抑制が原因と特定
  3. [結果] Qwen3-8B・DeepSeek-Distill-Qwen-7B・Olmo3-7B-Instructで最大40%の性能低下、OODタスクで損失が特に大きい
- 一言メモ: 金融専門用語SFTデータ作成に自己蒸留を使う際、不確実性表現の維持が必須という直接的な警告

### [T1+T2] MSA: Memory Sparse Attention for Efficient End-to-End Memory Model Scaling to 100M Tokens
- URL: https://arxiv.org/abs/2603.23516
- 重要度: 🔵Medium
- 要約:
  1. [課題] LLMの有効コンテキスト長が1Mトークン程度に制限され大規模コーパス要約や長期エージェント推論に対応できない
  2. [手法] スケーラブルスパースアテンション＋ドキュメントワイズRoPE＋Memory Parallelで学習・推論ともに線形計算量を実現、100Mトークンを2×A800 GPUで処理
  3. [結果] 16K→100Mトークンスケーリングで9%未満の品質劣化、フロンティアLLM・RAGシステムをlong-contextベンチマークで上回る
- 一言メモ: 社内文書・XBRL・決算資料の大規模RAGパイプラインにおける外部ストレージ依存を削減できる代替アーキテクチャとして検討価値

### [T3] SlopCodeBench: Benchmarking How Coding Agents Degrade Over Long-Horizon Iterative Tasks
- URL: https://arxiv.org/abs/2603.24755
- 重要度: 🔵Medium
- 要約:
  1. [課題] コーディングエージェントが長期的な反復拡張タスクでコード品質（冗長性・構造崩壊）が劣化することがシングルショット評価では見えない
  2. [手法] 20問題・93チェックポイントの言語非依存ベンチマークで verbosity（冗長コード比）とstructural erosion（複雑度集中度）を軌跡レベルで追跡
  3. [結果] 11モデル中どれもエンドツーエンドを解かず、エージェントコードは人間コードの2.2倍冗長で反復ごとに悪化
- 一言メモ: 金融レポート生成や構造化出力スキーマ生成のコーディングエージェントにおいて、長期品質保持が主要課題であることを示す
