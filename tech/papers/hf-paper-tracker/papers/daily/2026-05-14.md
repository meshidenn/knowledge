### [T1] Efficient Pre-Training with Token Superposition
- URL: https://arxiv.org/abs/2605.06546
- 重要度: ⚡High
- 要約:
  1. [課題] LLM 事前学習は FLOPs あたりのデータ処理効率が低く、大規模化すると学習時間が重くなりやすい。
  2. [手法] 連続トークンを bag 化して multi-hot cross-entropy で学習する superposition phase と、通常学習へ戻す recovery phase からなる TST を提案した。
  3. [結果] 並列化や tokenizer を変えずに 10B-A1B MoE を含む複数規模で頑健に機能し、同等 loss 条件で事前学習時間を最大 2.5 倍短縮した。
- 一言メモ: MoE ベースモデルを継続学習させる際のコスト削減策として現実味が高く、PCIe 環境でも学習予算を節約しやすい。

### [T1] Solve the Loop: Attractor Models for Language and Reasoning
- URL: https://arxiv.org/abs/2605.12466
- 重要度: 🔵Medium
- 要約:
  1. [課題] Looped Transformer は反復推論で有望だが、学習不安定性と反復深さ依存のコストがボトルネックだった。
  2. [手法] backbone が出した埋め込みを attractor module が fixed point まで反復精緻化し、implicit differentiation で学習する Attractor Models を導入した。
  3. [結果] perplexity と下流精度で標準 Transformer を上回り、小型モデルの難推論でも強く、推論時に solver を外しても性能を保つ現象を示した。
- 一言メモ: test-time compute を増やして精度を取りたい金融推論で、反復計算を学習時に内在化する発想は長期的に面白い。

### [T1/T4] A Single Layer to Explain Them All:Understanding Massive Activations in Large Language Models
- URL: https://arxiv.org/abs/2605.08504
- 重要度: 🔵Medium
- 要約:
  1. [課題] LLM 内部で生じる massive activations は性能劣化や attention sink と関係するが、発生源が十分理解されていなかった。
  2. [手法] Massive Emergence Layer を特定し、RMSNorm と FFN の寄与を解析したうえで massive activation token の硬直化を弱める補正法を提案した。
  3. [結果] instruction following と数理推論で一貫して性能を改善し、attention sink の緩和にもつながることを示した。
- 一言メモ: 長文 RAG と構造化出力で起きる注意の偏りを内部表現側から抑える手掛かりとして有用。

### [T2] IndustryBench: Probing the Industrial Knowledge Boundaries of LLMs
- URL: https://arxiv.org/abs/2605.10267
- 重要度: ⚡High
- 要約:

### [T2] Do not copy and paste! Rewriting strategies for code retrieval
- URL: https://arxiv.org/abs/2605.08299
- 重要度: 🔵Medium
- 要約:
  1. [課題] embedding ベース retrieval は表層表現に引きずられやすく、LLM による query/corpus 書き換えの効き方も未整理だった。
  2. [手法] stylistic rephrase、擬似コード化、完全自然言語化の 3 系統を query-corpus 両側または corpus 側のみで比較し、効果予測指標 Delta H を導入した。
  3. [結果] query と corpus を同時に書き換える設定で大幅改善しつつ、corpus-only は多くの設定で劣化し、事前判定指標が有効であることを示した。
- 一言メモ: 社内文書や XBRL 検索でも、埋め込みを替える前に query rewrite を条件付きで差し込む戦略が現実的。

### [T2/T4] A Causal Language Modeling Detour Improves Encoder Continued Pretraining
- URL: https://arxiv.org/abs/2605.12438
- 重要度: 🔵Medium
- 要約:
  1. [課題] ドメイン適応 encoder の継続事前学習は MLM 一択になりがちで、低層表現の更新が弱い可能性があった。

- 一言メモ: 金融文書向け encoder を作るなら、決算書や規程文書への continued pretraining 手順としてそのまま試せる。

### [T3] LLM Agents Already Know When to Call Tools -- Even Without Reasoning
- URL: https://arxiv.org/abs/2605.09252
- 重要度: ⚡High
- 要約:
  1. [課題] tool-augmented agent は不要なツール呼び出しを多発し、レイテンシと API コストを悪化させる。
  2. [手法] tool 必要性を測る When2Tool ベンチを作り、 hidden state から必要性を読む線形 probe と prefill 文で行動を誘導する Probe&Prefill を提案した。
  3. [結果] 6 モデルで tool 必要性が高精度に線形可読で、精度ほぼ維持のままツール呼び出しを 48% 削減した。
- 一言メモ: 市場データ API や社内検索を多用する金融エージェントでは、不要 call 削減がそのまま速度とコストに効く。

### [T4] Learning, Fast and Slow: Towards LLMs That Adapt Continually
- URL: https://arxiv.org/abs/2605.12484
- 重要度: ⚡High
- 要約:
  1. [課題] パラメータ更新だけに頼る学習は forgetting と plasticity 低下を招き、継続適応に弱い。
  2. [手法] slow weights としてモデル本体、fast weights として最適化された文脈を併用する Fast-Slow Training を導入した。
  3. [結果] RL 単独より最大 3 倍 sample-efficient で高い到達性能を示し、base model からの drift を抑えつつ次タスク適応も改善した。
- 一言メモ: 金融用語精度を上げながら汎用 reasoning を壊したくないので、SFT/RL の後段設計としてかなり重要。

### [T4] FocuSFT: Bilevel Optimization for Dilution-Aware Long-Context Fine-Tuning
- URL: https://arxiv.org/abs/2605.09932
- 重要度: ⚡High
- 要約:
  1. [課題] 長文 SFT では attention が位置的に目立つ token に吸われ、意味的に重要な文書断片へ十分な勾配が届かない。
  2. [手法] 内側ループで fast-weight memory を形成して relevant content へ attention を集中させ、その表現条件付きで外側 SFT を回す bilevel 学習を導入した。
  3. [結果] BABILong と RULER で大きく改善し、agentic tool use を含む GPQA でも pass@1 を 24% 相対改善した。
- 一言メモ: 長い決算資料や社内文書を読む金融 agent の fine-tuning で、注意散漫そのものを学習時に矯正できる点が強い。
