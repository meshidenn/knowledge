# 📊 2026-W21 週次トレンド分析
期間: 2026-05-18 〜 2026-05-24 | 分析論文数: 93本

## 1. テーマ別ハイライト
### T1 MoE / 効率的アーキテクチャ
今週のT1は、MoEそのものの設計刷新よりも、PCIe前提の実運用ボトルネックを直接削る推論最適化が主役だった。BEAM、Post-Trained MoE Can Skip Half Experts via Self-Distillation、TIDE は expert 発火数や offload を制御して、MoE を「賢く作る」より「安く回す」方向を強めている。KV 系では OSCAR、OScaR、KVServe、CompactAttention、WorldKV が、量子化・通信圧縮・再注入を組み合わせて長文脈と分離サービングの両立を狙っており、長文RAGやマルチエージェント実行の基盤に直結する。GQLA や Draft Less, Retrieve More も含め、単一の高速化手法より workload やハードウェア条件に応じて decode path を切り替える適応型制御が増えている。金融用途では、AWQ/INT4 に加えて KV 圧縮と通信最適化を重ねる前提で推論基盤を設計する週だった。

### T2 金融AI / ドメイン特化LLM
T2は、金融そのものの論文数は多くない一方で、金融RAGにそのまま刺さる評価軸と作業環境がかなり揃った。WildTableBench、CiteVQA、FINESSE-Bench、DocAtlas は、表理解、根拠位置、金融知識、多言語文書という現在の課題をそれぞれ明示的に測るため、社内ベンチの部品として使いやすい。Spreadsheet-RL は finance を含む spreadsheet 環境で RL を回しており、財務モデリング自動化の最短ルートになりうる。MLAIRE、Active Learners as Efficient PRP Rerankers、Toto 2.0 は、多言語検索・再ランキング・時系列予測を強化しており、社内文書RAGと市場データ活用を別系統でなく統合タスクとして扱う流れを後押ししている。今週のT2は、「金融知識を答えさせる」より「金融文書を正しく読み、根拠を示し、表計算や予測まで含めて動く」方向に重心が移った。

### T3 マルチエージェント / ツール利用
T3は件数でも密度でも今週の中心で、評価、記憶、計画、スキル統治、環境生成が一気につながった。WildClawBench、TerminalWorld、OpenComputer、π-Bench、SaaSBench、MINTEval、Auditing Agent Harness Safety は、モデル単体でなく harness・runtime・shared state を含めて測る流れを決定的にしている。PREPING、STALE、EvolveMem、PEEK、Evaluating Temporal Semantic Caching、S-Bus は、長手数エージェントの成否が memory quality と state coordination に強く依存することを示した。Maestro、SR^2AM、EnvFactory、AstraFlow、SkillsVote は、オーケストレーションや skill lifecycle そのものを学習・統治対象にしており、固定ワークフロー前提が崩れつつある。金融AIエージェントの観点では、「良いモデルを一つ選ぶ」より「どのモデル・スキル・メモリをどう切り替え、どう監査するか」が性能差の主戦場になっている。

### T4 ファインチューニング / アライメント
T4では、RLVR を大きく回すより、少量ステップで効く報酬設計と credit assignment を詰める研究が目立った。DelTA、Process Rewards with Learned Reliability、CEPO、Not Every Rubric Teaches Equally、You Only Need Minimal RLVR Training は、最終正解だけではなく、どの token・どの rubric・どの部分軌跡に報酬を効かせるかを細かく最適化している。uPRM、SCRL、Unsupervised Process Reward Models、From Reasoning Chains to Verifiable Subproblems は、専門家アノテーションを減らしつつ検証可能な中間信号を増やす方向で、金融のように高価な教師データ環境と相性が良い。Safety Alignment as Continual Learning や The Unlearnability Phenomenon in RLVR for Language Models は、後学習の副作用と限界も可視化しており、単純なRL追加が万能でないことを再確認させた。今週のT4は、構造化出力品質を上げるなら「長く学習する」より「正しい箇所にだけ強く学習を当てる」設計が重要だと示した。

## 2. テーマ横断マップ
今週いちばん強かった接続は、T1の計算効率化を土台に、T3の長手数エージェントを現実的に回し、T4の検証可能報酬で品質を詰める三層構造だった。BEAM、OSCAR、KVServe、Mix-Quant は、長文RAGや多段ツール実行で増える KV と通信コストを削る基盤であり、その上で SR^2AM、Maestro、EnvFactory、OpenComputer が plan-execute や model-skill orchestration を組み立てている。さらに DelTA、CEPO、uPRM、ACC が、そこで発生する実行軌跡を学習信号へ変換する役割を担っている。

T2とT3の接続では、ClinSeekAgent、Spreadsheet-RL、Nexus が象徴的だった。いずれも受動的QAではなく、証拠探索、表計算操作、時系列予測をエージェント行動として扱っており、金融AIでも「回答品質」より「根拠収集と作業遂行」を主対象にするべきだと示している。CiteVQA と WildTableBench は、その行動結果を根拠位置や表理解で採点する評価基盤として噛み合う。

T3とT4の接続も強く、ACC、AstraFlow、EnvFactory、Self-Distilled Agentic Reinforcement Learning、Learning to Build the Environment が、agent 軌跡を long-context SFT や RLVR に戻す閉ループを具体化した。これは金融エージェントの運用ログを、そのまま次の学習資産へ変える道筋を与える。今週の横断的な結論は、「推論最適化」「実行系評価」「軌跡再学習」が別々のテーマではなく、ひとつの運用ループとして統合され始めたことにある。

## 3. 金融AIエージェントへの示唆
### 今週のアクションアイテム（最大3つ）
- BEAM、OSCAR、KVServe を優先候補に、PCIe 前提の MoE 推論基盤で `expert 発火削減` と `KV 通信圧縮` の小規模ベンチを切る。
- CiteVQA、WildTableBench、FINESSE-Bench を参考に、`根拠位置一致` `表理解` `金融知識` を分けて測る社内評価セットを作る。
- ACC と DelTA を組み合わせ、既存の agent 実行ログから `long-context SFT データ` と `schema 準拠重視の RL 信号` を作る実験系を始める。

### 中期的に注視すべき方向
金融エージェントの差別化は、単一モデルの知識量よりも、runtime 制御層の質に移りつつある。具体的には、Maestro のような model-skill routing、PREPING や EvolveMem のような memory bootstrapping/self-tuning、OpenComputer や EnvFactory のような verifiable environment を組み合わせた運用基盤が重要になる。RAG も単なる検索補助ではなく、ClinSeekAgent や Spreadsheet-RL のように evidence-seeking と executable workflow を含む形へ広がる可能性が高い。学習面では、uPRM、CEPO、SCRL のような低ラベル・部分報酬型の RLVR が、金融の高コスト教師データ制約に対する現実解になりそうだ。

### 今週スキップしてよい理由
今週は world model や GUI/computer-use 系の論文も多かったが、直近の金融AIエージェント改善には優先度を落としてよい。特に純粋な personal assistant proactivity や terminal 操作の一部は、評価設計の参考にはなるものの、現時点では構造化出力品質、RAG根拠性、MoE推論効率ほど直接的ではない。先に取りにいくべきなのは、T1の推論基盤最適化、T2の根拠付き文書理解、T3/T4の軌跡再学習ループである。今週の論文群だけでも、十分に実装候補が過密で、探索を広げすぎるより絞って検証した方がよい。
