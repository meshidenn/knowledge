# 📊 2026-W15 週次トレンド分析
期間: 2026-04-06 〜 2026-04-12 | 分析論文数: 131本

## 1. テーマ別ハイライト
### T1 MoE / 効率的アーキテクチャ
今週のT1は、長文脈・高スループット・省メモリを同時に狙う実装寄りの改善が目立った。推論側では TriAttention、Flux Attention、DMax、MARS が、KV圧縮・並列デコード・複数トークン生成によって PCIe 環境でも効く現実的な高速化を示している。MoE では REAM や Brainstacks が、expert 削減より expert 統合や MoE-LoRA の積層によって品質を守りつつ運用可能性を上げる方向を示した。Swift-SVD や LightThinker++ まで含めると、今週の本流は「モデルを大きくする」より「既存資産を圧縮・整理して長く速く回す」方向にある。

### T2 金融AI / ドメイン特化LLM
T2では、RAG を最終回答だけでなく証拠寄与と検索分布まで掘って評価・改善する流れが明確だった。CUE-R、Learning to Retrieve from Agent Trajectories、Score Distribution 蒸留、Cross-Lingual Alignment、PLUME は、それぞれ証拠介入、軌跡由来学習、教師分布保持、多言語補正、latent reasoning embedding という異なる角度から検索品質を底上げしている。金融文脈では、Type-Checked Compliance と DeonticBench が規制・ルール推論を評価可能な問題として具体化し、確率的 guardrail の限界を超える方向を示した。全体として、社内文書・多言語開示・規制制約をまたぐ金融RAGの基盤技術が一段具体化した週だった。

### T3 マルチエージェント / ツール利用
T3は今週最も厚く、中心テーマは「記憶」「スキル」「安全性」「評価」の4本柱だった。CORAL、SkillX、Graph of Skills、Combee、SEVerA、Externalization review などから、エージェントは単発推論器ではなく、経験を外部化し再利用する運用システムとして設計され始めている。評価面では ClawBench、ClawsBench、ClawArena、KnowU-Bench、Agentic-MME が、成功率だけでなく evolving environment、interruptibility、stepwise process を測る方向へ進んだ。安全性でも AgentHazard、AgentWatcher、OpenClaw 分析、HDP が、権限・記憶・委譲 provenance を分離して管理すべきことを強く示している。

### T4 ファインチューニング / アライメント
T4は RL 後学習の安定化と、推論崩壊をどう防ぐかが主戦場だった。Self-Distilled RLVR、Sample Routing、RAGEN-2、Faithful GRPO、Graph-Based CoT Pruning、ThinkTwice、Learning to Hint は、学習信号の粗さ、過剰反省、入力非依存な思考テンプレ化を抑える具体策を出している。加えて Rethinking Generalization in Reasoning SFT や On the Step Length Confounding は、単に長い reasoning trace を集めても一般化しないことを整理し、データ選別の重要性を押し上げた。DataFlex と合わせてみると、今週のT4は「より強いRL」より「崩れにくいデータと更新則」の設計へ重心が移った。

## 2. テーマ横断マップ
今週もっとも強い接続は、T2の証拠中心RAGとT3の軌跡中心エージェント評価が、T4の安定な後学習へ接続している点である。CUE-R や agent trajectory retrieval が「どの証拠・どの途中判断が効いたか」を可視化し、RAGEN-2 や Self-Distilled RLVR がその信号を壊さず学習へ戻す構図が見えた。
第二の接続は、T3の memory / skill 外部化と T1 のコンテキスト圧縮・推論高速化の組み合わせである。LightThinker++、Squeez、TriAttention、MARS は、長期エージェントを成立させるために「全部を文脈へ残さない」設計が必須であることを補強した。
第三の接続は、金融用途に固有の強いシグナルとして Type-Checked Compliance が浮上した点である。これは T2 の規制推論、T3 のツール実行制御、T4 の構造化出力安定化を一本に束ねる論文であり、今週の全テーマを結ぶハブになっていた。

## 3. 金融AIエージェントへの示唆
### 今週のアクションアイテム（最大3つ）
- 構造化出力の後段に、Lean 4 風の決定論的ルール検証レイヤを置く最小プロトタイプを作る。まずは投資判断やコンプライアンス文面ではなく、社内スキーマ整合性チェックから始めるのが現実的。
- RAG 評価を最終正答率中心から、証拠除去・置換・重複に対する頑健性評価へ拡張する。CUE-R 型の介入評価を社内文書検索に導入し、どの資料断片が本当に効いているかを測るべき。
- 推論最適化では TriAttention、MARS、Squeez を優先して小規模検証する。RTX 6000 Pro × PCIe 環境では、モデル再学習より KV 圧縮・並列生成・ツール出力刈り込みの方が短期ROIが高い。

### 中期的に注視すべき方向
- スキルは数を増やすより、検索後の refinement と dependency-aware retrieval を持たせる方向が有望である。
- RL 後学習は報酬強化一本ではなく、trajectory triage、sample routing、reasoning collapse 監視を組み合わせた閉ループ運用が本命になりつつある。
- 金融向けでは、規制準拠・監査ログ・委譲 provenance を含む安全設計が、性能改善と同じ優先度に上がっている。

### 今週スキップしてよい理由
- dLLM やマルチモーダル推論の一部は興味深いが、現行の金融エージェント基盤に直結するのは自己回帰LLM・RAG・ツール利用・後学習の改善である。
- world model や汎用GUI agentの派手な結果より、今週は guardrail、evidence evaluation、memory compression の方が短期実装価値が高い。
- したがって、リソース配分は新規モデル系よりも、既存スタックの運用安定化と評価基盤強化に寄せてよい。
