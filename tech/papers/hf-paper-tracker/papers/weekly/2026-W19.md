# 📊 2026-W19 週次トレンド分析
期間: 2026-05-04 〜 2026-05-10 | 分析論文数: 99本

## 1. テーマ別ハイライト
### T1 MoE / 効率的アーキテクチャ
今週のT1は、MoEを「大きいが扱いにくいモデル」から「部分起動・挙動制御・PCIe前提運用が可能なモデル」へ寄せる動きが強かった。RoundPipeはPCIe接続の複数GPUでLoRA/QLoRA微調整を現実化し、RTX 6000 Pro級の制約に最も近い実装論文だった。EMOとUniPoolは、expertを全量常駐させずに済む構造や、層をまたいで共有するexpert poolにより、NVLinkなし環境での配備コスト圧縮に直結する。MASCingはMoEのルーティングを再学習なしに軽量制御できる点が重要で、安全制約や出力ポリシー切替を運用層で吸収する方向が見えた。Step-level Optimizationやspeculative decoding系の流れも含め、T1は単体推論高速化より「どこで重い計算を使うか」を設計する週だった。

### T2 金融AI / ドメイン特化LLM
今週のT2は、RAGを単なる埋め込み検索から「証拠束を揃える探索システム」へ再定義する論文が集中した。Rethinking Reasoning-Intensive Retrieval、Beyond Semantic Similarity、Hierarchical Abstract Tree、OpenSearch-VL、Web2BigTableは、top-k関連文書ではなく、補完的証拠を集める検索・構造化集約・直接コーパス操作の重要性を共通して示している。Chain of Evidenceは図表やレイアウトを含むPDF根拠をそのまま扱えるため、IR資料や決算説明会スライドを監査可能に扱う金融RAGと相性が良い。TabEmbed、Prior-Aligned Data Cleaning、BioToolも、表データ・実務API・ドメイン固有ツールを前提にした設計へ踏み込んでおり、金融向けにはXBRL・財務テーブル・価格APIをまとめて扱う基盤整備の優先度が上がった。総じて、金融RAGの差分は「何を埋め込むか」より「どう探索面と証拠管理を設計するか」に移っている。

### T3 マルチエージェント / ツール利用
今週のT3は、search agent、research agent、skill-evolving agent、workspace agentが一斉に進み、エージェント研究の中心が単発タスクから長期運用へ完全に移った。ARIS、OpenSeeker-v2、OpenSearch-VL、Auto Research with Specialist Agents、Intern-Atlasは、探索軌跡・監査・失敗履歴・手法系譜の蓄積まで含めて研究エージェントを設計している。SkillOS、Skill1、From Context to Skills、From Skill Text to Skill Structureは、スキルを増やすだけでなく、棚卸し・再利用・機械可読化まで最適化対象にし始めた。Claw-Eval-Live、Workspace-Bench、WindowsWorld、AcademiClawは、実務ワークフロー評価ではまだ成功率が低く、成果物と実行証跡を採点する必要を強く示した。金融AIエージェントにとっては、MCP連携の拡充より先に、探索ログ、スキル資産、監査可能なワークスペース評価を持つことが勝負どころになってきた。

### T4 ファインチューニング / アライメント
今週のT4は、RLを強くするというより、RLとSFTを壊さず運用するための安定化・不確実性制御・再評価が主題だった。Safety Drift After Fine-Tuningは、下流適応後に安全性が大きく揺れることを示し、金融向けLoRAやDPO後の再監査を必須工程に押し上げた。Tsallis loss continuum、T^2PO、A^2TGPO、Balanced Aggregation、ResRLは、cold start、turn単位credit、長さ依存バイアス、負例学習の副作用といった実装上の痛点をかなり具体的に潰している。The First Token Knows、HiL-Bench、Hallucinations Undermine Trustは、不確実性を推定して検索・確認・人手依頼に戻る制御を、評価と学習の両面から前面化した。構造化出力の品質を最重要改善ターゲットとする自プロジェクトにとって、今週のT4は「より賢く答える」より「危険時に止まる・聞く・再検索する」を学習させる週だった。

## 2. テーマ横断マップ
今週もっとも強かった横断軸は、「検索面の再設計（T2）」「長期エージェントの運用設計（T3）」「不確実性に基づく制御（T4）」の合流である。Web2BigTable、OpenSearch-VL、Beyond Semantic Similarity、Chain of Evidenceは、証拠収集をretrieverの問題ではなく、ツール付きエージェントの作業面設計として扱っていた。そこにHiL-Bench、The First Token Knows、Hallucinations Undermine Trust、T^2POが重なり、足りない証拠を感じたら再検索や確認に戻る制御ループが明示化された。さらにSkillOS、Skill1、Ctx2Skill、ARISが、その探索や確認の仕方自体をスキルとして資産化し、将来のエージェントへ引き継ぐ流れを作っている。T1側ではRoundPipe、EMO、UniPool、MASCingが、こうした重いオーケストレーションをPCIe前提の現実的な計算予算で回すための土台になっており、今週は4テーマがかなり噛み合っていた。

## 3. 金融AIエージェントへの示唆
### 今週のアクションアイテム（最大3つ）
- 社内文書・XBRL・決算資料向けに、埋め込み検索一本足から脱却し、`grep/read/script`型の直接コーパス操作と階層索引を併用する検索面プロトタイプを作る。
- LoRA/DPO後モデルに対して、構造化出力精度とは別に安全性・不確実性・help-seekingを測る再評価パックを作り、低信頼時の再検索・人手確認分岐を実装する。
- PCIe前提の学習基盤では、RoundPipe相当のパイプライン最適化と、MoEの部分起動戦略（EMO/UniPool/MASCing）を次の検証テーマとして優先する。

### 中期的に注視すべき方向
検索エージェントは、retriever改善よりも「証拠束をどう集めるか」「図表をどう根拠化するか」「どの時点で確認に戻るか」の統合設計へ進んでいる。金融向けでは、XBRL・表・PDF・IRスライド・社内メモを同じ作業面に置けるかが分水嶺になる。マルチエージェントは、役割分担の巧拙より、スキル棚卸し、停止判断、監査ログ、失敗履歴の資産化が主要テーマになっており、研究運用そのものを改善する基盤としても重要性が高い。アライメントは、追加学習で賢くする段階から、distribution driftと安全劣化を抑えつつ実務フローに接続する段階へ移った。

### 今週スキップしてよい理由
汎用GUIベンチやWebDevベンチの多くは、現状の限界確認としては有益だが、直近の金融AI改善には検索面・証拠管理・安全分岐ほど直接効かない。diffusion系やマルチモーダル生成寄りの論文群も、今週のプロジェクト課題である構造化出力、RAG、金融ツール利用への接続は弱い。まずは「金融コーパス探索」「Fine-tune後再監査」「PCIe前提MoE運用」の3本に絞る方が投資対効果が高い。
