# 📊 2026-W17 週次トレンド分析
期間: 2026-04-20 〜 2026-04-26 | 分析論文数: 106本

## 1. テーマ別ハイライト
### T1 MoE / 効率的アーキテクチャ
今週のT1は、MoEそのものの拡張戦略と、長文脈・低遅延推論を支えるキャッシュ/デコード制御の両輪が進んだ週だった。`Expert Upcycling` は小さめの学習済みMoEを段階的に増設する現実的な拡張路線を示し、PCIe前提で巨大MoEを一気に育てにくい自社条件にかなり合う。`Temporally Extended MoE` は token ごとの expert 切替を抑えて通信頻度そのものを減らす方向で、NVLinkなし環境のボトルネックに正面から効く。推論側では `KV Packet`、`River-LLM`、speculative decoding 系が「再計算しない」「早く降りる」「下書きを賢く使う」方向に収束しており、量子化済みMoEをどう運用で速くするかの設計解像度が一段上がった。

### T2 金融AI / ドメイン特化LLM
T2は、RAGを単なる検索から「探索可能な構造」へ変える流れがはっきりした。`Don't Retrieve, Navigate` と `UniDoc-RL` は、社内文書やPDFをフラットに引くのでなく、階層ナビゲーションや coarse-to-fine 行動で辿る設計を提示しており、XBRL・決算資料・IR文書にそのまま接続しやすい。評価面でも `ROSE`、`DAVinCI`、`Dual-View Training`、dense retriever の頑健性分析が出そろい、「取れたか」ではなく「意図に沿ったか」「根拠に帰属できるか」「ノイズや多言語でも崩れないか」を測る方向へ移った。金融文脈では、検索品質そのものより、根拠統合・claim検証・instruction-aware retrieval を一体で扱う必要があることが明確になった週だった。

### T3 マルチエージェント / ツール利用
T3は今週も最頻出で、焦点はモデル能力そのものより運用制御層にあった。`DR^3-Eval`、`AAR`、`GTA-2`、`AJ-Bench` などが示したのは、失敗の主因がツール呼び出しそのものではなく、正しい場所へ進むナビゲーション、途中評価、停止判断、再計画にあるという点だ。実装寄りには `SkVM`、`SemaClaw`、`VLAA-GUI`、`Scaling Test-Time Compute for Agentic Coding`、`TACO` が、スキルのコンパイル、ハーネス外出し、失敗軌跡圧縮、観測圧縮、judge/guardrail 分離を進めている。差分としては、先週までの「ツールを使えるか」から一歩進み、「長期軌跡をどう圧縮・再利用・検証するか」が中核テーマになった。

### T4 ファインチューニング / アライメント
T4は、単一スカラー報酬に依存した粗い後学習から、構造化された密な学習信号へ移行する流れが強かった。`TESSY`、`Self-Distillation Zero`、`C2`、`Target Policy Optimization`、`Near-Future Policy Optimization` は、教師と生徒の分布整合、自己改訂、rubric、target distribution、近未来checkpointといった形で、より実装可能な密信号を持ち込んでいる。一方で `Where does output diversity collapse in post-training?`、`Reward Hacking in the Era of Large Models`、`Benign Fine-Tuning Breaks Safety Alignment` は、学習を強く回すほど多様性低下・proxy最適化・安全性劣化が起きうることも明確化した。今週の含意は、金融向けSFT/DPO/RLでも性能向上だけでなく、候補多様性・abstention・calibration を同時に見ないと構造化出力品質はむしろ劣化しうる、という点にある。

## 2. テーマ横断マップ
今週いちばん強い横断テーマは、「検索・推論・行動の途中状態を、再利用可能で検証可能な資産に変える」流れだった。T2では `Don't Retrieve, Navigate` や `DAVinCI` が文書構造や claim 帰属を外在化し、T3では `SkVM`、`Scaling Test-Time Compute`、`VLAA-GUI` が skill・rollout・停止/回復判断を外在化し、T4では `C2`、`TPO`、`NPO`、`SD-Zero` が報酬や改訂軌跡を学習信号として構造化している。これは「最終回答だけを最適化する系」から、「途中経路を管理する系」への重心移動と見てよい。

もう一つの接続は、T1の効率化がT3/T4の長期運用を支え始めたことだ。`Expert Upcycling` と `Temporally Extended MoE` はMoE拡張と通信削減を、`KV Packet` や early-exit / speculative 系は再計算削減を進めており、長いRAG文脈や複数ロールアウトを前提にした agentic workload を現実のGPU制約の中へ引き戻している。つまり今週は、T1が単独最適化ではなく、T2/T3/T4の運用基盤を成立させる下支えとして接続された週だった。

## 3. 金融AIエージェントへの示唆
### 今週のアクションアイテム（最大3つ）
- 社内RAGの次段として、`Don't Retrieve, Navigate` を参考に文書ツリー化した探索型プロトタイプを作る。対象はまず決算資料・XBRL・社内メモの3系統で、検索精度より根拠統合と監査性を優先する。
- 構造化出力評価を `ROSE` と `DAVinCI` の考え方で見直す。SQL/JSON/財務判断メモに対して、「実行可否」や「形式一致」だけでなく、意図整合・claim帰属・不足情報時の abstain/clarification を評価軸へ入れる。
- 推論基盤では `KV Packet` と `Expert Upcycling` を優先検討する。前者は頻出コンテキスト再利用、後者は段階的MoE拡張として、RTX 6000 Pro × PCIe の制約下でも比較的着手しやすい。

### 中期的に注視すべき方向
- RAGは retrieval-centric から navigation-centric へ移る可能性が高い。金融文書は階層性と相互参照が強いため、この変化の恩恵を受けやすい。
- 後学習は binary reward 一本足ではなく、rubric・自己改訂・近未来checkpoint・target policy など複合的な密信号化へ進みそうだ。構造化出力品質を最重要とする現状と相性が良い。
- エージェント基盤は、モデル本体の強化より harness・memory・guardrail・judge・trajectory compression の差で性能が開く段階に入っている。金融用途ではこの層がそのまま競争優位になる。

### 今週スキップしてよい理由
今週はGUI・モバイル・汎用コンピュータ操作ベンチも多かったが、金融AIエージェントの直近課題は GUI 自動化そのものではなく、RAG監査性、構造化出力品質、MoE推論効率、後学習安定化にある。したがって、GUI操作の生性能や一般用途 personal agent の広い能力競争は追い過ぎなくてよい。追うべきなのは、その中で抽出できる停止判断、回復、guardrail、trajectory reuse といった再利用可能な設計原理に限る。
