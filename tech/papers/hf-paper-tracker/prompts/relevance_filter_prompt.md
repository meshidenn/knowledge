# 関連度フィルタプロンプトテンプレート

## 使い方

1. `claude "今週の論文に関連度フィルタかけて"` で実行、または日次インテークの出力を渡す
2. 日次インテークの出力を入力にして、自分のプロジェクトとの関連度でスコアリングする
3. 「読むべき順序」と「スキップしてよい理由」を明確にする

> **日次インテークとの使い分け:**
> - インテーク = テーマに該当するか（広く拾う）
> - 関連度フィルタ = 自分のプロジェクトに使えるか（厳しく絞る）
>
> 忙しい週は関連度フィルタだけ週末にまとめて回してもOK。

---

## システムプロンプト（追加分）

日次インテーク用のシステムプロンプトに以下を **追記** する。

```
### プロジェクトコンテキスト（関連度フィルタ用）

以下は私が現在取り組んでいるプロジェクトの具体的な構成です。
論文の関連度を判定する際、この情報を基準として使ってください。

#### プロダクト
- 金融専門家向けAIエージェントシステム
- ユースケース: 投資判断支援、ポートフォリオ管理、M&Aバリュエーション

#### アーキテクチャ
- ベースモデル: MoEモデル（Qwen3.5-397B-A17B級）
- 推論環境: RTX 6000 Pro × 複数枚、PCIe接続
- 推論戦略: Expert Parallelism + AWQ/INT4量子化
- ファインチューニング: LoRA/QLoRA によるドメイン特化SFT
- 最重要改善ターゲット: 構造化出力スキーマの品質

#### 技術スタック
- マルチエージェントオーケストレーション
- RAG（社内文書・XBRL・決算資料）
- DPO/KTO によるアライメント
- MCP連携（Google Workspace）

#### 現在の課題（優先度順）
1. 金融専門用語の精度向上（SFTデータ品質）
2. 規制コンプライアンス準拠のハルシネーション抑制
3. 社内データソースとの統合（RAGパイプライン）
4. リアルタイム市場データへのアクセス

#### 技術的制約
- PCIe環境（NVLink非搭載）→ Expert Parallelism前提
- メモリ制約 → 量子化必須（FP16フルは不可）
- 本番環境のレイテンシ要件 → 推論速度も重要
```

## 関連度フィルタプロンプト

### 標準版（日次インテークの出力に対して実行）

```
以下の論文リストに対して、私のプロジェクトとの関連度でスコアリングしてください。

## 今週の論文
{{日次インテークの出力、または論文タイトル+URLのリストを貼る}}

---

## スコアリング基準

各論文を以下の5段階で評価してください:

| スコア | 意味 | アクション |
|--------|------|-----------|
| S | 即座に読んで実装検討 | 今週中にアブストラクト+手法を精読 |
| A | 高い適用可能性 | 今週中に要点を把握 |
| B | 部分的に参考になる | 週次分析で言及、余裕があれば読む |
| C | 間接的な関連のみ | タイトルだけ記録、深追いしない |
| - | スキップ | 理由を1文で記載 |

## 評価の軸（重み付き）

1. **直接適用性（40%）**: 手法やアーキテクチャを自分のシステムにそのまま使えるか
2. **課題一致度（25%）**: 自分の「現在の課題」リストのどれかを解決しうるか
3. **環境互換性（20%）**: PCIe/メモリ制約の中で実際に試せるか
4. **タイミング（15%）**: 今のフェーズで必要か（将来的に有用でも今は不要なら下げる）

## 出力フォーマット

### スコアボード

| スコア | 論文タイトル | 主な理由 |
|--------|-------------|---------|
| S | ... | ... |
| A | ... | ... |
| ... | ... | ... |

スコアの高い順にソートしてください。

### S/Aランク詳細

Sランク・Aランクの論文のみ、以下を追記:

```
#### [S] 論文タイトル
- **適用先**: {{プロダクトのどのコンポーネントに使えるか}}
- **適用方法**: {{具体的にどう組み込むかの仮説、2〜3文}}
- **前提条件**: {{試すために必要な準備、データ、環境変更}}
- **リスク**: {{うまくいかない可能性とその理由}}
- **次のアクション**: {{最初の1ステップ}}
```

### スキップ理由サマリー

スキップした論文（C/-）について、パターンを分析:
- 「テーマには合致するが自分の環境では試せない」が多い場合 → 環境拡張の検討
- 「今のフェーズでは不要」が多い場合 → 将来読むリストに移動
- 「ドメインが違う」が多い場合 → テーマキーワードの精緻化
```

### 速攻版（論文タイトルだけで高速判定）

タイトルとURLしかない場合（arXivのリストを直接貼る時など）:

```
以下の論文タイトルを、私のプロジェクトとの関連度で3段階に振り分けてください。
アブストラクトは読まなくてOK。タイトルとキーワードからの推定で構いません。

🔴 読むべき / 🟡 余裕があれば / ⚪ スキップ

各論文に振り分け理由を1文で。

## 論文リスト
{{タイトルとURLを貼る}}
```

### 比較版（2本以上の候補からどれを優先するか）

```
以下の論文のうち、私のプロジェクトにとってどれを優先的に読むべきか判定してください。

## 候補論文
{{2〜5本の論文タイトル+URLを貼る}}

## 判定してほしいこと
1. **優先順位**: 1位〜N位のランキングと理由
2. **読む深さの推奨**: 各論文について
   - 精読（手法セクションまで）
   - 斜め読み（アブスト+結論）
   - アブストのみ
3. **組み合わせ価値**: 複数論文の手法を組み合わせると面白い場合は指摘
```

### arXiv URL直接入力版（web searchでアブストラクトを取得して判定）

```
以下のarXiv論文について、アブストラクトを取得した上で関連度を評価してください。

{{arXiv URLを1〜5本貼る}}

各論文について:
1. スコア（S/A/B/C/-）
2. アブストラクトの要点（3行）
3. 私のプロジェクトとの接点（具体的に）
4. Sランクの場合のみ: 次のアクション
```

## 週次バッチフィルタ

週末に1週間分をまとめてフィルタする場合:

```
今週（{{MM/DD}}〜{{MM/DD}}）の日次インテークで拾った論文を
まとめて関連度フィルタにかけてください。

## 今週の論文
{{1週間分の日次インテーク出力を貼る}}

## 追加指示
- S/Aランクの論文は週次トレンド分析の入力として使うので、
  テーマ横断の接続があれば明示してください
- 先週のS/Aランク論文: {{先週のS/A論文タイトルを列挙}}
  → 先週との連続性・発展があれば指摘
- 今週のスキップパターンを分析して、テーマキーワードの
  更新提案があれば末尾に追記
```

## 運用サイクルの全体像

```
月〜金 朝5分          週末 30分               週末 15分
┌──────────┐     ┌──────────────┐     ┌──────────────┐
│ 日次      │     │ 関連度       │     │ 週次          │
│ インテーク │ ──→ │ フィルタ     │ ──→ │ トレンド分析  │
│ （広く拾う）│     │（厳しく絞る） │     │（俯瞰する）   │
└──────────┘     └──────────────┘     └──────────────┘
                        │
                        ↓ S/Aランク論文
                  ┌──────────────┐
                  │ 論文深掘り    │  ← 随時（1本15分）
                  │ （実装検討）  │
                  └──────────────┘
```

### 時間の目安

| 作業 | 頻度 | 所要時間 | 累計/週 |
|------|------|---------|---------|
| 日次インテーク | 平日5回 | 5分 | 25分 |
| 関連度フィルタ | 週1回 | 15分 | 15分 |
| 週次トレンド分析 | 週1回 | 30分 | 30分 |
| 論文深掘り | 週1〜2本 | 15分/本 | 15〜30分 |
| **合計** | | | **85〜100分/週** |
